# REQUIREMENTS — Фаза 02: OLAP-отчёты (drill-down)

**Дата:** 2026-06-21
**База:** `docs/research/OLAP_REPORTS_RESEARCH.md`, запрос заказчика (кликабельные взаимосвязанные отчёты с детализацией по человек/проект/отдел/вид работ/категория + хлебные крошки).
**Безопасность:** server-RBAC невозможен без server-identity (CISO-005) → детализация по людям откладывается до identity. Обезличенные оси — безопасны.

---

## v1 — в скоупе (обезличенные оси, зелёная зона)

- **OLAP-01.** `/s/reports` обобщён до параметров: `groupBy` (любая ось), `filters[]` (контекст drill), курсорная пагинация. Обратная совместимость со старым ответом (дашборд + виджет «Бюджет» не сломать).
- **OLAP-02.** Оси drill: Отдел, Проект, Этап, Вид работ, Категория, Период. (Сотрудник — отдельно, v2 за RBAC.)
- **OLAP-03.** Drill-down: клик по строке → переход на дочернюю ось с накоплением фильтра. Состояние — React useReducer (стек уровней), без URL/host-DOM.
- **OLAP-04.** Cross-filter: клик по ячейке/категории добавляет фильтр без смены оси.
- **OLAP-05.** Хлебные крошки из стека drill, возврат на любой уровень.
- **OLAP-06.** UX impeccable (product): кликабельные строки (role=button+клавиатура), skeleton, пусто/ошибка/RBAC-состояния, tabular-nums, без модалок/host-DOM/градиент-текста.
- **OLAP-07.** Валидация параметров запроса на сервере (whitelist осей/фильтров, лимиты) — CISO-006.
- **OLAP-08.** Связь с «Людьми»: page-layout-tab «Трудозатраты» с FRONT_COMPONENT на record-page Employee/Department → предзаполнение `initialFilters` (один OLAP-движок).

## v2 — отложено (за RBAC/identity)

- **OLAP-09.** Срез/детализация **по сотруднику** + лист записей — редакция ПДн (CISO-007) + server-RBAC (руковод видит свой отдел, сотрудник — себя). Зависит от **CISO-005** (server-identity в logic) + **152FZ-001** (локализация для прода).

## Вне скоупа

- Экспорт OLAP (→ F-F отдельно). Сохранённые OLAP-виды (→ UI-B). Графики/чарты сверх баров (по потребности).

## Риски (из research)
- **R-IDENTITY** — logic не получает актора (RoutePayload без роли) → блок полного RBAC. Митигация: v1 обезличен.
- **R-RECORDID** — `useRecordId` deprecated → проверить актуальный API record-page (OLAP-08).
- **R-CONTRACT-COMPAT** — старый ответ читают дашборд + виджет «Бюджет» → расширять, не ломать (OLAP-01).
