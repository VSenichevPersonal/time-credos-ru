## ЧАСТЬ 1. TIMETTA

### Архитектура данных ввода (что вводим)
Три уровня, видно из `raw-timesheet-full.json`:

```
TimeSheet (неделя, Пн–Вс)
 └─ TimeSheetLine (Клиент → Проект → Работа → Вид работ → Тариф)   ← строка сетки
      └─ TimeAllocation (часы за конкретный день + комментарий)     ← ячейка дня
```

- **Часы — Decimal**, не HH:MM. `2.5` = 2 ч 30 мин (`TimeInputType: Decimal`, альтернатива HH:MM и «% от расписания»).
- `TimeAllocation` = одна ячейка (строка × день). У неё свои `hours`, `description`, `isBillable`, `stopwatchStarted`, опционально `issueId` (связь с задачей).
- `rowVersion` — оптимистичная блокировка на Line и Allocation.
- `TimeSheet.schedule[]` — план по дням недели (`[{date, hours:8}×5, {hours:0}×2]`) → выходные затемнены в сетке.

### Режимы просмотра/ввода
Из скриншота `app-main-page.png` (Моя работа → Учёт → Текущий таймшит):

1. **Недельная сетка (основной режим)** — главный экран. Колонки: чекбокс · **Клиент > Проект > Работа** · **Вид работ** · **Тариф** · 7 колонок дней (15.06…21.06, выходные красным/штриховкой) · **Итог** (по строке). Внизу — строка **Итого по дню** и (в макете) **План**.
2. **Детали / Активность / Вложения / Задания / Состояния / История** — вкладки под сеткой. «Детали» — плоский список аллокаций (Дата · Клиент>Проект>Работа · Вид работ · Тариф · Часы · Комментарий) с сортировкой.
3. **Таймшиты** (`shared.groups.timesheets`) — история своих недельных таймшитов (список периодов).
4. **Согласование** (`shared.groups.approval`) — режим менеджера: входящие на утверждение.
5. **По периодам** (`shared.groups.byPeriods`) — агрегаты.
6. **Доска / Список** — переключатель вида для проектов/клиентов (видно на `app-projects.png`, `app-clients.png`), к таймшиту напрямую не относится, но паттерн toggle используется везде.

Навигация по неделям: `‹ 15 июня 2026 — 21 июня 2026 ›` с заголовком-датой, кнопки **Предыдущий/Следующий период** + «Выбор периода» (`shared.actions.previousPeriod / nextPeriod / setPeriod`).

### Структура строки и способ ввода
- Кнопка **+ Добавить строку** (`shared.actions.addLine`) открывает выбор Клиент→Проект→Работа→Вид работ (каскад). Поля строки настраиваются шаблоном (см. ниже).
- Часы вводятся **inline прямо в ячейку дня** (десятичное число). Комментарий — на уровне аллокации (видно в «Деталях»: «Готовился к докладу…», «Не работал компьютер, ждала ИТ-ешников»).
- **Секундомер** (`useStopwatch:true`): `stopwatchStart` = «Запустить счётчик», `stopwatchStop` = «Остановить счётчик и учесть записанное время». Пишет в `stopwatchStarted` аллокации.
- **Отсутствия** вводятся отдельной секцией: `addTimeOffLine` / `addTimeOff` («ЗАЯВКИ НА ОТСУТСТВИЯ»), тип из справочника TimeOffTypes (П/БО/Д/У/Б).

### Колонки-шаблоны (TimesheetTemplate)
Из `raw-timesheet-full.json` → `template{}` — какие колонки показывать настраивается:
`showActivity` (Вид работ), `showRole` (Роль), `showClient` (Клиент), `showProjectCostCenter` (Центр затрат), `showTariff` (Тариф), `showBillCode` (Код оплаты) + кастомные поля. В нашем примере: client+activity+tariff = on, role/costCenter/billCode = off.

### Фильтры
- Каждый список (`Поиск…` + иконка-воронка + **«Сохранённый фильтр»** dropdown + «Период») — см. `settings-*.png`, `app-projects.png`.
- **Сохранённые фильтры** — это ключевая фича Timetta (dropdown «Сохранённый фильтр» на каждом гриде).
- В фильтрах учёта (i18n): `period.ph` = Фильтр по периоду, `timeOffType.ph` = Фильтр по типу отсутствия, `forPeriod` = За период.
- Группировка строк (`lineGrouping`, `rowGroups`, `byPeriods`).

### Итоги / агрегация
- **Итог по строке** (колонка «Итог») + **Итого по дню** (нижняя строка) + **План по дню** из `schedule`.
- `billableDuration` / `nonBillableDuration` — оплачиваемые/неоплачиваемые часы на уровне таймшита.
- Подсказка под днём: `dayHintWorkHours` = «По расписанию {{hours}} ч.» — показывает план под каждым днём.
- **Правила валидации** (`settings-timesheet-periods`, ValidationRules): «День +/- 2 ч от расписания» (`durationShortening`/`durationExcess`), «Рабочий день до 24 ч». Это подсветка отклонений факт-vs-план.
- «Итоги» / «Без итогов» / «Итого» — режимы агрегации (`shared.props.totals`, `noTotals`, `total`).

### Клавиатура / скорость ввода (главное, что делает Timetta быстрым)
- **Копировать строки из предыдущего** (`copyLinesFromPrevious`) — переносит структуру строк прошлой недели.
- **Копировать строки и часы из предыдущего** (`copyLinesWithHours`) — переносит структуру **вместе с часами**.
- **Создать строки по ресурсному плану** (`createLinesFromResourcePlan`) и **…с часами** (`createLinesFromResourcePlanWithHours`) — автозаполнение из планирования.
- Буфер обмена: `copyLinesToClipboard` / `pasteLinesToClipboard` (копи-паст строк), `moveLinesUp/Down`, `removeLines`.
- Контекстное меню ячейки: «Добавить строку до / после» (`addRowBefore` / `addRowAfter`).
- Пустое состояние: «Ваш таймшит пока пустой. **Добавьте** новую строку или **скопируйте** строки из предыдущего» (`tableViewComponent.messages.empty`).

### Жизненный цикл и отправка
- Кнопка **«Отправить на согласование»** (вверху экрана). Статусы: **Черновик → На согласовании → Согласовано** (или **Отклонено → Черновик**). State badge «Черновик» виден на скрине.
- `Срок отправки: 23.06.2026` (dueDate) + `Автор`.
- `revoke` = Отозвать (вернуть из согласования).
- Период согласования: недельный, dueDate = вторник след. недели.

## ЧАСТЬ 2. KIMAI

### Архитектура данных (проще — 3 уровня, плоские записи)
```
Customer → Project → Activity → Timesheet (одна запись = один интервал работы)
```
Timesheet — **не сетка, а отдельная запись** (`entity-Timesheet.php`):
- `begin` (DATETIME) · `end` (DATETIME, null = таймер бежит) · `duration` (INT, **секунды**) · `break` (перерыв, сек).
- `project` + `activity` (FK, оба обязательны) · `description` (TEXT) · `tags` (M:N) · `user`.
- `rate` / `internalRate` / `fixedRate` / `hourlyRate` · `billable` (bool) · `billableMode` = auto/yes/no/default.
- `exported` (bool) — **после экспорта запись блокируется** от правок.
- `metaFields` — кастомные поля key-value без миграций.
- `category` — work/holiday/sickness/parental/overtime (deprecated, осталось для совместимости).

### Режимы просмотра/ввода (из `controller-list.txt`)
1. **Список таймшитов** (`TimesheetController`) — таблица записей: дата, проект, активность, описание, теги, длительность, ставка. Сортировка/фильтр/пагинация (`page`,`size`).
2. **QuickEntry / быстрый ввод** (`QuickEntryController`) — **недельная сетка a-la Timetta**: строки Проект×Активность, колонки дней недели, в ячейку вводишь длительность. Это прямой аналог нашей «Недели».
3. **Календарь** (`CalendarController`) — дневной/недельный календарь с drag-блоками времени (как Google Calendar). Создание записи перетаскиванием.
4. **Reporting**: `UserWeek` / `UserMonth` / `UserYear` (мои часы), `ProjectDetails`, `CustomerMonthlyProjects`, `ProjectDateRange` — агрегированные отчёты.
5. **Dashboard** (`DashboardController`) — виджеты (часы за день/неделю/месяц, активные таймеры).
6. **TimesheetTeam** (`TimesheetTeamController`) — режим менеджера: чужие таймшиты команды.
7. **Favorites / Recent** (`FavoriteController`, индекс `IDX_TIMESHEET_RECENT_ACTIVITIES`) — избранные и недавние комбинации Проект+Активность для мгновенного старта.

### 3 режима учёта времени (конфигурируемо глобально)
1. **Duration-only** — вводишь только длительность (без begin/end).
2. **Begin/End times** — задаёшь начало и конец, длительность считается.
3. **Punch-in/punch-out** — таймер: жмёшь «старт» → бежит `end=null` → «стоп».
   + **Мульти-таймер** — несколько одновременно бегущих записей.

### Структура формы ввода
Поля: Проект (с автоподстановкой Клиента) · Активность (глобальная или проектная) · Начало/Конец **или** Длительность · Описание · Теги · Billable (галка) · ставка (обычно auto). Кнопка **Duplicate** (`createCopy()` / `__clone()`) дублирует запись (с `exported=false`).

### Фильтры (из RECON + reporting)
Customer · Project · Activity · **Tags** · **DateRange** (диапазон дат) · User (для менеджера) · billable · exported. Множественный выбор клиента/проекта/активности/тегов. Закладки-фильтры через `BookmarkController` (сохранённые наборы фильтров).

### Итоги / агрегация / биллинг
- Длительность × ставка → `rate`. Ставка берётся по приоритету: **Timesheet → Activity → Project → Customer → User** (более специфичная побеждает).
- **Бюджеты** (`BudgetTrait`): денежный `budget` + временной `timeBudget` на уровнях Customer/Project/Activity, прогресс-бары и % выполнения.
- Отчёты Week/Month/Year — суммы по дням/проектам/активностям.
- Инвойсинг: PDF/HTML/DOCX/ODS/XLSX/CSV из неэкспортированных записей.

### Клавиатура / скорость
- **Recent activities** — топ недавних Проект+Активность одним кликом запускает таймер.
- **Duplicate** записи.
- **QuickEntry** недельная сетка — массовый ввод за неделю без открытия модалок.
- **Restart** недавнего таймера.
- Глобальные настройки дефолтов (billable по умолчанию, режим времени).

## ЧАСТЬ 3. ЧТО ВЗЯТЬ В НАШ TIMESHEET

### Модель данных — гибрид
- **Trinity Timetta**: `TimeSheet(неделя) → Line(Клиент>Проект>Работа>ВидРабот) → Allocation(день+часы+коммент)`. Это ядро.
- **Часы Decimal** (Timetta) — `2.5`, не секунды Kimai. Читаемее в API/UI.
- **exported-флаг** (Kimai) — блокировка записи после выгрузки/согласования (аудиторский след).
- **Meta key-value** (Kimai) — кастомные поля без миграций.
- **rowVersion** (Timetta) — оптимистичная блокировка ячеек при параллельном редактировании.

### Режим «НЕДЕЛЯ» (главный экран — из Timetta + Kimai QuickEntry)
- Сетка: **колонка-связка** `Клиент > Проект > Работа` · опц. колонки `Вид работ / Роль / Тариф / Центр затрат` (вкл/выкл через TimesheetTemplate) · **7 колонок дней** · **Итог по строке**.
- Нижние строки: **Итого по дню**, **План по дню** (из schedule), подсветка отклонения «факт vs план ±2ч».
- Под каждым днём подпись «По расписанию N ч.». Выходные — штриховка/приглушение.
- Ввод часов **inline в ячейку** (Decimal). Комментарий — попап на ячейке.
- Кнопки скорости: **+ Добавить строку**, **Копировать строки из предыдущей недели** (со структурой) и **…с часами**, копи-паст строк, контекст-меню «строка до/после».
- Навигация неделями `‹ дата — дата ›` + выбор периода.
- **Отправить на согласование** + бейдж статуса (Черновик/На согласовании/Согласовано/Отклонено), dueDate.
- Отдельная секция **Заявки на отсутствие** (типы П/БО/Д/У/Б).

### Режим «ДЕНЬ» (из Kimai Calendar + список)
- Один день: список записей за день ИЛИ календарь-таймлайн (блоки времени, drag для создания).
- Быстрый ввод: **таймер** (старт/стоп, `stopwatchStarted`) + **Recent** (недавние Проект+Активность одним кликом) + **Дублировать** запись.
- Поддержать 2 способа ввода времени: **Длительность** (Decimal) и **Начало–Конец** (для тех, кто хронометрирует). Duration — дефолт.

### Режим «ПРОЕКТ» (из Timetta projects + Kimai reporting)
- Список/Доска проектов (`app-projects.png`): Наименование · Код · Организация · Менеджер · Состояние (бейдж) · Начало/Окончание.
- Внутри проекта: **План vs Факт часов**, прогресс-бар бюджета (время+деньги, Kimai BudgetTrait), разбивка по сотрудникам/работам/периодам.
- Отчёты Week/Month/Year (Kimai) + «Часы проектов по месяцам», «Утилизация сотрудников» (Timetta Reports).

### Фильтры (объединить оба)
- Набор: **Клиент · Проект · Вид работ/Активность · Период (DateRange) · Пользователь · Теги · Статус · Billable**.
- **Множественный выбор** по клиент/проект/активность/теги.
- **Сохранённые фильтры** (Timetta «Сохранённый фильтр» / Kimai Bookmarks) — обязательно, на каждом гриде.
- Глобальный поиск `Поиск…` + иконка-воронка.

### Клавиатурный / быстрый ввод (приоритет UX)
1. **Tab-навигация** по ячейкам сетки слева-направо, Enter — следующая строка.
2. **Копировать прошлую неделю** (структуру и/или с часами) — кнопка №1 по экономии времени.
3. **Recent / избранные** комбинации Проект+Работа — однокликовое добавление строки.
4. **Таймер** для тех, кто учитывает в реальном времени; ручной Decimal для остальных.
5. **Дублирование** строки/записи, копи-паст строк через буфер.
6. Inline-валидация: подсветка дней с недозагрузом/превышением относительно плана; «Заполните обязательные поля» перед отправкой.
7. **Автозаполнение из ресурсного/проектного плана** (Timetta `createLinesFromResourcePlan`) — если есть планирование.

### Итоги / биллинг
- Итог по строке + Итого по дню + Итого по неделе + План.
- Billable / non-billable разделение (`isBillable` на аллокации, billableMode auto/yes/no).
- Ставки по приоритету Allocation→Activity→Project→Customer→User (Kimai-каскад) поверх ProjectTariff/RateMatrix (Timetta).
- Бюджеты время+деньги на Клиент/Проект/Работа с прогресс-барами (Kimai).

### Резюме одной строкой
**Берём недельную сетку Line→Allocation и Decimal-часы из Timetta; таймер, Recent, календарь-день, Meta-поля, exported-флаг, каскад ставок и бюджеты из Kimai; сверху — сохранённые фильтры, копирование прошлой недели и Tab-ввод как главные ускорители.**
