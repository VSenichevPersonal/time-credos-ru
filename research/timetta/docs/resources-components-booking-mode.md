# Режимы бронирования

Обновлено: 05.08.2025

## Назначение

Режим бронирования ресурсов задаётся в области Настройки и применяется ко всей системе.

**Права доступа**

Для настройки режима бронирования требуется набор прав для роли Администратор.

В системе есть два режима бронирования:

1. Базовый — предполагает бронирование ресурсов в часах в виде плашек;
2. Детальный — предполагает ввод часов брони в ячейки календаря бронирования.

**Внимание**

Изменение режима бронирования — необратимая операция. При смене режима некоторые элементы бронирования могут быть утрачены без возможности восстановления.

В момент смены режима все активные сессии пользователей в системе будут завершены (потребуется обновление страницы). Также понадобится перезапустить открытые запросы — остановить воркфлоу и запустить заново.

## Базовый режим

В базовом режиме применяются типы бронирования Soft и Hard, а также методы бронирования. Подробнее о них можно узнать в разделе Типы и методы бронирования.

**Внимание**

При переключении режима бронирования с базового на детальный данные о типах Soft и Hard, а также о методах бронирования теряются.

## Детальный режим

В детальном режиме нет разделения на типы бронирования Hard и Soft и применения методов бронирования. Вместо этого бронирование происходит путём ввода конкретных часов в ячейки календаря.

**Внимание**

При переключении режима бронирования с детального на базовый все бронирования перейдут в тип Hard, а в качестве метода будет выбрано ручное планирование.
