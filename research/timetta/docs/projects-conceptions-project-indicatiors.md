# Типы показателей проекта

Обновлено: 14.08.2025

## Основные показатели проекта

Для управленческого учёта в системе есть следующие базовые показатели:

- **Часы** — трудозатраты по проектам
- **Себестоимость** — стоимость трудозатрат по проектам
- **Затраты** — прочие прямые затраты на проекты
- **Выручка** — выставленная клиенту цена выполненных по проектам работ
- **Биллинг** — денежный поток выставленных к оплате счетов
- **Общая длительность проектов** — продолжительность проекта в календарных днях

Производные показатели (например, рентабельность) строятся на основании базовых.

Выручка, себестоимость и затраты — финансовые показатели по методу начисления. Биллинг — выручка по кассовому методу.

## Типы показателей проекта

Для всестороннего план-фактного анализа есть следующие типы показателей:

- **Оценка (Estimated).** Текущие плановые показатели по проекту. Оценка должна быть актуальна и может постоянно корректироваться со стороны команды проекта.
- **План (Planned).** Согласованная и утверждённая в качестве мастер-версии оценка. Плановые показатели чаще всего фиксируются на старте проекта и не могут меняться после их утверждения. При этом установленный план может быть заменён на его альтернативную версию, если пересмотрены условия проекта (например, изменён контракт).

### Особенности плановых показателей

- У проекта может быть несколько версий в состоянии «Утверждённая». Одна из них должна быть помечена как мастер-версия. Именно она — источник плановых показателей.
- Права на утверждение версий могут быть у Менеджера проекта или только у отдельного сотрудника, например Финансового контролёра.
- Редактирование утверждённой версии проекта невозможно. Базовому плану должны доверять, а для этого он должен быть защищён от любых изменений.

- **Факт (Actual).** Фактические показатели отражают реальное положение дел на проектах. Они строятся на основании подтверждённых работ и финансовых операций (согласованные трудозатраты в таймшитах, оплаченные счета и так далее).

## Связь данных с показателями

Показатели в отчётах используют данные из определённых компонент. Таблица ниже показывает компоненты — источники данных для всех базовых показателей и их типов.

|  | Оценка | План | Факт |
|---|---|---|---|
| **Часы** | Ресурсный план (в карточке проекта) | Из версии проекта в состоянии «Утверждённая», отмеченной как мастер-версия | Таймшиты (в большинстве случаев только согласованные) |
| **Себестоимость** | Ресурсный план (в карточке проекта) | Из версии проекта в состоянии «Утверждённая», отмеченной как мастер-версия | Таймшиты (в большинстве случаев только согласованные таймшиты и с нормализацией) |
| **Выручка** | Выручка проекта (в карточке проекта) | Из версии проекта в состоянии «Утверждённая», отмеченной как мастер-версия | Выручка проекта (в карточке проекта) |
| **Затраты** | План затрат (в карточке проекта) | Из версии проекта в состоянии «Утверждённая», отмеченной как мастер-версия | Заявки на затраты (в большинстве случаев только согласованные) |
| **Биллинг (денежный поток)** | План биллинга (в карточке проекта) | Из версии проекта в состоянии «Утверждённая», отмеченной как мастер-версия | Счета (в большинстве случаев только согласованные) |

## Разрезы показателей

Для детального анализа доступны измерения (разрезы) показателей.

У всех показателей есть следующие измерения:

- **Времени** — дата, неделя, месяц и год.
- **Проектные** — проект, клиент, программа, этап проекта (работа верхнего уровня).

Специфические измерения:

- **Часы и себестоимость** — измерение «Роль» (проектная роль).
- **Фактические часы и себестоимость** — организационные измерения «Подразделение», «Ресурсный пул», «Пользователь».
- **Затраты** — измерения «Тип затрат» и «Оплачиваемые» (Pass-Through Expense).

Также измерения — любые свойства базовых измерений-сущностей, например Менеджер проекта (свойство проекта) или Руководитель сотрудника (свойство пользователя).

Кроме того, в системе можно создавать дополнительные поля, которые автоматически становятся измерениями для выбранной при создании сущности.

## Анализ показателей проектов в отчётах

Отчёты и их показатели:

- **Бронирование** — содержит показатели, связанные с забронированной ёмкостью ресурсов (часы, себестоимость), а также аналогичные плановые, оценочные и фактические показатели по проектам для сравнения их между собой. Максимальный уровень детализации — ресурс (либо пользователь) + проект.
- **Проекты** — содержит плановые, оценочные, прогнозные и фактические показатели с детализацией по проектам/клиентам/проектам. Показатели следующих категорий: часы, себестоимость, затраты, выручка, прибыль, рентабельность.
- **Работы проектов** — детализирует все показатели из отчёта «Проекты» до уровня работ.
- **Ресурсный план** — содержит плановые, оценочные и фактические часы по ресурсным планам проектов с детализацией до ресурса (либо пользователя) + работ проектов. Показатели: часы, себестоимость, выручка T&M.
- **Финансы (оперативные)** — содержит показатели, аналогичные финансовому отчёту (P&L). Это плановые, оценочные и фактические суммы по разным статьям (выручка, затраты, себестоимость). Детализация до работы проекта.

## Тип показателей «Прогноз»

**Прогноз (Forecasted)** — производный тип показателя, который строится как факт до начала отчётного периода и как оценка, начиная с отчётного периода. Прогноз служит для актуализации плана. На базе скопированного прогноза можно утвердить новую версию плана с учётом текущих реалий проекта. Подробнее — Принципы прогнозирования.
