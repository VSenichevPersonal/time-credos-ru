# Управление пользователями

## Назначение

В справочнике содержится список пользователей Timetta и определяются настройки для каждого из них.

Справочник **Пользователи** доступен при наличии набора прав для роли **`Администратор`** с активными гранулами: «Пользователи»; «Пользователи: Профили»; «Группы»; «Права доступа»; «Себестоимость»; «Виды работ»; «Расписания»; «Настройки»; «Заместители» — для доступа к настройке каждой из вкладок.

## Профиль

На вкладке содержится основная информация о пользователе. Список отображаемых полей:

- **Имя** — отображаемое имя пользователя
- **Эл. почта** — адрес электронной почты пользователя с выбором домена организации
- **Провайдер аутентификации** — способ аутентификации пользователя в системе
- **Действующий** — пользователь доступен для выбора в качестве члена команды проектов
- **Код** — код пользователя, используется в аналитике и отчётах
- **Телефон** — телефон пользователя
- **Должность** — должность пользователя
- **Начало работы** — дата начала работы сотрудника в компании
- **Окончание работы** — дата окончания работы сотрудника в компании
- **Субподрядчик** — пользователь как внешний исполнитель

Группа полей «Ресурс» содержит:

- **Руководитель** — непосредственный руководитель пользователя
- **Подразделение** — подразделение пользователя
- **Локация** — офис или площадка
- **Ресурсный пул** — ресурсный пул пользователя
- **Юридическое лицо** — юридическое лицо пользователя
- **Уровень** — должностной уровень
- **Грейд** — грейд пользователя
- **Роль** — роль пользователя
- **Компетенция** — компетенции пользователя
- **Тип трудоустройства** — тип трудоустройства из справочника

На вкладке профиля могут отображаться дополнительные поля, добавленные администратором.

### Действия:

- **Добавить альтернативную почту для уведомлений** — указать произвольную почту для получения уведомлений

Если указана альтернативная почта, на основную почту уведомления направляться не будут, но основная почта по-прежнему используется для аутентификации.

- **Изображение профиля** — аватар пользователя

На панели инструментов в меню **Управление** доступны действия:

- **Отправить приглашение на вход** — отправить письмо с приглашением на вход
- **Установить пароль** — принудительно установить новый пароль
- **Удалить конфигурацию OTP** — восстановить доступ к системе при потере доступа к приложению-аутентификатору

Чтобы пользователь мог выполнить вход в Timetta, у него должен быть активен признак **Действующий** и заданы права доступа.

## Лицензии

На вкладке перечисляются приложения, назначенные пользователю.

## Группы

На вкладке перечисляются группы, в которые включён пользователь.

Добавление пользователя в группы влияет на:

- Итоговые права доступа (объединяются индивидуальные права и права через группы)
- Доступные пользователю воркфлоу

Для добавления пользователя в группы:

1. Нажать на кнопку **Добавить группы**
2. В диалоговом окне выбрать одну или несколько групп
3. Нажать на кнопку **Добавить**
4. Сохранить изменения

Для исключения пользователя из группы:

1. В контекстном меню строки выбрать пункт **Удалить**
2. Сохранить изменения

## Права доступа

На вкладке пользователю присваиваются наборы прав для различных системных ролей.

## Роли

На вкладке перечисляются дополнительные функциональные роли и компетенции, назначенные пользователю.

Пользователю можно добавить несколько ролей, но одну роль необходимо выбрать в качестве роли по умолчанию на вкладке **Профиль**. Роль, выбранная по умолчанию, автоматически назначается пользователю при добавлении в команду проекта.

Пример: пользователю назначена роль «Руководитель проекта» на вкладке **Профиль** и две роли на вкладке **Роли** — «Архитектор» и «Консультант». Тогда по умолчанию при добавлении в команду будет назначена роль «Руководитель проекта», но её можно изменить на другие.

## Расписания

На вкладке задаётся расписание пользователя.

Расписание — паттерн для календаря рабочего времени. На основе сведений о расписании рассчитывается продолжительность каждого рабочего дня в часах.

По умолчанию добавлено расписание «40 часов в неделю»: 8 часов в день с понедельника по пятницу и 0 часов в субботу и воскресенье.

Начальное расписание задаётся в поле **Расписание**. Если расписание изменяется, новое расписание следует добавить в табличной части:

1. Нажать на кнопку **Добавить строку**
2. В строке указать:
   - **Начало действия** — дата начала действия нового расписания
   - **Расписание** — расписание, действующее с указанной даты

Не рекомендуется изменять расписание пользователя задним числом, так как это может привести к некорректному расчёту показателей.

Если рабочий график пользователя «плавающий» и не соответствует какому-либо паттерну, можно установить индивидуальное расписание. Для индивидуального расписания продолжительность каждого рабочего дня устанавливается вручную.

## Себестоимость

На вкладке задаётся ставка себестоимости пользователя за час работы и отражаются изменения ставки во времени.

Если пользователь является сотрудником внешней организации, значение поля **Себестоимость** заполняется как стоимость часа работы специалиста для вашей компании.

В поле **Себестоимость** указывается начальное значение ставки. Себестоимость пользователя можно указывать в любой валюте, внесённой в справочник **Валюты**.

Табличная часть предназначена для отражения изменений ставки себестоимости:

1. Нажать на кнопку **Добавить строку**
2. В строке указать:
   - **Начало действия** — дата начала действия новой ставки
   - **Себестоимость** — новое значение ставки

При изменении ставки себестоимости задним числом себестоимость часов из согласованных таймшитов не пересчитывается. Для пересчёта потребуется вернуть таймшиты в стадию «Черновик» и повторно перевести в стадию «Согласовано».

В поле **Правило нормализации** задаётся правило нормализации себестоимости для таймшитов пользователя.

## Таймшиты

На вкладке устанавливаются параметры таймшита пользователя.

В поле **Шаблон** выбирается шаблон, используемый для формирования таймшитов. Выбор из справочника **Шаблоны таймшитов**.

Опция **Ограничить виды работ** позволяет определить перечень видов работ, доступных пользователю при заполнении таймшита. Если опция не активна, пользователю доступны все записи справочника **Виды работ**.

## Настройки

На вкладке устанавливаются базовые системные настройки для пользователя.

Настраиваемые параметры:

- **Часовой пояс** — часовой пояс пользователя для отображения времени в интерфейсе
- **Язык интерфейса** — русский или английский язык интерфейса
- **Региональные настройки** — отображение календарных дат и финансовых показателей
- **Способ ввода времени** — формат ввода данных в таймшиты:
  - В часах (стандартный способ, число с десятичной частью)
  - В часах и минутах (формат чч:мм)
  - В процентах от расписания
- **Тема оформления** — светлое или тёмное оформление интерфейса
- **Использовать счётчики времени** — включение счётчиков времени при работе с таймшитами

При создании нового пользователя все параметры заполняются значениями по умолчанию, заданными на вкладке **Система**. Пользователь может изменять эти параметры самостоятельно в настройках личного профиля.

## Уведомления

На вкладке настраиваются личные уведомления пользователя о различных событиях в Timetta.

Параметры уведомлений пользователь может изменять самостоятельно в настройках личного профиля.

## Заместители

На вкладке настраивается список заместителей пользователя.

Заместитель получает права на вход в Timetta от имени замещаемого с аналогичными правами доступа к приложениям, данным и действиям.

Параметры замещения:

- **Заместитель** — пользователь, который будет замещать текущего
- **Начало** и **Окончание** — период замещения (если значение не выбрано, период неограничен)

Пользователь может изменять эти параметры самостоятельно в настройках личного профиля.

## Массовые операции с пользователями

В списке пользователей системы при выборе одного или более пользователей на панели действий появляются кнопки **Удалить** и **Изменить**.

Кнопка **Удалить** позволяет удалить одного или несколько пользователей из списка.

Кнопка **Изменить** позволяет изменить выбранный параметр для всех пользователей в списке:

- Правило нормализации себестоимости
- Руководитель
- Подразделение
- Добавить новое расписание
- Провайдер аутентификации
- Ресурсный пул
- Юридическое лицо
- Грейд
- Локация
