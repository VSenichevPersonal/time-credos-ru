## Назначение

Вкладка **Команда** предназначена для управления ресурсами, которые выполняют работу в рамках проекта.

### Права доступа

Для управления командой необходимо:

- наличие прав в роли **Управление проектами** с активным гранулом «Команда» (минимум — просмотр)

Для отображения ставок себестоимости:

- необходимо активировать гранул «Команда: Себестоимость» в правах роли **Управление проектами** (просмотр)

---

## Использование

Вкладка используется при работе с разными элементами системы:

1. **Структура работ** — исполнители из команды проекта могут быть назначены на работы
2. **Ресурсный план** — часы в ресурсном плане распределяются по участникам команды и работам
3. **Бронирование** — планирование и оформление запросов на бронирование ведётся в рамках состава команды проекта

---

## Обзор

Состав команды можно задать в карточке проекта на вкладке **Команда**.

Настроить представление таблицы ресурсов можно с помощью иконки шестерёнки в правом верхнем углу вкладки.

Ресурсы, включённые в команду проекта:

- получают доступ к базовым сведениям о проекте
- могут быть назначены исполнителями работ
- могут списывать время на проект (если назначены исполнителями работ)

На участников команды можно:

- планировать часы в ресурсном плане
- бронировать время в календаре бронирования

## Типы ресурсов

В команду проекта можно добавить:

- пользователей
- подразделения
- универсальные ресурсы

### Подразделения

Если в команду проекта добавлено **Подразделение**, все его участники смогут списывать время на работы проекта. Подразделение должно быть указано исполнителем этих работ либо явно, либо через назначение всей команды.

> **Примечание.** Подразделения рекомендуется добавлять только в команды внутренних проектов, где нужен массовый доступ сотрудников.

> **Примечание.** На подразделения нельзя планировать трудозатраты в ресурсном плане проекта.

### Универсальные ресурсы

**Универсальные ресурсы** — это безымянные ресурсы с набором свойств (роль, уровень, грейд, ресурсный пул и др.).

Такие ресурсы заменяют пользователей на старте планирования. После определения точного списка исполнителей могут быть заменены на конкретного пользователя или пользователей.

## Роль, Компетенция, Мастер-тариф, Индикаторы

Для каждого ресурса, вне зависимости от его типа, могут быть указаны:

- единственная **Роль** в проекте
- **Компетенция** (если есть)

### Мастер-тариф

Также можно задать **Мастер-тариф**, который используется:

- при формировании счетов по T&M-проектам — подробнее: [Биллинг T&M работ](/ru/docs/billing/billing-concept#billing-tandm-rabot)
- при расчёте оценки выручки через мастер выручки — подробнее: [Мастер выручки](/ru/docs/projects/instructions/revenue-master#metod-raschyota-po-tarifnym-stavkam)

### Индикаторы

Представление **Индикаторы** отображает случаи, когда ставка себестоимости превышает установленный мастер-тариф.

Используется для анализа отклонений и контроля за экономической эффективностью проектов.
