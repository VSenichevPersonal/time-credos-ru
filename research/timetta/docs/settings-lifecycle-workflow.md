# Воркфлоу

## Назначение

Эта компонента предназначена для настройки **воркфлоу** — процессов согласования сущностей в Timetta. Для доступа требуется роль администратора с правами на гранулу «Жизненные циклы».

## Общие сведения

Воркфлоу представляет собой бизнес-процесс для согласования различных сущностей (таймшитов, заявок на затраты, актов и прочего). Для каждой сущности можно создать несколько воркфлоу с разными условиями согласования.

В рамках воркфлоу система:
- Создает задачи для участников процесса
- Выдает права доступа для исполнителей на просмотр согласуемой сущности
- Изменяет стадию жизненного цикла сущности

## Создание воркфлоу

Процесс создания включает:
1. Переход к компоненте **Воркфлоу**
2. Нажатие кнопки **Создать воркфлоу**
3. Заполнение базовых реквизитов:
   - Наименование воркфлоу
   - Выбор жизненного цикла сущности
   - Указание стартового состояния
4. Нажатие кнопки **Создать**

## Базовые настройки воркфлоу

Основные параметры включают:
- **Наименование** — название воркфлоу
- **Жизненный цикл** — цикл согласуемой сущности
- **Инициатор может отменить процесс** — опция прерывания воркфлоу инициатором
- **Сущность активна** — признак активности
- **Стартовое состояние** — стадия для запуска воркфлоу
- **Описание** — свободная заметка

## Инициаторы воркфлоу

Определяет пользователей с правом запуска воркфлоу. В качестве инициаторов указываются:
- Роли (Все, Автор, Руководитель автора)
- Группы пользователей
- Конкретные пользователи
- Наборы прав

## Действия воркфлоу

Раздел определяет наименования действий запуска и отмены воркфлоу. Допускается добавление локализации для русского и английского интерфейсов.

## Схема воркфлоу

Схема состоит из функций и переходов между ними. Всегда должны присутствовать начальная (**Старт**) и конечная функции (**Финиш**). С помощью кнопки **Новая функция** добавляются функции типа «Задача», «Условие» и «Уведомление».

## Настройка функции «Старт»

Атрибуты:
- **Наименование** — обычно «Старт»
- **Следующая функция** — функция после старта
- **Переместить сущность в состояние** — стадия жизненного цикла после старта

## Настройка функции «Финиш»

Содержит только атрибут **Наименование** (обычно «Финиш»).

## Настройка функции типа «Задача»

### Базовые свойства

- **Наименование** — название функции (без символов `:` и `;`)
- **Локализация наименования** — отображение на разных языках

Допускается использование регулярного выражения `{{Name}}` для подставления названия согласуемой сущности.

### Исполнители функции

Могут указываться:
- Пользователи
- Группы пользователей
- Роли (Автор, Руководитель автора, Менеджер проекта, Менеджер клиента и др.)

Функция создает задачи для каждого исполнителя отдельно.

### Действия в функции

Определяют кнопки, доступные исполнителю задачи. Для каждого действия можно задать:
- Наименование и локализацию
- Иконку (формат `bi-{icon_name}`)
- Параметры формы перехода (запрос комментария и обязательность)

### Переходы в функции

Определяют движение между функциями и условия переходов:
- **Все** — переход, если все исполнители выполнят задачу с ожидаемым результатом
- **Первый** — переход при выполнении хотя бы одним исполнителем
- **Иначе после всех** — переход при невозможности определить исполнителей

Каждая функция должна содержать переход типа «Иначе после всех».

## Не создавать задачи

Раздел позволяет исключить создание задач при выполнении определенных условий:
- **Исполнитель задачи это инициатор воркфлоу** — исключает создание задачи для инициатора
- **Условия** — бизнес-правила для пропуска задач отдельным исполнителям

## Рекомендации по настройке функций

- Избегайте назначения конкретных пользователей исполнителями; используйте роли или группы
- Минимизируйте число функций в воркфлоу
- Каждая функция должна содержать переход типа «Иначе после всех»
- Сначала создавайте функции и переходы, затем расставляйте актуальные связи
