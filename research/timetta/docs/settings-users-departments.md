# Подразделения

Обновлено: 20.08.2025

## Назначение

В справочнике содержится список подразделений компании. Его записи используются при заполнении поля Подразделение на вкладке Профиль в карточке пользователя.

Для каждого пользователя можно указать подразделение компании, к которому он относится, а затем использовать эти сведения при формировании отчётов.

> **Примечание**
>
> Настройка подразделений доступна в компоненте Подразделения, если пользователю назначен набор прав для роли Администратор с активной гранулой «Подразделения».

## Создание подразделения

Чтобы создать подразделение, нужно:

- Перейти к компоненте Подразделения.
- На панели действий нажать на кнопку Создать подразделение.
- Указать наименование нового подразделения и нажать на кнопку Создать.

## Настройка подразделения

Карточка подразделения:

- Наименование — название подразделения
- Код — код подразделения, его краткий идентификатор
- Входит в — подразделение, ведущее по отношению к текущему
- Роль — роль, ассоциированная с подразделением
- Сущность активна — признак активности подразделения

С помощью поля Входит в можно задавать структуру подчинённости подразделений. В нём доступны для выбора только те подразделения, которые не создадут циклической ссылки. Подразделения, которые входят в текущее, для выбора недоступны.

Если в команду проекта добавлено Подразделение, то его пользователи в поле Роль таймшита смогут выбирать роли, ассоциированные с этим подразделением.

## Пример структуры

- Департамент маркетинговой аналитики входит в подразделение Стерлинг Купер Консалтинг.
- Отдел технического анализа входит в Департамент маркетинговой аналитики.
- Подразделение Департамент маркетинговой аналитики не может входить в подразделение Отдел технического анализа (циклическая ссылка).
