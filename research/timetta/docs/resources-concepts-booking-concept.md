# Концепция бронирования

## Зачем нужен календарь бронирования

Timetta предоставляет два инструмента для планирования использования сотрудников: ресурсный план проекта и календарь бронирования.

**Ресурсный план** служит менеджерам проектов для оценки трудозатрат и себестоимости. Он может быть детализирован до уровня отдельных задач и поддерживает использование универсальных ресурсов (ролей), что полезно на этапах первоначальной оценки.

**Календарь бронирования** предназначен для коллективной работы или ресурсных менеджеров и решает следующие задачи:

- Долгосрочное распределение сотрудников по проектам
- Контроль ожидаемой загрузки каждого сотрудника
- Выявление конфликтов за ресурсы и овербукинга
- Балансировка загрузки для предотвращения простоев и перегрузок
- Оптимальное распределение ресурсов с учётом навыков

## Подходы к бронированию

### Децентрализованный подход

При этом подходе все менеджеры проектов совместно работают с календарём бронирования как с общей доской. Требуется предоставить им права редактирования календаря. Решения о перераспределении ресурсов принимаются коллективно на регулярных встречах.

### Централизованный подход

Этот подход предполагает взаимодействие менеджеров проектов и ресурсных менеджеров через запросы ресурсов. Менеджеры проектов не могут самостоятельно резервировать сотрудников; они должны обращаться к ресурсным менеджерам, которые принимают решения о бронировании.

## Базовый и детальный режимы бронирования

### Базовый режим

Использует "плашки" с параметрами периода, проекта и процента загрузки. Плашки могут быть в статусе Hard (подтверждённое) или Soft (предварительное). Лучше подходит для децентрализованного управления благодаря простоте работы с визуальными элементами.

### Детальный режим

Представлен таблицей с явным указанием часов для каждого таймслота. Обеспечивает более точное распределение, но сложнее воспринимается. Идеален для централизованного управления и работы с запросами ресурсов.
