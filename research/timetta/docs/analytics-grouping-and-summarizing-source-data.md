# Группировка и суммирование данных источника

Обновлено: 29.08.2025

## Основные концепции

В источниках данных существуют два типа полей:

- **Аналитики** (например, проект, пользователь)
- **Значения** (например, часы, сумма)

Любой запрос данных приводит к группировке по полям-аналитикам с суммированием по полям-значениям.

## Пример: источник «Фактические данные»

При построении отчёта из источника «Фактические данные» информация о списании времени группируется по выбранным аналитикам.

Если выбрать поля "Проект (Наименование)" и "Часы Всего", итоговый набор данных будет сгруппирован по наименованию проекта с суммированием часов.

При добавлении поля "Пользователь (Имя)" отчёт будет детализирован по пользователям в рамках каждого проекта.

### Рекомендация по оптимизации

"Чтобы отчёт был компактным, рекомендуется выводить только нужные аналитики или ограничивать набор данных фильтрами."

При создании детальных журналов с полями "Дата" и "Комментарий" следует использовать фильтры для ограничения временного интервала.

## Поля-значения и агрегация

### Различия между полями источника и представления

- Поля-значения источника фиксированы
- Поля-значения представления можно выбрать при настройке отчёта

Если добавить нечисловое поле в поля-значения представления, система применит агрегацию по количеству. Например, можно получить отчёт о числе уникальных пользователей, списывавших время на проекты.
