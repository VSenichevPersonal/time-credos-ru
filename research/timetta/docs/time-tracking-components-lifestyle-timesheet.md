## Жизненный цикл таймшита

Жизненный цикл по умолчанию:

![Схема ЖЦ по умолчанию](https://timetta.com/entity-assets/7a2f9075-9968-4a3c-a803-7d2433a54726/lc-ts.png)

Жизненный цикл таймшита можно настраивать с помощью компоненты **[Жизненные циклы](https://app.timetta.com/settings/lifecycles)**. Подробнее — [Жизненные циклы](https://timetta.com/ru/docs/settings/config/life-cycle).

### Права на переходы

Права доступа позволяют указать исполнителей для каждого перехода. В качестве исполнителей можно указывать:

- Роль
- Группу пользователей
- Конкретных пользователей
- Наборы прав

Для жизненного цикла таймшита поддерживаются следующие роли:

| Роль | Описание |
|------|----------|
| Все | Любой пользователь системы |
| Автор | Пользователь — автор документа |
| Менеджер проекта | Пользователь или пользователи, указанные в свойствах проекта как *Менеджер проекта*, добавленных в строки таймшита |
| Руководитель автора | Пользователь, указанный в качестве непосредственного руководителя в карточке пользователя автора |
| Менеджер клиента | Пользователь или пользователи, указанные в качестве *Менеджеров клиентов*, в карточке клиента, добавленными в строки таймшита. |
| Менеджер программы | Пользователь, который указан менеджером в свойствах выбранной программы |
| Со-менеджеры программы | Пользователи, которые указаны в свойствах, как *Со-менеджеры* программ |
| Со-менеджеры проекта | Пользователи, которые указаны в свойствах, как *Со-менеджеры* проекта |

### Системные состояния

Системные состояния запускают внутри системы другие процессы:

| Состояние | Описание |
|-----------|----------|
| Черновик | Начальное состояние таймшита, допускающее его редактирование при переходе в другое состояние — происходит валидация заполненности полей |
| Согласовано | При переходе таймшита в состояние «Согласовано»: <ul><li>Таймшит блокируется от изменений</li><li>Для часов, учтенных в таймшите, рассчитывается себестоимость</li><li>Создаются проводки для начисления затрат по статье «Себестоимость труда» в разрезе проектов и задач и по статье «Себестоимость отсутствий» для оплачиваемых типов отсутствий</li><li>Создаются записи с типом «Начисление» в компоненте **Баланс отсутствий** (для отсутствий с заданной политикой начисления типа, типом начисления «Начисление переработок»)</li></ul> |
