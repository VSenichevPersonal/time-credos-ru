# Системные настройки

Обновлено: 12.08.2025

## Назначение

Компонента **Настройки системы** предназначена для настройки базовых параметров абонента Timetta.

Примечание: Для доступа требуется роль **Администратор** с активными правами для гранулы «Управление абонентом».

## Главное

На вкладке **Главное** находится основная информация:

- **Отображаемое наименование** — название абонента, обычно краткое наименование организации
- **Владелец системы** — имя, фамилия и контакты ответственного сотрудника. На указанную почту приходят уведомления об обновлениях, обслуживании и истечении лицензий
- **Разрешить вход поддержке** — опция для предоставления доступа сотрудникам службы поддержки Timetta

## Платёжная информация

Используется при генерации счетов и актов при покупке лицензий:

- **Организация** — реквизиты организации-плательщика
- **Регион** — Страна и Валюта расчётов (RUB)
- **Почтовый адрес** — для отправки актов и бухгалтерских документов
- **Контактное лицо** — данные ответственного сотрудника по финансам
- **Использование ЭДО** — опция электронного документооборота

## Система

Раздел **Значения по умолчанию** задаёт параметры для всех пользователей:

- **Часовой пояс** — влияет на отображение времени операций в интерфейсе
- **Язык интерфейса** — русский или английский
- **Региональные настройки** — формат отображения дат и финансовых показателей

Индивидуальные параметры может изменить сам пользователь в личном профиле или администратор в карточке пользователя.

## Приложения

### Финансы

- **Базовая валюта** — определяется автоматически при развёртывании, недоступна для изменения
- **Включить расширенный Финансовый отчёт** — добавляет показатели корпоративного налога, оборотного капитала и непогашенных затрат
- **Поддержка НДС** — включает функцию НДС в счетах

### Затраты на капитал

Устанавливается первоначальная **Ставка затрат на капитал** в процентах. Возможно назначение новой ставки с указанием даты начала действия. Предусмотрены варианты начисления:

- На оборотный капитал
- На непогашенные затраты
- На оба показателя одновременно

### Бронирование

Доступны два режима:

- **Базовый** — бронирование ресурсов в часах в виде плашек (по умолчанию)
- **Детальный** — ввод часов в ячейки календаря

**Фиксированный масштаб бронирования** позволяет зафиксировать масштаб (дни, недели, месяцы) по всей системе.

### Прогнозирование

**Период прогнозирования** влияет на:

- Алгоритм расчёта прогнозных показателей проектов
- Отображение представления Прогноз в ресурсных планах

### ИИ ассистент

Выбирается, какие данные абонента могут использоваться:

- **Передавать данные проектов** — данные проектов и портфелей
- **Передавать данные организаций** — организации системы
- **Передавать вики-пространства** — вики-пространства и страницы
- **Передавать данные пользователей** — активные пользователи системы

**Обновить данные в ИИ ассистенте** — принудительно обновить копию данных, запрос обрабатывается асинхронно.

### Иерархия задач

Настраивается упорядоченный список типов задач для построения иерархии. Доступные действия:

- **Добавить** — новая строка
- Стрелки **вверх/вниз** — изменение порядка
- **Удалить** — удаление строки

### Интеграция с GitLab

Параметры подключения к GitLab:

- **Включить интеграцию с GitLab** — включает/отключает интеграцию
- **Базовый URL GitLab** — корневой адрес сервера
- **Новый GitLab токен** — Personal access token для вызовов API
- **Новый секрет** — секрет для входящих вебхуков (кнопка Сгенерировать)

## Провайдеры аутентификации

На вкладке **Провайдеры аутентификации** определяются способы входа в Timetta. Можно задать:

- Период обязательного обновления пароля
- Требования к сложности пароля
- Ограничения по количеству попыток
- Длительность блокировки
- Использование многофакторной аутентификации (MFA)

## Домены

Используются при формировании адресов электронной почты пользователей и определяют принадлежность к абоненту.

Только один домен может быть основным (указанный при регистрации). Дополнительные домены требуют подтверждения перед использованием.

## API-токены

Вкладка для генерации API-токенов. При создании выбирается пользователь — токен создаётся с аналогичными наборами прав. Любой токен можно отозвать. Следует записать токен при создании, так как позже его больше нельзя отобразить.

## Удаление данных

После регистрации абонент заполняется демонстрационными данными. Перед использованием системы в рабочем режиме эти данные следует удалить.

**Внимание:** Удаление данных — необратимая операция.
