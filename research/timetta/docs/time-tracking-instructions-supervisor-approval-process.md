# Согласование таймшитов и заявок на отсутствие

## Общие сведения

Система предусматривает базовый процесс согласования для таймшитов и заявок на отсутствие. Этот процесс можно кастомизировать через настройки Воркфлоу, позволяя установить индивидуальные маршруты согласования и определить список согласующих лиц (как конкретных пользователей, так и роли).

Процесс согласования завершается успешно только при одобрении всеми согласующими. Если кто-либо отклоняет документ, процесс прерывается и сущность возвращается на доработку автору.

## Особенности прав доступа

Менеджеры проектов видят по умолчанию только строки своих проектов в таймшитах. Для доступа к другим проектам требуется гранула "Строки таймшитов" в наборе прав для роли управления проектами.

Остальные пользователи получают доступ к таймшитам через гранулу "Таймшиты" в наборе прав для роли управления командой.

## Согласование таймшитов и заявок на отсутствие

Сущности, требующие согласования, появляются в разделе "Входящие" области "Моя работа". Согласующие получают уведомления по электронной почте и внутри системы.

Из карточки сущности доступны команды:
- **Согласовать** — одобрить документ
- **Отклонить** — отклонить с указанием причины в поле комментария

## Получение сведений о процессе согласования

В карточке сущности отображаются:
- Текущее состояние
- Информация об авторе
- Сведения о воркфлоу

История согласования доступна на вкладке "Активность". Список входящих показывает статус каждой сущности, представление можно настроить через кнопку изменения представления.

## Прерывание процесса согласования

Автор может остановить согласование при наличии соответствующих настроек воркфлоу. Согласующий может прекратить процесс при наличии прав на принудительное изменение состояний.

## Дополнительные возможности

- **Повторное согласование**: пользователи пропускают этап, если уже согласовали и доступные строки не изменились
- **Пропуск автором**: автор таймшита пропускает этап согласования, если он участник процесса
- **Администрирование**: менеджер по кадрам может принудительно согласовать сущность или вернуть её в состояние черновика, минуя стандартный процесс
