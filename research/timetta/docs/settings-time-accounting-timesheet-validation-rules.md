# Правила валидации таймшитов

## Назначение

Правила валидации предназначены для проверки корректности заполнения таймшитов перед отправкой на согласование. Функция требует прав администратора с активированной гранулой "Правила валидации".

## Использование

Правила валидации применяются в двух компонентах:

- **Шаблоны таймшитов** — в карточке шаблона указываются применяемые правила
- **Таймшиты** — проверка выполняется при попытке отправить таймшит на согласование

## Создание и активация нового правила

Процесс создания включает следующие шаги:

1. Перейти в компоненту "Правила валидации"
2. Нажать кнопку "Создать правило"
3. Указать наименование и тип правила
4. Внести условия проверки
5. Добавить правило в соответствующий шаблон таймшита

## Уровни правил

Предусмотрено два уровня:

- **Ошибка** — пользователь обязан устранить замечание перед отправкой
- **Предупреждение** — система уведомляет, но позволяет отправить таймшит

Администратор может установить пользовательский текст для каждого уровня.

## Типы правил

| Тип | Описание |
|-----|---------|
| Дата отправки на согласование | Устанавливает допустимые дни перед окончанием периода для отправки |
| Количество строк таймшита | Определяет минимальное число заполненных строк |
| Лимит рабочего дня в часах | Ограничивает часы в день (например, максимум 24) |
| Обязательность полей таймшита | Делает обязательными поля: вид работ, роль, центр затрат, коды оплаты, тариф, комментарий и пользовательские поля |
| Отклонение от расписания в % | Контролирует отклонение по дням или за весь период (например, 25%/125%) |
| Отклонение от расписания в часах | Аналогично предыдущему, но в часах (например, 2ч/2ч) |
| Превышение плановых трудозатрат | Ограничивает списание часов по отношению к плану работы или проекта |
| Пустые строки таймшита | Запрещает отправку таймшитов с незаполненными строками |
| Указан актуальный центр затрат | Проверяет соответствие юридического лица и статус активности центра затрат |
| Выбор тарифа | Пропускает проверку обязательности тарифа, если пользователю не назначен тариф в проекте |

## Настройка правила

Каждое правило содержит следующие атрибуты:

- **Тип правила** — принадлежность к категории валидации
- **Наименование** — название правила
- **Текст ошибки/предупреждение** — сообщение для пользователя
- **Описание** — текстовое пояснение
- **Уровень** — ошибка или предупреждение
- **Сущность активна** — признак активности (неактивные правила недоступны для выбора)
