# Типы отсутствий

## Назначение

Классификатор **Типы отсутствий** используется при формировании заявок на отсутствие. Тип отсутствия указывается при создании заявки.

Для настройки классификатора требуется роль администратора с активной гранулой «Типы отсутствий».

## Создание типа отсутствия

Чтобы создать тип отсутствия:

1. Перейти к компоненте **Типы отсутствий**
2. На панели действий нажать кнопку **Создать тип отсутствия**
3. Указать код, единицу учёта, наименование и нажать **Создать**

## Настройка типа отсутствия

Атрибуты типа отсутствия:

- **Наименование** — название типа отсутствия (обязательное)
- **Код** — код типа отсутствия, отображается в ячейках графика (рекомендуется 1–2 символа, обязательное)
- **Оплачиваемый** — признак оплачиваемого отсутствия; для них рассчитывается ставка себестоимости
- **Единица учёта** — определяет единицы начислений и списаний; доступные значения: «Рабочий день», «Календарный день», «Час» (обязательное, недоступно после создания)
- **Описание** — заметка о типе отсутствия
- **Активно** — признак активности типа; неактивный тип нельзя выбрать в заявке

## Политика начисления

На вкладке **Политики** можно задать политику начисления, которая позволяет начислять доступные дни/часы для каждого пользователя.

Поддерживаются два типа политик:

### Фиксированные начисления

Используются для регулярного начисления доступных часов, рабочих или календарных дней. Применяются для типов отсутствий с единицами учёта: час, календарный день, рабочий день.

**Атрибуты:**

- **Период начисления** — периодичность (неделя, месяц, год); начисление выполняется в первый день периода
- **Единиц к начислению** — количество дней для начисления в каждом календарном периоде для каждого пользователя

### Начисления переработок

Позволяют начислять часы переработок на основе согласованных таймшитов. Применяются только для типов отсутствий с единицей учёта «Час».

**Атрибуты:**

- **Только оплачиваемые проекты** — исключает из расчёта часы на неоплачиваемые проекты
- **Исключаемые типы отсутствий** — исключает из расчёта часы на выбранные типы отсутствий

#### Пример 1

Учитываются все проекты, исключаемые типы не заданы:

| День | ПН | ВТ | СР | ЧТ | ПТ | СБ | ВС |
|------|----|----|----|----|----|----|---|
| По расписанию | 8 | 8 | 8 | 8 | 8 | 0 | 0 |
| По проектам | 8 | 10 | 6 | 8 | 8 | 2 | 0 |
| Переработка | 0 | 2 | -2 | 0 | 0 | 2 | 0 |
| К начислению | 0 | 2 | 0 | 0 | 0 | 2 | 0 |

**Итого к начислению: 4 часа**

#### Пример 2

Учитываются только оплачиваемые проекты, исключаемые типы не заданы:

| День | ПН | ВТ | СР | ЧТ | ПТ | СБ | ВС |
|------|----|----|----|----|----|----|---|
| По расписанию | 8 | 8 | 8 | 8 | 8 | 0 | 0 |
| Оплачиваемые часы | 4 | 8 | 0 | 12 | 2 | 2 | 0 |
| Неоплачиваемые (исключаются) | 4 | 0 | 8 | 2 | 2 | 0 | 2 |
| Переработка | -4 | 0 | -8 | 4 | -6 | 2 | 0 |
| К начислению | 0 | 0 | 0 | 4 | 0 | 2 | 0 |

**Итого к начислению: 6 часов**

#### Пример 3

Учитываются все проекты, заданы исключаемые типы отсутствий:

| День | ПН | ВТ | СР | ЧТ | ПТ | СБ | ВС |
|------|----|----|----|----|----|----|---|
| По расписанию | 8 | 8 | 8 | 8 | 8 | 0 | 0 |
| По не исключаемым типам | 0 | 0 | 4 | 4 | 6 | 0 | 0 |
| По исключаемым типам | 8 | 4 | 0 | 0 | 0 | 0 | 0 |
| По проектам | 2 | 0 | 12 | 6 | 3 | 2 | 0 |
| Переработка | -6 | -8 | 8 | 2 | 1 | 2 | 0 |
| К начислению | 0 | 0 | 8 | 2 | 1 | 2 | 0 |

**Итого к начислению: 13 часов**

#### Пример 4

Учитываются только оплачиваемые проекты, заданы исключаемые типы отсутствий:

| День | ПН | ВТ | СР | ЧТ | ПТ | СБ | ВС |
|------|----|----|----|----|----|----|---|
| По расписанию | 8 | 8 | 8 | 8 | 8 | 0 | 0 |
| Оплачиваемые часы | 8 | 8 | 0 | 6 | 12 | 2 | 0 |
| Неоплачиваемые (исключаются) | 0 | 2 | 12 | 2 | 0 | 0 | 2 |
| По не исключаемым типам | 8 | 0 | 0 | 4 | 0 | 0 | 0 |
| По исключаемым типам | 0 | 0 | 8 | 0 | 2 | 0 | 0 |
| Переработка | 8 | 0 | -8 | 2 | 4 | 2 | 0 |
| К начислению | 8 | 0 | 0 | 2 | 4 | 2 | 0 |

**Итого к начислению: 16 часов**

**Важно:** дни с отрицательной переработкой не учитываются при расчёте начислений.

## Политика списания

На вкладке **Политики** можно задать политику списания, определяющую правила создания записей типа «Списание» в справочнике **Баланс отсутствий**.

Настройка доступна только для типов отсутствий с единицей учёта «Календарный день».

**Атрибуты:**

- **Единица учёта** — единица измерения длительности отсутствия
- **Учитывать исключения из календарной длительности** — исключает государственные праздники и нерабочие дни из расчёта длительности отсутствия
