# Журнал изменений

**Обновлено: 14.08.2025**

## Назначение

Компонент **Журнал изменений** доступен по адресу `https://app.timetta.com/settings/history-log` и позволяет просматривать системный журнал — список всех изменений, которые пользователи внесли в данные системы.

Доступ требует роли Администратора с активным правом **«Журнал изменений»**.

## Обзор

Интерфейс отображает записи об изменениях со следующими столбцами:

- **Пользователь** — сотрудник, выполнивший действие
- **Создано** — дата и время изменения
- **Тип** — тип операции: `Add` (создание), `Modify` (изменение), `Remove` (удаление)
- **Тип сущности** — код системного объекта
- **Наименование сущности** — название изменённого объекта
- **Id сущности** — уникальный системный идентификатор

## Поиск в журнале изменений

Поиск выполняется по наименованию сущности. Результаты включают не только саму сущность, но и все связанные с ней объекты (атрибуты).

Пример: поиск по таймшиту (`TimeSheet`) также отображает изменения строк таймшита (`TimeSheetLine`) и ячеек (`TimeAllocation`).

Поддерживается фильтрация по нескольким свойствам одновременно.

## Просмотр деталей изменения

Карточка детальной записи содержит:

- Тип операции, код и наименование сущности
- Дату и время изменения, пользователя, выполнившего действие

**Раздел «Изменения»** — таблица, показывающая какое свойство было изменено, со старым и новым значением.

**Раздел «Атрибуты»** — связанные сущности, отображаемые для отслеживания связей при поиске.

## Логируемые сущности

В журнале отображаются только те сущности, которые зарегистрированы в **Менеджере сущностей** с включённым параметром **«Используется в логировании»**.
