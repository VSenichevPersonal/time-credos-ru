# Учет затрат на субподрядчиков

## Введение

В системе предусмотрен отдельный контур работы с субподрядчиками. Он позволяет планировать, учитывать и анализировать затраты на привлечение внешних исполнителей в проектах, а также контролировать расчёты с ними. Ниже описана общая логика работы этого контура.

## Что такое субподрядчик

**Субподрядчик** — внешняя организация или поставщик, который участвует в выполнении проекта и получает оплату за выполненные работы, оказанные услуги или поставленные материальные ценности.

Роль субподрядчика в проекте заключается в выполнении части работ или поставке ресурсов, необходимых для реализации проекта.

В системе субподрядчики представлены через организации и используются как аналитика для:

- планирования затрат,
- учёта фактических расходов,
- контроля актирования,
- управления расчётами.

> **Примечание:** В качестве субподрядчика может выступать любая компания из справочника **Организации**.

## Типы участия субподрядчиков

Субподрядчики могут участвовать в проекте в разных формах:

### Аутсорсинг / аутстаффинг

В этом случае специалисты субподрядчика работают как проектные ресурсы и отражают свою работу через таймшиты. Планирование затрат выполняется с помощью **Ресурсного плана** проекта, а учет фактических затрат — через **Таймшиты**.

> **Примечание:** Сотрудники субподрядчика, вовлеченные в проекты, добавляются в справочник **Пользователи**. В карточке пользователя предусмотрено поле **Субподрядчик** с выбором значения из справочника **Организации**. Для ведения таймшитов таким пользователям потребуется лицензия.

### Поставщики услуг или материальных ценностей

Субподрядчик также может выступать поставщиком:

- услуг по фиксированной цене,
- оборудования,
- материалов,
- других ресурсов, необходимых для выполнения проекта.

В этом случае сотрудники субподрядчика не ведут таймшиты. Планирование затрат выполняется на вкладке **Затраты** карточки проекта, а учет фактических затрат — через **Заявки на затраты**.

## Оценка и планирование

Планировать можно как **начисление** затрат по субподрядчикам (для отражения в Фин. отчете проекта), так и **выплаты** (для формирования платежного календаря и контроля платежей по проекту).

### Оценка и планирование затрат (P&L)

Затраты на субподрядчиков могут планироваться несколькими способами.

#### Для аутсорсинга и аутстаффинга

Оценка формируется через **Ресурсный план** проекта. Сотрудники субподрядчика добавляются в команду проекта, далее оценка и планирование выполняется так же, как для штатных сотрудников.

На основе ресурсного плана для сотрудников-субподрядчиков рассчитывается оценка затрат по учетной статье **Себестоимость труда субподрядчиков**. При этом используется ставка себестоимости ресурса. Для субподрядных ресурсов эта ставка фактически отражает стоимость одного человеко-часа работы субподрядчика для нашей организации.

#### Для поставщиков услуг или материальных ценностей

Для оценки затрат используется вкладка **Затраты** в карточке проекта. Оценка вводится вручную.

> **Примечание:** В карточке **Оценки затрат** предусмотрено поле **Субподрядчик** с выбором значения из справочника **Организации**.

### Оценка и планирование платежей

Помимо оценки затрат для P&L, в системе можно планировать будущие выплаты субподрядчикам.

Важно различать две даты:

- **Дата счёта** — дата, когда субподрядчик выставляет счёт.
- **Дата оплаты** — дата, когда фактически планируется проведение платежа.

Эти даты могут отличаться, например из-за условий договора или внутреннего процесса согласования.

> **Примечание:** Оценка и планирование оплат выполняется на вкладке **Затраты** в карточке проекта. Предусмотрено два календаря:

- **Счета субподрядчиков** — для формирования план-графика получения счетов от субподрядчиков.
- **Оплаты субподрядчикам** — для формирования план-графика оплат субподрядчикам.

При планировании оплат можно указать **учетную статью**. Это поле не является обязательным, однако может быть полезно:

- для детализации финансовой аналитики,
- при интеграциях с внешними системами,
- для формирования отчётности.

Также можно указать **ставку НДС**. Это поле опционально, но может быть полезно в двух случаях:

- при исходящих интеграциях, когда сторонние системы должны знать, по какой ставке НДС будет производиться оплата;
- в отчетах по платежному календарю, где можно будет анализировать суммы с НДС и без НДС.

> **Важно:** Отчет «Платежный календарь» — в разработке.

### Фиксация плана

План начислений затрат и платежей по субподрядчикам фиксируется стандартным способом — путем создания, утверждения и назначения **мастер-версии** проекта.

## Учет факта

### Начисление фактических затрат (P&L)

В зависимости от типа подрядчика для начисления фактических затрат используются различные механизмы, но в конечном счете P&L проекта формируется на основе финансовых проводок.

> **Примечание:** В карточку финансовой проводки предусмотрено поле **Субподрядчик**.

#### Для аутсорсинга и аутстаффинга

Учет ведется на основе **таймшитов**.

Механика выглядит следующим образом:

- Сотрудники-субподрядчики заполняют таймшиты.
- Таймшиты проходят процесс согласования.
- Когда таймшит согласован, на его основе автоматически создаются финансовые проводки по учетной статье **Себестоимость труда субподрядчиков**.
- Сведения из финансовых проводок отражаются в P&L проекта.

#### Для поставщиков услуг или материальных ценностей

Учет ведется через **Заявки на затраты**.

Механика выглядит следующим образом:

- Создается заявка на затраты.
- Заявка на затраты проходит процесс согласования.
- Когда заявка согласована, на её основе автоматически создаются финансовые проводки.
- Сведения из финансовых проводок отражаются в P&L проекта.

> **Примечание:** В карточке **Заявки на затраты** предусмотрено поле **Субподрядчик** с выбором значения из справочника **Организации**.

### Фиксация факта начисления через входящий акт

> **Важно:** Входящие акты — в разработке.

Для подтверждения и сверки затрат используется документ **Входящий акт** (Incoming Act of Acceptance).

**Входящий акт** — это документ приёмки работ и сверки затрат на субподрядчика.

Он выполняет reconciliation — сверку двух источников данных, чтобы убедиться, что суммы и записи совпадают, и выявить возможные расхождения.

#### Назначение

Входящий акт выполняет несколько задач:

- фиксирует факт принятия работ субподрядчика,
- сверяет сумму акта с уже признанными затратами,
- при необходимости формирует корректировки.

#### Логика работы

- Акт выставляется субподрядчиком и регистрируется в системе.
- В акте указывается период актирования работ.
- Система сверяет сумму по строке с учетной статьей «Себестоимость труда субподрядчиков» с уже зафиксированными проводками за указанный период.
- Если обнаружено расхождение:
  - система сообщает об этом оператору,
  - создаётся корректировочная проводка.

Такой подход позволяет:

- максимально быстро отражать затраты в Фин. отчете с помощью таймшитов и заявок на затраты,
- при этом сохранять гибкость, необходимую для реальных условий работы.

#### Контроль актирования

Для контроля актирования используется аналитика «Актировано субподрядчиком». Она доступна в отчёте «Таймшиты детально» и позволяет видеть, какие трудозатраты уже подтверждены актом.

#### Сверка по другим затратам

Аналогичная сверка выполняется и по другим статьям затрат. В частности, система сопоставляет суммы акта с затратами, отражёнными через заявки на затраты.

## Расчёты с субподрядчиками

Для отслеживания расчетов с подрядчиками используются Входящие счета.

> **Важно:** Входящие счета — в разработке.

### Входящий счёт

Входящий счет — документ расчётов и обязательства к оплате, создаваемый на основании Входящего акта.

Он разделяет два разных контура учёта:

- учёт затрат (P&L, метод начисления),
- расчёты и платежи (кассовый метод).

### Регистрация счёта

Входящий счёт регистрируется в системе на основании счёта, выставленного субподрядчиком. В документе указывается ссылка на Входящий акт, а система контролирует соответствие сумм. По одному входящему акту может быть зарегистрировано несколько счетов.

Входящие счета, как и другие документы, привязаны к проекту. Весь учет в системе проектно-центричный, поэтому расчёты с субподрядчиками также анализируются в разрезе проектов.
