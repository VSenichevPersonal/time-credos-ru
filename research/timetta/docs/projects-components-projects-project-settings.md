# Свойства проекта

## Назначение

**Свойства** представляют собой основные параметры проекта. Часть этих параметров задается при создании проекта, а некоторые можно редактировать впоследствии. Доступ к редактированию осуществляется через вкладку **Свойства** в карточке проекта.

## Основные свойства проекта

Блок основных свойств включает:

- **Наименование** — краткое уникальное имя проекта
- **Менеджер проекта** — назначенный руководитель
- **Расписание** — график работы для расчета выходных дней
- **Начало и Окончание** — период реализации проекта
- **Разрешить списание времени без выбора работы** — опция для учета времени без детализации
- **Пропускать менеджера в воркфлоу для таймшитов** — пропуск этапа согласования
- **Код** — уникальный идентификатор проекта
- **Организация** — клиент, в интересах которого выполняется проект
- **Программа** — название программы, в которую входит проект
- **Внешняя ссылка** — URL для дополнительной информации
- **Модель этапов** — структура проекта с набором этапов

## Финансовые параметры

Раздел "Финансы проекта" содержит:

- **Тип учета** — тип оплаты проекта
- **Юридическое лицо проекта** — сопровождающая организация
- **Ставка корпоративного налога** — налоговые отчисления
- **Начислять затраты на капитал** — опция для включения капитальных затрат
- **Валюта** — валюта проекта (неизменяемая после создания)
- **Способ биллинга** — ручное или автоматическое создание счетов

## Дополнительная информация

Блок "Дополнительно" включает:

- **Описание** — краткое описание проекта
- **Дополнительные поля** — пользовательские параметры
- **Вложения** — прикрепленные файлы

## Со-менеджеры проекта

Роль со-менеджера позволяет расширить команду управления. Эти пользователи получают права на управление проектом в соответствии с их ролью "Управление проектами".
