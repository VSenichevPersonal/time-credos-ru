# Конфигурация дополнительных полей

## Назначение

Дополнительные поля позволяют расширить стандартные свойства системных сущностей. Для работы с ними требуется роль администратора с активированным правом на гранулу "Дополнительные поля сущностей".

Эти поля можно использовать для:
- Отображения в виде колонок в списках
- Фильтрации информации
- Включения в аналитические отчеты

## Создание дополнительного поля

Процесс создания включает следующие шаги:

1. Откройте раздел "Дополнительные поля" в системных параметрах
2. Нажмите на кнопку "Создать поле"
3. Укажите параметры:
   - **Наименование** — название поля
   - **Сущность** — целевой объект
   - **Тип** — тип данных (строка, число, дата, справочник, пользователь, логический, редактор текста и др.)
4. Завершите создание нажатием кнопки "Создать"

## Настройка дополнительного поля

### Раздел Главное

Содержит базовые параметры:
- Наименование поля
- Описание назначения
- Статус активности
- Тип сущности и тип поля (только для чтения)
- Имя свойства (только для чтения)

### Раздел Настройки

Определяет видимость поля в интерфейсе:
- Включение в фильтры списков
- Отображение в списках как колонка
- Показ в карточках сущностей
- Использование в отчетах
- Опция "только для API"

### Раздел Локализация

Позволяет задать отображаемые тексты для разных языков:
- Выбор языка
- Заголовок поля
- Текст-подсказка в пустом поле

### Раздел «Свойства»

Содержит дополнительные параметры:
- Значение по умолчанию
- Расположение на форме
- Порядковый номер отображения
- Флаг обязательности заполнения

## Важные замечания

Созданные поля появятся в сущности после перезагрузки страницы. Удаление элементов справочника, которые уже используются, невозможно. Неиспользуемые элементы можно деактивировать, чтобы скрыть их от выбора.
