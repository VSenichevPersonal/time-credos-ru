# Исключения расписаний

## Назначение

В справочнике формируются наборы исключений, которые можно добавлять в расписания.

**Примечание:** Для настройки исключений расписания требуется набор прав для роли **Администратор** с активной гранулой «Исключения расписаний».

## Создание набора исключений

1. Перейдите к компоненте Исключения расписаний.
2. Нажмите на кнопку **Создать исключение**.
3. Укажите наименование исключения и нажмите **Создать**.
4. Заполните карточку исключения, включая атрибуты исключений (доступны для заполнения после нажатия **Добавить строку**):
   - **Дата** — дата исключения
   - **Наименование** — наименование исключения
   - **Продолжительность** — продолжительность рабочего дня в часах на дату. Например, если это укороченный день — 7 часов, выходной — 0 часов
   - **Исключать из календарной длительности** — признак для обозначения нерабочих праздничных дней по ТК

Для применения набора исключений перейдите в Расписания, выберите нужное расписание и в карточке выберите пакет **Исключения**.

**Примечание:** В «Политиках списания» для «Типов отсутствий» с единицей учёта «Календарный день» можно указать, учитывать ли исключённые из календарной длительности дни при расчёте длительности отсутствий.

## Отличие от индивидуальных расписаний

Расписание создаётся с использованием шаблона отрезка времени (например, недели) и числа рабочих часов на каждый день. Все недели в таком расписании, в общем случае, идентичны. Допускается ввод исключений на конкретные даты: например, для указания национальных праздников.

Расписания обычно используются для установления единого графика работы сразу для многих пользователей: например, одно расписание для офиса, другое расписание для работающих на полставки.

Однако если у пользователя есть индивидуальный график на определённый период, необходимо использовать индивидуальные расписания пользователей.
