# Жизненные циклы и воркфлоу

## Жизненные циклы

**Жизненный цикл** представляет собой "набор состояний сущности и правил переходов между ними." **Состояние** характеризует текущий этап обработки и поведение сущности.

Например, счёт может проходить через состояния: Черновик, Выпущен, Оплачен, Отменён.

Система позволяет настраивать права на переходы между состояниями. К примеру, только Финансовый контролёр может перевести счёт из состояния Оплачен в Отменён. Состояния также ограничивают редактирование — счёт может быть отредактирован исключительно в состоянии Черновик.

## Воркфлоу

**Воркфлоу** — это инструмент управления процессами для координации последовательности действий при работе с сущностью, включая взаимодействие между сотрудниками.

Воркфлоу дополняет жизненный цикл пошаговой обработкой. В примере со счётом:

1. Финансовый менеджер создаёт счёт, переводя его в состояние Выпущен
2. Менеджер проекта получает задание на согласование с клиентом
3. При замечаниях задание возвращается финансовому менеджеру
4. При оплате счёт переходит в состояние Оплачен
5. При отказе счёт переходит в состояние Отменён

## Общие рекомендации по настройке

Жизненного цикла достаточно, когда не требуется организованное взаимодействие между пользователями и нет необходимости назначать конкретных исполнителей.

Воркфлоу необходим для регламентации взаимодействия между сотрудниками, как при согласовании таймшитов с участием сотрудника, менеджера проекта и линейного руководителя.

## Настройка воркфлоу для каждой сущности

Каждой сущности можно назначить уникальный воркфлоу для создания заданий исполнителям и изменения состояния. Сущности типа Счета, Заявки и Документы могут иметь индивидуальные процессы.

## Совместимость прав доступа и настроек воркфлоу

Рекомендуется применять наборы прав и настройки воркфлоу одновременно. Администраторам могут быть предоставлены права на принудительное изменение состояний, например для переоткрытия таймшита после согласования. Однако принудительное изменение остановит воркфлоу, нарушая автоматизированные шаги.
