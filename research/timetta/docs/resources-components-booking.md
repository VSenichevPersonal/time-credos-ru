# Календарь бронирования

## Назначение

Компонента **Бронирование** отображает информацию о ресурсах и их временных ёмкостях в календарном формате. Система позволяет оптимизировать распределение ресурсов между проектами, выявлять конфликты использования и оценивать долгосрочные потребности.

## Особенности прав доступа

Доступ к бронированию ресурсов требует соответствующих разрешений:
- В области **Проекты**: необходим набор прав для роли **Управление проектом** с активной гранулой Бронирование
- В области **Ресурсы**: требуется набор прав для роли **Управление ресурсами** с активной гранулой Бронирование

Минимальный уровень доступа — "Просмотр".

## Обзор

Календарь служит общим пространством для менеджеров проектов и ресурсных менеджеров по резервированию ресурсов. Главные элементы включают:

- Навигацию по периодам
- Выбор единицы бронирования (часы, проценты, FTE, дни)
- Масштаб просмотра (дни, недели, месяцы)
- Фильтрацию ресурсов
- Отображение загрузки или оставшейся ёмкости
- Визуализацию бронирования в режимах Hard и Soft
- Периоды отсутствия ресурсов

## Различия по режимам бронирования

**Базовый режим**: бронирования отображаются плашками с названием проекта. Плашки можно передвигать и редактировать в карточке бронирования.

**Детальный режим**: бронирования представлены строками проектов с часами, распределёнными по дням. Добавление проектов осуществляется через кнопку «+».

## Ключевые принципы

Важно понимать, что:
- Календарь бронирования функционально отделён от ресурсных планов проектов
- Бронирование выполняется на уровне проекта без детализации до работ
- Это инструмент стратегического, а не оперативного планирования
- Возможны расхождения между планом и бронью на детальных масштабах
- Конфликты бронирования отражаются в карточке проекта, но не требуют немедленного устранения
