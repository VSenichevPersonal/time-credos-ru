# Матрицы ставок

## Назначение

**Матрицы ставок**:
- определяют ставки биллинга и ставки себестоимости универсальных ресурсов;
- зависят от аналитик, описывающих ресурсы компании.

### Права доступа

Для настройки матриц ставок требуется набор прав для роли `Администратор` с активной гранулой «Матрицы ставок».

## Типы матриц ставок

- **Ставки биллинга**. Матрицы тарифов позволяют задать тарифную сетку (определить ставки биллинга) на определённый период времени.
- **Ставки себестоимости**. Этот тип позволяет определить ставки себестоимости для универсальных ресурсов.

## Использование

- **Проекты**:
  - Биллинг для клиентов. Для проекта выбираются тарифы из матрицы ставок биллинга. Эти ставки используются для расчёта стоимости услуг, предоставляемых клиенту.
  - Расчёт себестоимости. Для универсального ресурса себестоимость определяется на основе ставок из матрицы ставок себестоимости.
- **Организации**. Для организаций также назначаются тарифы из доступной матрицы ставок биллинга. Это определяет стоимость услуг для каждой конкретной организации.

### Примечание

Пользователи с активными гранулами **«Тарифы»** имеют доступ к просмотру матрицы ставок биллинга. Матрица доступна во вкладке «Тарифы» проекта или организации. На основе этой матрицы пользователи могут добавлять необходимые тарифы в свой проект или организацию.

## Создание матрицы ставок

Чтобы создать матрицу ставок, необходимо:

1. Перейти к компоненте **Матрицы ставок**.
2. На панели действий нажать на кнопку **Создать матрицу**.
3. Указать:
   - наименование новой матрицы ставок;
   - тип матрицы;
   - начало действия.
4. Нажать **Создать**.

## Карточка матрицы ставок

Карточка матрицы ставок состоит из вкладок **Главная** и **Строки**.

На вкладке **Главная** находятся основные свойства матрицы ставок:

- **Наименование** — название матрицы.
- **Структура матрицы ставок** — список аналитик матрицы.
- **Тип матрицы** — тип матрицы ставок («Ставки биллинга» или «Ставки себестоимости»).
- **Начало действия** — дата начала действия матрицы.
- **Окончание действия** — дата окончания действия.
- **Описание** — текстовое описание матрицы.

### Примечание

**Окончание действия** проставляется автоматически относительно последующей созданной матрицы ставок.

### Важно

Удаление аналитик из структуры матрицы ставок приведёт к удалению её строк с выбранными аналитиками.

## Структура

В карточке задаётся структура матрицы — список аналитик, используемых для её формирования. Доступные аналитики:

- роль;
- компетенция;
- уровень;
- грейд;
- ресурсный пул;
- юридическое лицо;
- локация.

### Иерархия аналитик

Аналитика «Компетенция» не может быть выше, чем аналитика «Роль», а аналитика «Грейд» не может быть выше, чем аналитика «Уровень».

## Заполнение строк

На вкладке **Строки** карточки формируется таблица ставок биллинга. Формировать можно двумя способами:

- Вручную — по кнопке **Создать**.
- Автоматически — по кнопке **Заполнить матрицу**. При этом в таблицу будут добавлены все комбинации из указанных в структуре матрицы аналитик.

### Особенности автоматического формирования

Автоматическое формирование строк матрицы ставок:

- выполняется на основе всех её аналитик;
- удалит прежде созданные строки.
