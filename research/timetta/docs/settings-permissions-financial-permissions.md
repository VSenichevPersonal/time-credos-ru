# Права доступа к финансовым сведениям

Обновлено: 10.01.2024

## Общие сведения

В Timetta права, регулирующие доступ к финансовой информации, определяются гранулами и доступны в наборах прав для нескольких ролей:

- **Финансовые сведения (ProjectFinancial)** — определяет доступность финансовых сведений по проектам.
- **Сотрудники: финансовые показатели (UserFinancial)** — определяет доступность финансовых сведений по сотрудникам.
- **Текущая ставка ресурса (ResourceCurrentRate)** — определяет доступность сведений о текущей ставке себестоимости ресурсов.

### Финансовые сведения

Гранула содержится в наборах прав для роли Управление проектами и определяет доступность следующих данных:

- плановая, фактическая, оценочная, прогнозная себестоимость труда, прибыль и рентабельность проекта;
- плановая, фактическая, прогнозная выручка и затраты проекта;
- плановые, фактические, прогнозные сведения о счетах и оплатах.

Гранула регулирует доступность следующих компонентов:

- показателей **Себестоимость, Затраты, Выручка, Прибыль, Рентабельность, Биллинг** на вкладке **Аналитика** в карточках **Проектов, Программ, Организаций**;
- финансового отчёта (P&L) в карточках **Проектов, Программ, Организаций**;
- представления **Календарь** на вкладках **Выручка** и **Затраты** в карточке проекта;
- показателей по **Себестоимости, Затратам, Выручке, Прибыли, Рентабельности** в списках проектов, программ, организаций.

Права на просмотр финансовых сведений доступны для следующих областей:

- **Мои проекты** — просмотр финансовых сведений по проектам, где пользователь назначен менеджером или со-менеджером проекта.
- **Проекты моих клиентов** — просмотр финансовых сведений по проектам, относящимся к клиентам (организациям), менеджером которых является пользователь.
- **Проекты моих программ** — просмотр финансовых сведений по проектам, относящимся к программам, менеджером которых является пользователь.
- **Все** — просмотр финансовых сведений по всем проектам.

### Сотрудники: финансовые показатели

Гранула содержится в наборах прав для роли Управление командой и определяет доступность следующих данных:

- плановая, фактическая, оценочная себестоимость труда сотрудников;
- показатели выручки в проектах с типом T&M.

Гранула регулирует доступность следующих компонентов:

- показателя **«Выручка»** в списке сотрудников (приложение **«Команда»**);
- показателя **«Себестоимость»** в сводной информации по сотрудникам (разделы **«Ресурсы»** и **«Проекты»**);
- финансовых показателей (себестоимость отсутствий) в общем отчёте P&L (приложение **«Аналитика»**);
- разделов **«Себестоимость»** и **«Выручка T&M»** в отчёте **«Пользователи»**;
- раздела **«Себестоимость»** в отчёте **«Таймшиты детально»**.

Права на просмотр финансовых сведений доступны для следующих областей:

- **Мои подчинённые** — просмотр финансовых показателей по всем пользователям в подчинении.
- **Все пользователи** — просмотр финансовых показателей по всем пользователям.

### Текущая ставка ресурса

Гранула содержится в наборах прав для ролей Управление проектами, Управление командой, Управление ресурсами и определяет доступность сведений о текущей ставке себестоимости ресурса.

Гранула регулирует доступность следующих компонентов:

- показатели **«Себестоимость»**, **Ставка себестоимости** в списке сотрудников (приложения **«Команда»** и **«Ресурсы»**);
- отображение текущей ставки себестоимости сотрудников (пользователей) в команде проекта;
- отображение текущей ставки себестоимости универсальных ресурсов в команде проекта;
- показателя **«Ставка себестоимости»** в отчётах **«Таймшиты детально»**, **«Пользователи»**.

Для роли Управление проектами доступны области:

- **Сотрудники: Текущая ставка** — определяет доступность сведений о текущей ставке себестоимости сотрудников, в частности, в команде проекта.
- **Универсальные ресурсы: Текущая ставка** — определяет доступность сведений о текущей ставке себестоимости универсальных ресурсов, в частности, в команде проекта.

Для роли Управление командой доступны области:

- **Сотрудники: Текущая ставка** для области **Мои подчинённые** — определяет доступность сведений о текущей ставке себестоимости сотрудников по иерархии подчинённости.
- **Сотрудники: Текущая ставка** для области **Все пользователи** — определяет доступность сведений о текущей ставке себестоимости всех сотрудников.

Для роли Управление ресурсами доступны области:

- **Сотрудники: Текущая ставка** для области **Мои пулы** — определяет доступность сведений о текущей ставке себестоимости сотрудников, относящихся к ресурсным пулам, где пользователь назначен менеджером.
- **Сотрудники: Текущая ставка** для области **Все ресурсы** — определяет доступность сведений о текущей ставке себестоимости всех сотрудников.
- **Универсальные ресурсы: Текущая ставка** для области **Все ресурсы** — определяет доступность сведений о текущей ставке себестоимости всех универсальных ресурсов.

> **Изменение в интерфейсе**
>
> В диалоговом окне выбора ресурсов для добавления в команду проекта информация о ставке не отображается. Отображение ставки реализовано непосредственно в интерфейсе команды проекта.

## Неявные права

Роль Управление биллингом даёт права на все финансовые сведения, доступные в системе.

> **Важно**
>
> При создании нового абонента права на гранулу **«Текущая ставка ресурса»** по умолчанию предоставляются стандартным наборам прав для следующих ролей: Администратор проектов, Руководитель, Ресурсный менеджер.
