# Вход в Timetta

Обновлено: 22.07.2025

## Вход в систему

Для входа в систему перейдите по ссылке: [Вход в Timetta](https://app.timetta.com/my/timesheets/current).

> **Примечание**
>
> Войти в систему может только зарегистрированный пользователь. Регистрацию выполняет *администратор* вашей организации. Инструкция по созданию новой организации доступна по [ссылке](https://timetta.com/ru/docs/get-started/registration).

## Идентификация

Для входа в систему введите вашу корпоративную электронную почту.

Способ аутентификации определяет *администратор* системы. Возможные варианты:

- ввод пароля;
- подтверждение через внешних провайдеров SSO (например, ADFS, Keycloak);
- вход через рабочую учётную запись Майкрософт (Azure AD).

## Вход по паролю

Если *администратор* вашей организации выбрал для вашей учётной записи локальную аутентификацию, то после регистрации пользователя администратором на вашу корпоративную почту придёт письмо со ссылкой для установки пароля.

Процесс входа:

1. Введите вашу корпоративную электронную почту и нажмите `Продолжить`.
2. На следующем экране введите пароль и нажмите `Войти`.

## Восстановление пароля

Если вы забыли пароль:

1. Перейдите по ссылке: https://auth.timetta.com/password-recovery.
2. Введите вашу корпоративную электронную почту.
3. Нажмите кнопку `Восстановить пароль`.

В течение нескольких минут вы получите письмо с инструкциями по сбросу пароля.

> **Примечание**
>
> Восстановление пароля для учётных записей, привязанных к внешним провайдерам аутентификации (SSO, Azure AD), осуществляется через соответствующие сервисы.

## Вход через внешних провайдеров (SSO)

1. Перейдите на страницу [Вход в Timetta](https://app.timetta.com/my/timesheets/current).
2. Введите вашу корпоративную электронную почту.
3. Вы будете перенаправлены на страницу входа вашей компании.
4. Пройдите аутентификацию, используя учётные данные, принятые в вашей системе (например, логин и пароль, одноразовый код).
5. После успешной проверки вы автоматически получите доступ к Timetta.

Уточните у вашего *системного администратора*, какой метод аутентификации используется в вашей организации.

## Не удаётся войти

Если при попытке входа возникает ошибка «Аккаунт не найден» или «Не удалось найти пользователя с таким адресом электронной почты»:

Рекомендуемые действия:

1. Проверьте email: убедитесь, что адрес электронной почты, включая домен, введён правильно.
2. Повторите ввод пароля: введите пароль заново, проверяя раскладку клавиатуры и регистр.
3. Восстановите пароль: воспользуйтесь функцией [восстановления пароля](https://auth.timetta.com/password-recovery).
4. Обратитесь к *администратору* Timetta в вашей организации, если описанные выше действия не помогли. Возможные причины:
   - учётная запись не была создана;
   - учётная запись деактивирована;
   - для учётной записи не назначены права доступа.
