# Интерфейс Timetta

Обновлено: 22.07.2025

## Общий вид интерфейса

Интерфейс Timetta организован вокруг нескольких основных компонентов. Система предоставляет пользователям интегрированную среду для управления проектами, ресурсами, финансами и другими бизнес-процессами.

Основные элементы включают:

1. Области системы
2. Панель навигации
3. Настройки панели навигации
4. Маршрут навигации
5. Главное меню
6. Панель действий
7. Рабочее пространство

## Области системы

Области — это функциональные разделы системы, предназначенные для решения конкретных задач. Доступ между ними осуществляется через главную панель приложений.

В зависимости от прав доступа и лицензий, пользователю доступны разные наборы областей:

- **Моя работа** — основная область для любого пользователя, включающая таймшиты, заявки на отсутствия и затраты
- **Команда** — управление сотрудниками и их показателями
- **Проекты** — управление проектами, программами и портфелями
- **Ресурсы** — управление ресурсами и бронированием
- **Биллинг** — отслеживание работ и выставление счётов
- **Финансы** — управление финансовыми операциями
- **Клиенты** — управление клиентами и сделками
- **Аналитика** — создание отчётов и панелей мониторинга
- **Настройки** — конфигурация системы (только для администраторов)

### История навигации

Система отслеживает историю переходов между сущностями, позволяя пользователям быстро вернуться к предыдущим страницам.

## Панель навигации

Панель навигации структурирована следующим образом:

1. **Группы пунктов** — категории (Проекты, Программы, Клиенты, Ресурсы)
2. **Пункты навигации** — конкретные ссылки внутри групп
3. **Кнопка настроек** — для изменения видимости элементов
4. **Кнопки управления** — скрытие и развёртывание панели

## Настройка панели навигации

При открытии настроек доступны следующие возможности:

- Скрытие ссылок с помощью иконки "глазок"
- Раскрытие скрытых ссылок
- Сохранение изменений
- Отмена изменений
- Сброс к значениям по умолчанию

## Маршрут навигации

Маршрут навигации ("хлебные крошки") отображает текущее местоположение пользователя в системе. Типичный маршрут включает:

- Название группы навигации
- Название пункта навигации
- При открытии карточки — наименование сущности
- При навигации в подкомпоненты — название компоненты

## Главное меню

Главное меню в верхней части страницы содержит следующие блоки:

1. Индикатор счётчика времени (Таймтрекер)
2. Меню быстрого создания для новых сущностей
3. Панель уведомлений
4. Меню справки
5. Кнопка смены темы приложения
6. Личное меню пользователя

## Панель действий

Контекстная панель, появляющаяся при открытии любой сущности (задачи, проекта, сделки и т.д.), содержит необходимые кнопки для работы с конкретным объектом.

## Рабочее пространство

Центральная область приложения для работы с данными. Здесь пользователи могут:

- Просматривать и редактировать сущности
- Взаимодействовать с компонентами
- Использовать различные представления данных
- Фильтровать и сортировать информацию

### Страницы сущностей

Предназначены для работы с конкретными объектами системы. Основные элементы:

1. Наименование страницы
2. Панель реквизитов сущности
3. Панель вкладок с различной информацией

### Страницы компонент

Используются для работы с функциональными инструментами (отчётами, настройками, панелями). Включают:

1. Панель инструментов
2. Область с данными

### Представления

Различные способы отображения одних и тех же данных помогают работать с информацией удобным способом.

#### Представление «Список»

Классическое табличное представление, где:

- Каждая строка — одна запись
- Столбцы — атрибуты сущности
- Поддерживается добавление/удаление столбцов
- Доступна сортировка по столбцам

#### Представление «Доска»

Визуальное представление с карточками, перемещаемыми между колонками:

- Каждая колонка соответствует состоянию жизненного цикла
- Поддерживается сортировка и группировка карточек
- Возможны произвольные колонки представления

Представления "Список" и "Доска" отображают один набор данных под разными углами без потери информации.

### Фильтры данных

На страницах с представлениями "Список" и "Доска" реализован механизм поиска и фильтрации.

**Поиск** предназначен для оперативного нахождения записей по текстовым атрибутам.

**Настраиваемый фильтр** обеспечивает детализированное отбирание данных с ключевыми функциями:

1. **Выбор сохранённого фильтра** — применение ранее сохранённых наборов условий
2. **Быстрый выбор** — часто используемые свойства для оперативной настройки
3. **Детальный выбор** — полный перечень свойств для формирования сложных условий
4. **Сохранение настроек** — сохранение конфигурации фильтра под уникальным именем
