# Timetta: Подразделения (скачано 2026-06-22, /ru/docs/settings/users/departments)

**Поля:** Наименование · Код · **«Входит в»** (родительское подразделение = иерархия; автозащита от циклических ссылок) · **Роль** (ассоциированная с подразделением) · Сущность активна.
**Иерархия:** многоуровневая через «Входит в»; цикл невозможен (в выборе только не-циклические).
**Привязка сотрудника:** поле «Подразделение» на вкладке Профиль карточки пользователя. Множественной привязки / %  / дат — НЕТ в их модели.
**Влияние:** подразделение в команде проекта → его юзеры выбирают ассоциированные роли в таймшите. Скоупинг/согласование от подразделения — в доке не детализировано.

## Вывод для REQ-0018 (наше)
- `parentDepartment` (иерархия) = ✓ как Timetta «Входит в» + ДОБАВИТЬ защиту от циклов.
- `head` (руководитель отдела) = НАШЕ расширение (у Timetta нет head-поля, они через роли/воркфлоу). Оставляем head — проще для «кто чей руковод» в Кредо-С.
- employee×dept FTE+даты (REQ-0011) = НАШЕ расширение сверх Timetta (у них single). Осознанно — мульти-отдел реальность Кредо-С.
