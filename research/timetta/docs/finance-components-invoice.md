# Счёт

Обновлено: 29.07.2025

## Назначение

Компонента Счёт позволяет формировать счета к оплате. Счёт включает в себя список оказанных работ и понесённых в рамках проектов затрат.

### Права доступа

Для доступа к счетам требуется набор прав для роли Управление биллингом с активной гранулой «Биллинг».

## Обзор

### Карточка счёта

Карточка счёта содержит основные сведения о счёте, строки для добавления работ и затрат с суммами, а также дополнительные сведения:

**Основные сведения:**

- **Клиент** — организация, которой будет выставлен счёт. После создания счёта выбрать другого клиента нельзя.
- **Проект** — проект, для которого создаётся счёт. После создания счёта изменить проект нельзя.
- **Шаблон** — шаблон для формирования счёта. Подробнее — Шаблон счёта.
- **Описание (назначение)** — назначение счёта. Добавляется в форму счёта.
- **Адрес для счёта** — адрес для выставления счёта. После создания можно изменить, только если счёт находится в состоянии «Черновик».
- **Номер счёта** — номер счёта.
- **Дата выпуска** — дата выпуска счёта. По умолчанию устанавливается текущая дата.
- **Срок оплаты** — плановая дата оплаты. По истечении этой даты оплата будет считаться просроченной.

**Команды добавления строк.** Состав команд зависит от типа проекта. Например, если тип — «Время и затраты», то в карточке счёта будет доступна команда «Добавить T&M работы».

**Выбранные строки.**

**Дополнительные сведения:**

- **Заметки для клиента** — текстовый блок, который вставляется в форму счёта.
- **Внутренние заметки** — текстовый блок, доступный только при просмотре в веб-клиенте (не включается в форму и недоступен клиенту).

Для счёта доступна стандартная Лента активности.

### Шаблон счёта

Шаблон счёта выбирается в карточке счёта и определяет:

- реквизиты счёта (например, название юридического лица, платёжные реквизиты);
- форматирование печатной формы;
- применение НДС.

Шаблоны счетов настраивают администраторы. Подробнее — Шаблоны счетов.
