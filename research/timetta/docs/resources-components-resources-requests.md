# Запрос на подбор ресурсов и актуализацию бронирования

## Назначение

Запросы на ресурсы организуют взаимодействие между менеджером проекта и ресурсным менеджером. Менеджер проекта формирует запрос о потребности в ресурсах, а ресурсный менеджер подбирает и бронирует подходящих сотрудников.

Эти запросы помогают планировать нагрузку, поддерживать утилизацию на нужном уровне и избегать простоя или перебукинга.

## Особенности прав доступа

Различные действия требуют специфических наборов прав:

- Создание запросов требует роли "Управление проектом" с гранулой "Запросы ресурсов"
- Работа с бронированием в области "Ресурсы" требует роли "Управление ресурсами" с гранулой "Бронирование"
- Работа с входящими запросами требует роли "Управление ресурсами" с гранулой "Запросы ресурсов"

Права на управление состояниями определяются настройками жизненного цикла и воркфлоу.

## Использование

Запросы применяются в:
- **Проекты** — замена универсального ресурса конкретным доступным
- **Ресурсы** — отражение потребности проекта в ресурсах
- **Отчёты** — информация по запросам в рамках проектов

## Типы запросов

**Запрос на подбор** — первичный запрос для определения сотрудника с выделенным количеством часов. Новое бронирование добавляется к существующему.

**Запрос на актуализацию** — вторичный запрос для уточнения сотрудника или часов. Результат заменяет существующее бронирование.

## Запрос на подбор ресурса

Создаётся для универсальных ресурсов. Карточка содержит:

- Член команды (наименование универсального ресурса)
- Роль, ресурсный пул
- Параметры: уровень, грейд, локация, компетенция
- Юридическое лицо и тип трудоустройства
- Период (начало, окончание)
- Запрошено (количество часов)
- Заметка (комментарий)

Вкладка "Дополнительно" может содержать предпочитаемых сотрудников. Строка запроса заполняется из ресурсного плана и может редактироваться вручную.

## Запрос на актуализацию бронирования

Используется, когда пользователь уже в команде, но требуется добавить или обновить бронирование. Имя пользователя в поле "Член команды" заполняется автоматически.

## Стандартный воркфлоу запроса на ресурс

По умолчанию исполнители на переходы не назначены и задаются в воркфлоу. Инициатор — автор запроса (менеджер проекта), исполнитель — ресурсный менеджер.
