# Масштаб бронирования

Обновлено: 23.07.2025

## Назначение

Масштаб бронирования используется в компонентах [Бронирование](https://timetta.com/ru/docs/resources/components/booking) и [Потребность в ресурсах](https://timetta.com/ru/docs/projects/components/projects/booking-conflicts).

**Права доступа**

Для настройки масштаба бронирования требуется набор прав для роли **Администратор**.

## Обзор

Доступно три масштаба бронирования: по дням, неделям и месяцам. Масштаб бронирования ресурсов можно зафиксировать в области [Настройка](https://app.timetta.com/settings/settings/applications):

![Зафиксировать масштаб бронирования](https://timetta.com/entity-assets/ef793db8-310b-4eb8-9eb6-44c911388b95/fix-the-scale.jpg)

После фиксации изменение масштаба в компонентах будет недоступно:

![Зафиксированный масштаб](https://timetta.com/entity-assets/ef793db8-310b-4eb8-9eb6-44c911388b95/fixed-scale.png)

Если опция не включена, пользователям будет доступно переключение масштаба бронирования в любой момент в соответствующих компонентах.

Распределение значений большего масштаба (например, с месяца на день) производится пропорционально доступному рабочему времени на каждый период по расписанию пользователя.

![Незафиксированный масштаб](https://timetta.com/entity-assets/ef793db8-310b-4eb8-9eb6-44c911388b95/non-fixed-scale.png)

> **Совет**
>
> Рекомендуется фиксировать масштаб бронирования. Если **Менеджер проекта** и **Ресурсный менеджер** работают в разных масштабах бронирования, могут возникать конфликты брони.
