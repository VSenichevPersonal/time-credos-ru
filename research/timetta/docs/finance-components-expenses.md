# Затраты проекта

## Назначение

Компонент **Затраты** позволяет вводить оценочные показатели и служит аналитическим отчётом, отображающим плановые, фактические и прогнозные значения. Он упрощает актуализацию оценки и контроль затрат по проекту.

Суммы по каждому типу затрат отображаются в Финансовом отчёте.

### Требования доступа

Для просмотра оценки затрат требуется набор прав роли "Управление проектами" с активными гранулами "Проекты" и "Оценка затрат" (минимум "Просмотр").

Для доступа к календарю затрат требуется набор прав роли "Управление проектами" с активной гранулой "Финансовые сведения".

## Обзор

Календарь затрат сравнивает значения оценки, плана, факта и прогноза по прямым затратам на проект. Данные агрегируются по временным периодам: дням, неделям или месяцам.

### Элементы интерфейса

1. Переключатель представлений между календарём затрат и списком оценок затрат
2. Кнопки навигации по периодам
3. Выбор масштаба группировки (дни, недели, месяцы)
4. Кнопка создания периодических затрат
5. Настройка отображения учётных статей
6. Шкала времени
7. Опции выбора отображаемых групп показателей
8. Типы показателей: план, оценка, факт, прогноз
9. Кнопка развёртывания всех строк затрат
10. Работы проекта с разбивкой по типу затрат

### Прочие показатели

Строки "Прочий факт" и "Прочий план" появляются при наличии скрытых данных по удалённым работам или работам из других версий проекта. Иконка глаза позволяет отобразить скрытые данные.

## Особенности календаря затрат

В структуре работ календаря отображаются:

- Работы из текущей открытой версии или проекта
- Значения из нужной версии или проекта при наличии связей между работами

Если в проекте есть фактические данные по работам, отсутствующим в текущей версии, они отображаются в строке "Прочий факт" без разбивки. Если в мастер-версии есть плановые данные по отсутствующим работам, они показываются в строке "Прочий план".

Примечание: Себестоимость в календаре затрат всегда группируется по этапам.
