# Настройка наборов прав

## Назначение

Компонента предназначена для создания и конфигурирования наборов прав, а также управления ими.

Наборы прав:
- Дают доступ к областям системы, данным и действиям с ними
- Могут быть назначены пользователям и группам пользователей

**Примечание:** В статье описывается создание и конфигурирование наборов прав. Подробнее о них — [Как работают права доступа](/ru/docs/settings/permissions/how-premissions-sets-works).

## Создание наборов прав

Для создания нового набора прав следует:

1. Перейти к компоненте **Наборы прав**
2. На панели действий нажать на кнопку **Создать набор прав**
3. Задать системную роль, для которой создаётся набор прав (поле **Роль**), и его **Наименование**
4. Нажать на кнопку **Создать**

**Совет:** Создание дополнительных наборов прав затрудняет администрирование, поэтому лучше минимизировать их количество. В большинстве случаев для разграничения прав доступа достаточно стандартных наборов, которые создаются в Timetta по умолчанию при развёртывании абонента.

## Конфигурация набора прав

Ключевые элементы:

1. Наборы прав состоят из **гранул** — минимальных единиц настройки доступа (например, «Проекты», «Задачи»). Состав гранул в наборе прав зависит от системной роли, для которой он создан.

2. Большинство гранул разделяются на области — выборки данных, для которых применяется права доступа (например, «Мои проекты», «Все проекты»).

3. Для каждой гранулы назначаются права — разрешения на чтение, редактирование, удаление или выполнение других действий с указанной выборкой данных.

### Пример

Допустим, пользователю назначен набор прав для роли **Управление проектами**. В этом наборе активны права «Просмотр» для области «Все проекты» гранулы «Проекты». Это значит, что пользователь сможет просматривать в области **Проекты** список всех созданных для этого абонента проектов.
