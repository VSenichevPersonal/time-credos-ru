# Периоды таймшитов

Обновлено: 26.08.2025

## Назначение

**Период таймшита** — правило, которое определяет периодичность формирования таймшитов для пользователей и календарные рамки каждого таймшита. По умолчанию в Timetta применяется недельный период.

> **Примечание**
>
> Для настройки периодов таймшитов требуется набор прав для роли `Администратор` с активной гранулой «Периоды таймшитов».

> **Важно**
>
> Настройка предназначена для опытных администраторов. Не рекомендуется менять настройки по умолчанию без реальной необходимости. Некорректная настройка периодов может привести к невозможности создания отдельных таймшитов.

## Типы периодов

- **Недельный** — таймшиты создаются последовательно, на каждую календарную неделю.

- **Недельный по конец месяца** — таймшиты создаются последовательно на каждую календарную неделю, но с «разрывом» на начало нового месяца. Таким образом, не будет таймшитов, которые включают даты двух месяцев. Это удобно, если в компании принят месячный учёт.

  > **Пример**
  >
  > Для календарной недели с 28.08.2023 по 03.09.2023 будет создано два таймшита: с понедельника по четверг (до 31.08.2023) и с пятницы (с 01.09.2023) по воскресенье.

- **Месячный** — таймшиты создаются последовательно на календарный месяц.

- **Полумесячный** — таймшиты создаются последовательно с длительностью, выбранной при создании. По умолчанию это таймшиты с 1 по 14 день месяца и с 15 по конец каждого месяца. Таким образом, на каждый месяц приходится по два таймшита.

- **Произвольный** — позволяет указать произвольный период. На весь произвольный период будет создан один таймшит. Этот тип используется только для закрытия «окон» при переходе с одного типа периода на другой.

## Области действия

- **Подразделение** — период будет применяться ко всем сотрудникам конкретного подразделения. Допустима ситуация, когда для одного подразделения используются недельные таймшиты, а для другого — месячные.
- **Вся система** — период будет применяться ко всем сотрудникам компании.

## Создание периода

Чтобы создать период таймшита, нужно:

1. Перейти к компоненте **Периоды таймшитов**.
2. На панели действий нажать на кнопку **Создать период**.
3. Заполнить в форме создания **Тип периода** и **Область** его действия.
4. Нажать на кнопку **Создать**.

## Настройка периода

Для разных периодов и областей действия карточки отличаются. Пример карточки для типа «Недельный по конец месяца» и области «Подразделение»:

- **Тип** — выбранный тип периода. После создания редактировать нельзя.
- **Область** — выбранная область действия периода. После создания редактировать нельзя.
- **Действует с** — дата начала действия периода.
- **День начала недели** (для типов «Недельный» и «Недельный по конец месяца») — устанавливает первый день недели, с которого будет начинаться таймшит.
- **День начала месяца** (для типа «Месячный») — устанавливает число месяца, с которого будут начинаться месячные таймшиты.
- **День начала первого периода** (для типа «Полумесячный») — устанавливает число месяца, с которого будет начинаться первый таймшит каждого месяца.
- **День начала второго периода** (для типа «Полумесячный») — устанавливает число месяца, с которого будет начинаться второй таймшит каждого месяца.
- **Действует по** (для типа «Произвольный») — устанавливает дату окончания таймшита, создаваемого на произвольный период.
- **Подразделение** (для области «Подразделение») — устанавливает подразделение, на которое распространяется период таймшита.
- **Сущность активна** — признак активности периода. Если сущность неактивна, она не учитывается.

## Как работают периоды таймшитов

Таймшиты создаются автоматически при первом обращении пользователя к текущему таймшиту или при переходе к следующему в карточке таймшита.

Таймшиты создаются таким образом, чтобы они шли последовательно и без «разрывов».

При запросе таймшита на конкретный день или следующего таймшита проверяется, создан ли таймшит на период. Если не создан, то он создаётся с учётом текущего действующего периода.

Иногда не получается создать таймшит без разрыва. Например, на первую неделю месяца уже был создан таймшит с периодом «Недельный». Затем период был изменён на месячный. В этом случае создаётся нестандартный таймшит, который закрывает разрыв.

Таким образом, можно в любой момент создать период таймшитов и указать дату начала действия. Система сама выполнит переход на новый период, обеспечив неразрывность учёта.

> **Совет**
>
> Уже созданные таймшиты автоматические не изменяются — даже если изменить период, в который попадает таймшит.
>
> Если нужно изменить период и обновить уже созданные таймшиты, то их необходимо предварительно удалить. При обращении таймшиты будут созданы заново с учётом нового периода.
