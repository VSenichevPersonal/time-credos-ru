## Фаза 01 — Time MVP

**Цель:** работающий SDK-app учёта трудозатрат: 5 отделов, проекты/этапы, недельная сетка + таймер, планирование загрузки, отключаемое согласование, сид-данные янв–июнь 2026 (реальные ОВ + мок). Всё на русском. Устанавливается в workspace.

**Зависимости:** локальный Twenty 2.x (Docker) для дев. Форк — только для финальной установки (по ходу).

**Критерии успеха (наблюдаемые):**
- В сайдбаре workspace — раздел-папка «Трудозатраты» с пунктами.
- 7 объектов `tt*` созданы, видны (view+nav), на русском.
- Недельная сетка открывается, часы вводятся/сохраняются, итоги считаются.
- Таймер start/stop создаёт запись.
- Доска планирования показывает загрузку отделов по неделям.
- Согласование включается/выключается на отделе/проекте.
- Сид: реальные ОВ (3501) + мок 4 отделов загружены за янв–июнь 2026.
- Все надписи русские; нейминг `tt`; код в зелёной зоне (ноль правок ядра).

## Принципы исполнения
- Каждый план — отдельный Agent-вызов со своим `<context>`; после — коммит + SUMMARY.
- Вертикальные срезы (объект+поля+связь вместе), не горизонтальные слои.
- Перед кодом плана — `yarn twenty dev --once --dry-run` (предпросмотр).
- Видимый объект ВСЕГДА = defineObject + defineView + defineNavigationMenuItem.
- Мутации из front — только через `/s/`-route.
- Dev-report по каждой значимой сессии в `.planning/dev-reports/`.
