# UI Playbook — гардрейлы фронта в виджетах Twenty

**Владелец:** Dev 1. **Зачем:** front-component в Twenty — это **виджет фиксированного размера**, корень которого почти всегда `overflow: hidden`, и в нём **нет React-порталов** к `body`. Отсюда целый класс багов: всё, что вылезает за границу виджета (поповеры, длинный текст, скролл), **молча обрезается**. Этот плейбук — чтобы такие баги не появлялись, а не чинились по скриншоту.

> Триггер создания: баг add-row (дропдаун у нижней кромки таймшита уходил под границу виджета → клип). Системный фикс — авто-флип в `grid/autocomplete.tsx`. См. §2.1.

---

## 1. Базовый закон

**Виджет — это коробка фикс-высоты с `overflow: hidden`.** Любой `position: absolute` элемент, выходящий за её низ/право, обрезается. Портала наружу нет. Значит:

- ❌ Нельзя рассчитывать, что поповер «просто откроется поверх всего».
- ✅ Всё держим **внутри** коробки: меняем направление, ограничиваем высоту, скроллим локально.

CLAUDE.md pitfall (verbatim): *«Creating a front-end component that has a scroll instead of being responsive to its fixed widget height and width, unless it is specifically meant to be used in a canvas tab.»*

---

## 2. Правила по классам элементов

### 2.1 Поповеры / дропдауны / автокомплиты / тултипы

- **Авто-флип направления.** На открытии измерь `getBoundingClientRect()` инпута. Если места снизу мало — открывай **вверх**. Никаких безусловных `top: 33`.
- **Кап высоты.** `maxHeight = clamp(120, доступное_место − отступ, 260)` + `overflowY: auto`. Меню само скроллится, не растягивает виджет.
- **Переиспользуй SSOT.** Комбобокс уже флипается — `grid/autocomplete.tsx`. **Не плодить инлайновые дропдауны**: новый — оборачивай в общий компонент или вынеси флип-хук.

Эталон (реализован):
```ts
const openMenu = () => {
  const r = inputRef.current?.getBoundingClientRect();
  if (r) {
    const below = window.innerHeight - r.bottom - 8;
    const above = r.top - 8;
    const up = below < 200 && above > below;
    setDropUp(up);
    setMaxH(Math.max(120, Math.min(260, up ? above : below)));
  }
  setOpen(true);
};
// style: ...(dropUp ? { bottom: 33 } : { top: 33 }), maxHeight: maxH
```

### 2.2 Длинный текст (имена проектов, ФИО, виды работ)

- Всегда `overflow: hidden; textOverflow: ellipsis; whiteSpace: nowrap` + `title` с полным текстом.
- Колонка имени — фикс-ширина (`width` + `minWidth`), не `flex` без предела.
- Названия каскадов (`Клиент · Проект · Работа`) собирать через хелпер, обрезать целиком, не по слову.

### 2.3 Таблицы / сетки (таймшит, capacity)

- **Горизонталь:** скролл-контейнер (`overflow: auto`) + `sticky left: 0` на колонке имени и угле. Не давать строке раздуть виджет.
- **Вертикаль:** скроллится **только** внутренний регион строк (`flex: 1; overflow: auto`), а шапка/тулбар/футер/add-row — фиксированы. Не весь виджет.
- **Числа:** `fontVariantNumeric: 'tabular-nums'`, формат через `format.ts`, ширина ячейки не пляшет от значения.

### 2.4 Тулбары / фильтры

- `flexWrap: wrap` если чипов много — не уезжать вправо за кромку.
- Чипы фильтров с длинным значением — тоже ellipsis + `title`.

## 3. Чеклист перед `[signal-arch]` (любой UI-PR)

Прогнать в голове/в UI **на узком и на коротком** виджете:

- [ ] Открыл каждый поповер/дропдаун **у нижней и правой кромки** — не обрезан, флипается.
- [ ] Подставил **самый длинный** реальный текст (проект/ФИО) — ellipsis, не ломает раскладку.
- [ ] Прокрутил список/сетку — скроллится **внутренний** регион, не дёргается весь виджет; sticky-колонка на месте.
- [ ] Состояния: **загрузка / пусто / ошибка** — все есть и помещаются.
- [ ] Клавиатура: Tab/↑↓/Enter/Esc работают, фокус-кольцо видно.
- [ ] Нет хардкода ярлыков/статусов — из констант/GLOSSARY.
- [ ] `yarn oxlint <файл>` + typecheck чисто.

## 4. Anti-patterns (стоп-лист)

- ❌ Безусловный `top`/вниз у поповера. → авто-флип (§2.1).
- ❌ Скролл всего виджета вместо локального региона. → §2.3.
- ❌ `flex` на колонке текста без `minWidth`/ellipsis. → §2.2.
- ❌ Инлайновый самописный дропдаун рядом с готовым `autocomplete.tsx`. → переиспользовать.
- ❌ side-stripe `border-left` как акцент, gradient-text, glassmorphism по умолчанию (impeccable absolute bans).
- ❌ Хардкод русских ярлыков в компоненте (рассинхрон с GLOSSARY).

---

## 5. Когда нужен портал/оверлей наружу

Если элемент **физически** не помещается в виджет (большой пикер, модалка) — это сигнал, что фича просит **отдельный экран/таб или command-menu**, а не поповер внутри плотного виджета. Поднять `[design-proposal]`, не городить клиппящийся оверлей.
