# DP-0003 — Редизайн визуализации категорий в «Отчётах»

**Автор:** Dev 1 (design) · **Дата:** 2026-06-21 · **Статус:** PROPOSED
**Запрос заказчика:** «категории вижу, но легенда непонятна, и у всех не состав как диаграмма — а 1 цвет. Не понимаю. Пересмотри UX/UI.»

## Диагноз (по скриншоту + коду `reports/breakdown-table.tsx`)

Строка несёт 2 бара: `Bar` (факт vs норма) + `CategoryBar` (стек byCategory). На текущем срезе (**Проекты**) оба вводят в заблуждение:

1. **Стек категорий на проекте бессмыслен.** Проект принадлежит ОДНОЙ категории → стек = один цвет на всю ширину → выглядит «сломанным/пустым». Состав категорий имеет смысл только там, где **микс**: Отдел / Сотрудник / Итого.
2. **Бар «факт vs норма» на проекте врёт.** У проекта `norm = null` → бар масштабируется по `max(факт)` строк. Это не утилизация и не прогресс — просто относительный размер. Читается как доля, но это не доля.
3. **Легенда неразличима.** 6 категорий, цвета мутно-сине-серые, почти одинаковы (`report-tokens`). Глаз не отличает.
4. **Колонки без подписи** — что левый/правый бар, не сказано.

## Редизайн

### 1. Палитра категорий (SSOT) — `shared/category-colors.ts` (готово)
6 различимых спокойных цветов: Клиент=акцент-синий (главное/биллабл), Пресейл=фиолет, Пилот=бирюза, Внутренний=нейтраль, Инфраструктура=янтарь, Обучение=зелёный. Заменяет мутные `report-tokens` категории.

### 2. Срез «Проекты» — категория = ЧИП, не стек
```
ОПИБ-2026-002 · КрафтХайнц · Анализ защ.   ●На клиента   11ч   план 40ч · остаток 29ч
```
- Категория — **цветной чип** (точка+ярлык) у имени. Один цвет — это нормально для чипа.
- Вместо фейкового «норма-бара» — **бюджет проекта**: `план (plannedEffort) · остаток / перерасход` (мини-бар факт/план, терракот при перерасходе). Это PRJ-FACT, осмысленно для проекта. (Нужен `plannedEffort` в byProject — есть в бюджет-агрегате F-A.)
- утил/недогруз для проекта = «—» (нормы нет) — честно.

### 3. Срезы «Отделы»/«Сотрудники»/«Итого» — стек + расшифровка
```
ОПИБ  [████████░ 82%]  [▇На клиента ▃Пресейл ▁Внутр]  247ч  65%  +120
                         └ наведение → Explainable: На клиента 180ч 73% · Пресейл 40ч 16% · Внутр 27ч 11%
```
- Стек категорий **только здесь** (реальный микс).
- Стек обёрнут в **`<Explainable>`** (готов, DP-0002): наведение/клик → состав (категория · часы · доля). Прямо закрывает «кликабельно/расшифровка».
- Бар «факт/норма» подписать; недогруз — цвет (зелёный недогруз / терракот перегруз) уже есть.

### 4. Легенда
Новые свотчи (различимые), **наведение на категорию подсвечивает её** во всех барах (фокус). Скрывать категории с нулём в текущем срезе.

### 5. Подписи колонок
Шапка: «Загрузка (факт/норма)» · «Категории» · «Факт» · «Утил.» · «Недогруз». Убирает «что это за полоски».

## Что готово (моя зона, design-system)
- `shared/category-colors.ts` — палитра. ✅
- `shared/explainable.tsx` — `<Explainable>` поповер (формула+состав, DOM-free). ✅

## Импл (reports/ — зона второго фронт-инстанса)
- `category-bar.tsx` — рендерить стек только для агрегатов; для проекта — `<CategoryChip>`. Цвета из `shared/category-colors`.
- `breakdown-table.tsx` — проект-строка: чип+бюджет; агрегат-строка: стек в `<Explainable>`. Подписи колонок. Заменить норма-бар для проектов на бюджет-бар.
- `category-legend` — новые свотчи + hover-highlight.

**Координация:** палитра + `<Explainable>` отдаю как design-system. Импл в reports/ — второй фронт по делёжу, ЛИБО я кроссю по слову заказчика.
