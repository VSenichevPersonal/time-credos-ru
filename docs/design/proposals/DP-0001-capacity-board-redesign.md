# DP-0001 — Редизайн доски «Планирование» (capacity-board)

**Автор:** Dev 1 · **Дата:** 2026-06-20 · **Статус:** PROPOSED (ждёт `[arch-ok]`)
**Файлы:** `apps/time/src/front-components/capacity/*` · **Схема:** не трогаем (P0–P1). Только P3 просит поле у Dev 2.

## Проблема

Раздел «непонятный». Корень: **спека и UI отвечают на разные вопросы.**
- CAPACITY_PLANNING.md §1: *«Когда отдел освободится, чтобы взять новый проект? → продажи обещают клиенту старт».* Метрика §2: `свободно = ёмкость − загрузка`.
- UI показывает **стену % загрузки**. Ответ «свободен с сентября / свободно 120ч» нигде не написан — его надо вычислять глазами по 16 колонкам.

Доп. находки (critique impeccable, ~22/40): ёмкость спрятана в `title`; 16 недель без месячных бэндов и «сегодня» — колонку не сопоставить с календарём; mode-switcher overview/detail избыточен; цвет инвертирован (перегруз орёт, свободное приглушено) — задом наперёд для доски «где продать»; допущения (`headcount × capacityFactor`) скрыты.

## Решение (scope согласован с заказчиком: полный P0+P1)

### Ячейка — тоггл метрики (дефолт «свободно ч»)
Переключатель вверху доски: **`% загрузки · свободно ч · план ч`**. Дефолт — **свободно ч** (`+120ч / −40ч`), т.к. это то, что продажи могут ещё обещать (= метрика спеки). Фон ячейки красит по `ratio` (как сейчас), число — по выбранной метрике. `tabular-nums`.

### P0-1 «Свободен с» — ответ на вопрос
Ведущий бейдж на строке отдела: **«свободен с {месяц}»** = первый период, где `ratio < порога` (дефолт 0.9). Если перегружен весь горизонт — «нет окна». Считается из готовых `LoadCell[]`, без новых данных.

### P1-1 Каркас времени
2-строчная шапка: **месяц-бэнд → неделя** (для granularity=week). Левая грань колонки «сегодня» (токен `T.todayCol` уже есть). 16 недель станут читаемыми.

### P1-2 Перекрас под цель
Свободная ёмкость — **позитивный видимый сигнал** (спокойная заливка по объёму свободного, токен `T.okSoft/ok`), нейтрально когда полно, тревога (`T.overSoft/over`) только при `ratio > 1.0`. Сейчас наоборот. Обновить `loadTone()` в `cap-tokens.ts`.

### P1-3 Убрать mode-switcher
Каждая строка отдела **всегда раскрываема** (▸ → проекты + «без плана»). Минус один контрол (overview/detail), детализация по требованию. Granularity-свитч (Недели/Месяцы) оставить.

### P2 (если в этот же заход)
- Строка-итог **«Все отделы»** — загрузка по компании.
- **Видимые допущения** чипом на отделе: `8 чел × 0.8` (сейчас в `title`).
- **Budget-бар проектов** в детализации: `plannedEffort` израсходовано/осталось (Kimai BudgetTrait).

### P3 (нужен Dev 2 — отдельный REQ)
- Поле `tentative` на проекте → **what-if пресейл-бронь** (спека §4 упоминает флаг): продажи видят импакт сделки до закрытия.
- **CSV-export** снимка ёмкости для встречи продаж.
