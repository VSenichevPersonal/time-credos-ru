# DP-0006 — UX доски планирования: прозрачность, понятность, разнообразие интерфейсов

**Автор:** Dev 1 · **Дата:** 2026-06-22 · **Статус:** PARTIAL (батч 🟢 внедрён) + ROADMAP
**Запрос заказчика:** «доска планирования — делать прозрачнее/понятнее/удобнее; разнообразить интерфейсы». Серия UX-замечаний по скриншотам (метрика в drill, мульти-отдел сумма, срок-дата, цвет без легенды).

## Контекст
Доска (`capacity-board`) = плотный heatmap загрузки: оси Отделы/Люди × периоды (Недели/Месяцы), метрика Свободно ч / Загрузка % / План ч, режим «Планировать». Цвет: зелёный=свободно (чем больше — ярче), прозрачный=полная, терракот=перегруз (`cap-tokens.loadTone`). Отвечает на вопрос продаж «где/когда продать».

## Roadmap (приоритизировано)

### 🟢 Дёшево, высокая читаемость
- **R1 Спарклайн загрузки** в строке отдела/человека: мини-тренд за горизонт рядом с названием → пик/провал виден без чтения цифр. Чистый фронт (SVG-path или div-бары, DOM-free). Реюз `loadTone`.
- **R2 Today-маркер**: вертикаль «сейчас» в шкале (как `todayCol` в таймшите). Сейчас только тик 1-й колонки.
- **R3 Разделители месяцев** жирнее (бэнды Июнь/Июль сливаются).
- **R4 «Без плана» → кликабельно**: серый текст внизу детализации → сразу планировать.

### 🟡 Удобство
- **R5 Inline-план без режима**: клик по ячейке проекта → ввод часов на месте (убрать модальный режим «Планировать»).
- **R6 Пресеты срока**: «+1 мес», «до конца квартала».
- **R7 Поиск/фильтр** по проекту/отделу.

### 🔵 Разнообразить интерфейсы (новые виды)
- **R8 Timeline/Гантт**: проекты-полосы по времени (старт→срок), толщина=часы → видно **перекрытия** проектов и окна. Альтернатива гриду (тумблер вида). Песочница-safe (абсолютное позиционирование по %, без host-DOM).
- **R9 Профиль загрузки** (area-чарт на отдел): загрузка vs ёмкость по времени — провалы/пики наглядно.
- **R10 What-if «куда влезет проект N ч?»**: ввёл часы + отдел → подсветка первого окна, где помещается (реюз `firstFreePeriod`-логики). Прямой ответ продажам «когда обещать старт».
- **R11 People-карточки**: на человека — его проекты-полосы + свободные окна.

### Роль-зависимость цвета (открытый вопрос UX)
Зелёный=свободно: продажам = возможность, деливери = простой ресурса (плохо). Предложение: тумблер «взгляд: Продажи / Ресурсы» либо подпись метрики под аудиторию. Решение — с заказчиком.

## Зоны / риск
- Всё внедрённое + R1-R11 — **фронт (Dev 1)**, без Dev2. Песочница-safe (DOM-free, SVG/проценты).
- R8 Timeline — крупнейший, отдельная итерация.
- Данные те же (`useCapacity` cells/periods/shares) — новые виды не требуют новых REST.

## Рекомендация
Следующий батч: **R1 спарклайн + R2 today + R3 месяцы** (дёшево, читаемость). Потом **R10 What-if** (прямая ценность продажам). **R8 Timeline** — когда будет запрос на альтернативный вид.
