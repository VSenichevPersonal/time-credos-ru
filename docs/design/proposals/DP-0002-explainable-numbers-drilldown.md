# DP-0002 — Объяснимые числа (click-to-explain / drilldown)

**Автор:** Dev 1 · **Дата:** 2026-06-21 · **Статус:** PROPOSED (план, ждёт раздачу arch)
**Запрос заказчика:** в Отчётах (OLAP drilldown уже в плане) **и в Планировании** — по **каждой цифре** клик → видно **как считается** (формула) и **из чего складывается** (компоненты).

## Цель

Любой агрегат на экране — не «чёрный ящик». Клик по числу → поповер/панель:
1. **Формула** с подставленными значениями: напр. `Утилизация = клиент / факт = 120 / 247 = 49%`.
2. **Состав** — строки, которые суммируются в это число (с их вкладом), сортировка по убыванию.
3. **Ссылки на источник** (drill): переход к записям/проекту/сотруднику, формирующим число.

Доверие к цифрам + самообъяснение вместо обучения. Закрывает «почему тут столько».

## Surfaces (где применяем)

| Экран | Число | Объяснение |
|---|---|---|
| **Планирование** (capacity) | ячейка загрузки (свободно/% /план) | формула `ёмкость = раб.часы × числ. × коэф`; состав = проекты отдела с их `plannedEffort`-вкладом в период; «свободно = ёмкость − Σплан» |
| Планирование | бейдж «свободен с {мес}» | первый период где ratio < порога — показать ratio по периодам |
| **Отчёты** | `util` | `client / fact`; состав = byCategory (клиент vs прочее) |
| Отчёты | `norm` | `база(календарь) × числ. × коэф`; состав = рабочие дни периода |
| Отчёты | `under` (недогруз) | `norm − fact`; обе части кликабельны рекурсивно |
| Отчёты | `fact` | состав = byProject / byEmployee (drill вглубь среза) |
| **Таймшит** | «Итого по дню/неделе», «План» | состав = строки (проект×вид работ), вклад каждой |

## UX-паттерн

- Число с объяснением — **подчёркнуто пунктиром** (`text-decoration: underline dotted`) + `cursor: help/pointer`. Сигнал «кликни».
- Клик → **поповер** (в фикс-виджете — UI_PLAYBOOK §0/§2.1: без DOM-замеров, направление структурное) ИЛИ **боковая панель** для глубокого drill.
- Поповер: заголовок (что за число) · формула-строка (моноширинный, подставленные значения) · мини-таблица состава (название · значение · доля) · кнопка «Открыть записи →» (`navigate`, см. RECIPES).
- Рекурсия: части в составе сами кликабельны (drill вниз по OLAP-осям) — но визуально ограничить 2-3 уровня, дальше → переход на отфильтрованный список/дашборд.

## Технически (front-first)

- **Поверхностный explain (P1):** данные уже загружены на клиенте (capacity cells знают проекты-вклад; reports byCategory/byProject/byEmployee). Поповер строится из имеющегося — **без новых запросов**. Чистые `explain*()`-функции рядом с `calc-load.ts` / reports.
- **Глубокий drill (P2):** ссылка на источник — `navigate(AppPath.RecordIndexPage, ...)` с фильтром, или подгрузка записей по REST в панель.
- **Контракт от бэка:** где состав не выводится на клиенте (серверная агрегация /s/reports) — Dev 2 возвращает `breakdown` в ответе (OLAP research уже это закладывает). Согласовать в OLAP-контракте.
