# Плейбук: ведение мануалов (аналитик)

Как держать `docs/user/` + `docs/developer/` в соответствии с продом. Зона аналитика (см. [[analyst-maintains-user-dev-docs]]).

---

## 0. Принцип

**Мануал = зеркало кода, не зеркало решений.** Документируем то, что РЕАЛИЗОВАНО и в проде, а не то, что отвечено/запланировано. Решения живут в `docs/analysis/`; мануал — в `docs/user/` + `docs/developer/`.

Главный риск — **противоречие код↔мануал** (пользователь сделает не то). Это опаснее пробела. Ловить и править первыми.

---

## 1. Когда обновлять

При каждом значимом `[report]` разработчика в SIGNALS:
1. Прочитать что изменилось (файлы из отчёта).
2. Найти соответствующий раздел мануала.
3. Дописать/исправить дельту.
4. Сверить с кодом (не с отчётом — отчёт может опережать).

## 3. Чек-лист сверки (перед «готово»)

1. **Клавиши** — сверить таблицу с `grid/keymap.ts` (SSOT). Дрейф = баг.
2. **Форматы ввода** — сверить с `format.ts parseHours` (запятая/точка/HH:MM/1ч30м).
3. **onDelete/поля** — сверить с `objects/*.object.ts` (RESTRICT vs CASCADE, nullable).
4. **Статусы безопасности** — сверить с `docs/security/RISK_REGISTER.md` (CISO-XXX OPEN/RESOLVED/CLOSED).
5. **ПДн/ФИО** — сверить с `revealEmployeeNames` (не писать «скрыто навсегда» — это настройка).
6. **Формулы** — сверить с кодом (утилизация = client/fact, НЕ /норма; норма = календарь SSOT; coverage = план−распланировано ≠ budgetRemaining = план−факт).
7. **Версия** — README ссылается на актуальный RELEASE_NOTES + тег.

## 4. Частые ловушки (были реально)

- **«запятая не поддерживается»** — код принимает запятую. Не утверждать обратное.
- **«Shift+Enter = заполнить»** — нет, Shift+Enter = вверх; заполнить = Alt+→.
- **«наведите мышь для контролов»** — нет, меню ⋯ всегда видимо (hover убран, RemDOM).
- **«ФИО скрыты до RBAC»** — нет, управляется настройкой revealEmployeeNames.
- **«Прошлые периоды без ограничений»** — нет, есть lockdown по дате.
- **Утилизация «от нормы»** — нет, от ФАКТА (client/fact).
- **CASCADE удаление проекта/сотрудника** — нет, RESTRICT (архивируют).

---

## 5. Стиль

- По-русски, инфостиль (конкретика, глаголы, без воды).
- Таблицы для параметров/клавиш/полей.
- `> Заметка/⚠️ Важно` для предупреждений.
- Термины едины: «Согласовано/На согласовании/Черновик/Отклонено»; «распланировано» ≠ «освоено»; «бюджет (plannedEffort)».
- Скриншоты — плейсхолдером `> 📸 [Скриншот: …]` (заполнит UI-роль).

---

## 6. Процесс «doc-recheck»

После правок — пинг в SIGNALS @Dev1/@Dev2/@arch перепроверить свою зону (фронт→user, бэк→developer). Найденные рассинхроны — править за итерацию. Код-gaps (решено, не реализовано) → роутить в бэклог, НЕ документировать как готовое.
