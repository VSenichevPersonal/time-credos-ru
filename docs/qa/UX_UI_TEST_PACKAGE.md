## Чек-лист ручного смоука (не-юнит-тестируемый UX)

Браузер: chrome-devtools (snapshot-first). Среда: dev-workspace «Трудозатраты».
Прогон → `[smoke-ok]` или `[bug] #N` с repro. Легенда: ⬜ не прогнан · ✅ ok · ❌ bug · ⚠️ замечание.

### S1. Редакция ФИО в UI (визуальная проверка пункта 1)
- ⬜ `/s/reports`, тоггл `revealEmployeeNames=false` (дефолт): в срезе «По сотрудникам», OLAP-оси employee, крошках и пилюлях фильтра — виден **читаемый КОД** (`Сотрудник·<отдел>·XXXX`), **НЕ** сырой UUID, **НЕ** пустая ячейка.
- ⬜ Тоггл `revealEmployeeNames=true` (админ): те же места показывают **ФИО**.
- ⬜ CSV-экспорт detail при reveal=false: колонка «Сотрудник» = КОД, не UUID/не пусто.
- ⬜ Деактивированный сотрудник с записями в периоде: ярлык — КОД, не сырой UUID (регресс старого бага утечки UUID).

### S2. Валидация ввода в гриде (визуальная проверка пункта 2)
- ⬜ Ввод >24 ч/день: ячейка/поле подсвечено как ERROR, сохранение блокируется (не уходит в API).
- ⬜ Ввод между порогом переработки и лимитом (напр. 13 ч при пороге 12): WARNING-индикатор, но сохранить **можно**.
- ⬜ Недобор недельной нормы (если порог включён): мягкое предупреждение, не блок.

### S3. Lock APPROVED-ячейки (визуальная проверка пункта 3)
- ⬜ Ячейка периода в статусе APPROVED: визуально read-only (приглушена/замок), клик не открывает редактор.
- ⬜ Попытка bulk-fill по строке с APPROVED-днями: согласованные дни не перезаписываются (no-op), остальные — заполняются.
- ⬜ Копирование прошлой недели не перетирает APPROVED-ячейки текущей.

### S4. Resource Gap — ячейка капасити (визуальная проверка пункта 4)
- ⬜ Капасити-доска, метрика «gap»: дефицит со знаком `+`, профицит со знаком `−`, баланс (|gap|≤5%) — нейтральный фон.
- ⬜ Цвета: дефицит 5–15% янтарь, >15% терракот; профицит синий (сильнее при >15%).
- ⬜ Иконка-дублёр цвета (для доступности): баланс ●, дефицит ▲, профицит ▼ — видна и читается.
- ⬜ Ячейка без ёмкости (capacity=0): пустая, без процентов/знака.

### S5. CSV-экспорт открывается в Excel/1С (визуальная проверка пункта 5)
- ⬜ Кнопка «Экспорт CSV» в отчётах скачивает файл; имя/расширение корректны.
- ⬜ Открытие в RU-Excel/1С: кириллица не «кракозябры» (BOM), разделитель `;` разносит по колонкам, значения с `;`/кавычками не ломают строки.

### S6. OLAP drill-down (визуальная проверка пункта 6)
- ⬜ Клик по строке агрегата → drill в дочерний срез; breadcrumbs наращиваются и кликабельны назад.
- ⬜ Сумма дочерних визуально сходится с родительской строкой; подписи осей (dimLabel) читаемы, без сырых UUID.

### S7. Booking в капасити (визуальная проверка пункта 7)
- ⬜ HARD-бронь: уменьшает «свободно», входит в load/ratio/gap (потребляет ёмкость).
- ⬜ SOFT-бронь с включённым тогглом `tentativeBookingEnabled`: рисуется отдельным приглушённым слоем, **не** уменьшает свободную ёмкость; тоггл выкл → SOFT скрыт.
- ⬜ Овербукинг (план+HARD>capacity): терракот-обводка ячейки + знак ▲, без смены заливки.

### S8. Missing-timesheets (визуальная проверка пункта 8)
- ⬜ Отчёт/виджет «не сдали табель»: список адресный (сотрудник+период), при reveal=false — КОД, не UUID.
- ⬜ Сотрудник, закрывший норму, в списке отсутствует; не сдавший — присутствует.
