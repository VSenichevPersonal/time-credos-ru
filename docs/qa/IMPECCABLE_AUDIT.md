# IMPECCABLE AUDIT — модуль time.credos.ru

Аудит UX/UI всех front-components по критериям impeccable (register=**product**: инструмент, плотная таблица, спокойная палитра). READ-ONLY ревью, без правок кода. Параллельно работают dev-агенты.

- **Дата:** 2026-06-21
- **Скоуп:** `apps/time/src/front-components/` — grid (таймшит), capacity (доска), reports, my-time, settings, project-team/summary/budget, shared.
- **Метод:** чтение кода компонентов + сверка с product.md / audit.md impeccable + анти-паттерны. Браузерный прогон не делался (sandbox front-component, host-DOM недоступен) — оценка статическая по коду.

## Что уже хорошо — НЕ трогать

- **Токен-дисциплина:** единый SSOT (`shared/tokens.ts`), нейтрали тинтованы в холодный hue, нет `#fff`/`#000` в компонентах, акцент ≤10% площади. В компонентах найдено всего 3 захардкоженных hex (border-тинты тостов) — это допустимо.
- **Статус не только цветом (a11y):** gap ▲▼●, замок read-only ячейки, точка save-индикатора, ведущие иконки в тостах/баннерах. Образцовое дублирование семантики.
- **Никаких анти-паттернов:** нет side-stripe (явно отмечено в комментариях как бан), нет gradient-text, нет hero-metric плашек, нет glassmorphism, нет modal-first — всё инлайн (approval-bar, cheatsheet, filter-chip, экспорт-поповер).
- **tabular-nums** на всех числовых данных (часы, проценты, gap, бюджет) — таблицы не «прыгают».
- **Пустые/ошибка/загрузка состояния:** обучающие пустые состояния (breakdown-table «Смените период или срез», Unlinked-профиль с действием), ErrorState с retry + «Подробнее», ErrorBoundary на каждой вкладке. Не «nothing here».
- **Клавиатура и focus-ring:** Segmented/breadcrumbs/filter-pills/NavBtn имеют hover+focus состояния с видимым `box-shadow` focus-ring через `T.accentRing`. Автокомплит полностью клавиатурный (↑↓ Enter Tab Esc).
- **aria-live дисциплина:** assertive для ошибок валидации, polite для предупреждений/save-статуса.
- **UX-копирайтинг:** ясный, без воды, с подсказками-hint снимающими неоднозначность. Напоминания/баннеры дают конкретное действие («Внесите трудозатраты на странице Записи»).
- **Motion:** переходы только на color/background/box-shadow (не на layout-свойствах — правило impeccable соблюдено), длительность 120–160ms (в product-норме 150–250ms).

## Систематические паттерны (положительные)

- Дизайн-решения документированы в комментариях со ссылками на impeccable, Timetta, Kimai и внутренние REQ/DP/ADR. Это снижает риск дрейфа при правках dev-агентами.
- Сверка с референсами (Timetta resource-gap нормы ±5/15%, Kimai duplicate-row, graceful-error) выполнена — идеи заказчика проверены по легальным референсам.

## Систематические замечания (к улучшению)

- **Плотность тулбаров:** и grid/toolbar, и capacity/board-toolbar склонны к перегрузу контролами в одну строку (6–8 элементов). Это рекуррентно. Решение — overflow-меню для второстепенных действий.
- **title-as-tooltip зависимость:** 76 `title=` против 24 `aria-label`. Многие подсказки/полные имена живут только в `title`, недоступном на touch/клавиатуре. Системный пробел recognition-доступности.
- **ARIA grid-семантика отсутствует** в таблицах-сетках (таймшит, доска) — данные верстаются на div+grid без role=grid/row/cell. Самый весомый a11y-долг.
