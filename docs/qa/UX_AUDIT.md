## 3. Консистентность терминов с глоссарием

Сверка с `L10N_GLOSSARY.md` / `GLOSSARY.md` / `labels.ts` — **расхождений нет.** Термины используются верно:

- «Вид работ» / «Виды работ» — везде (objects, view, nav, grid, filters). НЕ «Активность» / «Activity». OK.
- «Запись трудозатрат» / «Записи трудозатрат» — objects, view. НЕ «таймшит-запись», НЕ «запись времени». OK.
- «Проект», «Отдел», «Этап», «Связь с 1С», «Производственный календарь» — соответствуют. OK.
- Статусы «Черновик / На согласовании / Согласовано / Отклонено» — `labels.ts` ENTRY_STATUS_LABELS, совпадают с глоссарием. OK.
- Категории работ и группы видов работ — `labels.ts`, совпадают. OK.
- «Работник» (`ttEmployee`) — в objects labelSingular «Работник», в nav-item «Сотрудники», во view «Все работники». Глоссарий допускает «Работник»/«Сотрудник» как синонимы для `ttEmployee`; формально консистентно, но nav «Сотрудники» ↔ объект «Работники» — лёгкая разнобойность. **P3-косметика**, на усмотрение arch (не нарушает глоссарий).

Запрещённые UI-термины («активность» для видов работ, «запись времени», «coming soon») в видимых строках **не найдены** (вхождения «активн» — только в комментариях кода про «активную ячейку»).
