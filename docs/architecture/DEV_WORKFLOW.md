# Модель разработки и установки SDK-app

**Дата:** 2026-06-20
**Статус:** принятая модель (см. ADR-0002)
**Кратко:** SDK-пакет разрабатываем В ЭТОМ репозитории. В Twenty (форк CredosCRM1) его УСТАНАВЛИВАЕМ, а не мерджим исходники.

---

## Что где

| | Этот репо (`time-credos-ru`) | Форк CredosCRM1 (workspace) |
|---|---|---|
| **Что** | SDK-пакет (app-приложение) | работающий Twenty 2.x |
| **Делаем** | пишем код: `defineObject`, `defineFrontComponent` (сетка/таймер), `defineLogicFunction`, `defineNavigationMenuItem`, `defineView`, `definePageLayout`, `defineRole` | принимает установку app |
| **Артефакт** | `npx create-twenty-app` → проект → build → пакет/tarball | `application-install` ставит пакет в workspace |
| **Git** | свой, независимый | свой, не трогаем под app |

## Поток

```
1. Тут:   npx create-twenty-app  → пишем app → build
2. Связь: собранный пакет (tarball / registration / marketplace)
3. Туда:  install в workspace CredosCRM1
4. Итог:  app живёт в Twenty (объекты, сетка, раздел сайдбара), но КОД — здесь
```

```
time-credos-ru (этот репо)            CredosCRM1 (форк, 2.x GA)
┌──────────────────────────┐          ┌──────────────────────────┐
│ SDK-app (свой пакет)      │──build──▶│ Twenty ядро + старые     │
│ defineObject/View/Front-  │ tarball  │ credos-модули (как есть)  │
│ Component/Nav/Logic/Role  │─install─▶│ workspace ← app живёт тут │
│ (разработка здесь)        │          │ ядро НЕ трогаем           │
└──────────────────────────┘          └──────────────────────────┘
   разрабатываем здесь                   устанавливаем туда
```

## Важное «да / нет»

- ✅ **Да:** здесь разрабатываем именно SDK-пакет.
- ✅ **Да:** в workspace CredosCRM1 его **устанавливаем**, не копируем исходники.
- ❌ **Не** «мердж в репо форка» — никакого слияния кода. Install = деплой собранного app, как плагин.
- 🔁 Обновил app здесь → пересобрал → переустановил/обновил версию в workspace. Форк при этом не меняется.

## Аналогия

Как браузерное расширение: код расширения — в своём репо; в браузер его **устанавливаешь**, а не вшиваешь в исходники браузера. Здесь так же: app = «расширение», Twenty = «браузер».

## Что всё-таки может попасть в форк (отдельно, минимум)

- Только если понадобится **СБИС-style продукт-селектор** (полная смена сайдбара) — отдельный маленький core-PR в форк (жёлтая зона, маркеры + `core-changes.md`). Сам app этого НЕ требует: раздел-папка «Время» ставится самим app через `defineNavigationMenuItem`.

## Предусловие (блокер всего пути)

Чтобы install реально работал — workspace должен быть на **Twenty 2.x GA** (рабочий app-runtime).
Форк CredosCRM1 создан 14 марта 2026 — за ~5 недель до 2.0 GA (21 апреля 2026); SDK-файлы в нём — пред-GA снимок.
**Первый шаг перед кодом app:** оценить/сделать upstream-sync форка до 2.x GA. Без этого ставить app некуда.
Подробности и fallback — ADR-0002.
