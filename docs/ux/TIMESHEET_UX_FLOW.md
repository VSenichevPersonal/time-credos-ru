# Timesheet UX Flow (референс)

**Источник:** документация + UI-скриншоты
**Дата:** 2026-06-21

---

## Заполнение таймшита (пошагово)

### Экран
- Верх: период (08.04–14.04), кнопки ← →
- Сетка: строки = проекты/работы, столбцы = дни недели
- Справа: итоги за день
- Снизу: итоги за неделю (40ч план, 40ч факт)
- Панель: «Добавить строку», «Копировать», «Вставить»

### Навигация
- Enter → следующая ячейка (вниз, тот же день)
- Tab → следующая ячейка (вправо, следующий день)
- Клик → прямой выбор ячейки
- Популярные значения (8, 4, 0, 0.5) — кнопки при клике на ячейку

### Быстрый ввод
- **Time Tracker:** чёрный треугольник в строке → запуск счётчика
- **Кнопки-пресеты:** 8, 4, 0, 0.5 (настраиваются в шаблоне)
- **Копирование прошлой недели:** кнопка «Копировать» → «Из предыдущего периода»

### Валидация при отправке
- **Блокировка** (ошибка) — нельзя отправить пока не исправлено
- **Предупреждение** — можно отправить, игнорируя
- Настраивается в шаблоне таймшита → Правила валидации
- Пустые дни: НЕ блокируют (если нет правила)

---

## Экран руководителя

### Список таймшитов команды
- Путь: Команда → Все таймшиты
- Колонки: Сотрудник, Период, Статус (цвет), Часы, Дата отправки
- Фильтры: по сотруднику, статусу, периоду, проекту, отделу

### Статусы (цвета)
- ⚪ Черновик (серый)
- 🟡 На согласовании (жёлтый)
- 🟢 Согласован (зелёный)
- 🔴 Просрочен (красный)

### Действия руководителя
- Открыть таймшит
- Согласовать (одобрить)
- Отклонить с комментарием
- Просмотр просроченных: отдельный фильтр

---

## Что берём для своего UI

1. **Сетка дней×строк** — основной элемент
2. **Time Tracker** (счётчик) — для быстрого ввода
3. **Кнопки-пресеты** (8, 4, 0.5) — ускоряют заполнение
4. **Копирование прошлой недели** — критично для UX
5. **Enter/Tab навигация** — без мыши
6. **Цветные статусы** (4 цвета)
7. **Фильтр просроченных** для руководителя
8. **Отклонение с комментарием** — обязательное поле
