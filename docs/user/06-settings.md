# Настройки модуля (для администратора)

Экран **«Настройки»** (singleton — одни параметры на всё приложение). Изменения сохраняются автоматически (без кнопки «Сохранить»).

---

## Общие параметры

| Параметр | Что задаёт |
|---|---|
| **Норма часов в день** (`normHoursPerDay`) | Резервное значение нормы, если производственный календарь недоступен. При загруженном календаре норма берётся из него (короткий день/праздник). |
| **Часы шаблона заполнения** (`fillTemplateHours`) | Значение по умолчанию для «Заполнить будни». |
| **Коэффициент ёмкости** (`defaultCapacityFactor`, дефолт 0.8) | Доля рабочего времени, доступная под проекты (накладные/совещания вычтены). Применяется к ёмкости отдела; переопределяется per-отдел. |
| **Горизонт планирования** (`planningHorizonWeeks`) | Сколько недель показывает доска загрузки. |
| **Порог переработки** (`overtimeWarnHours`) | Часы/день, выше которых — предупреждение (не блок). |
| **Макс. часов в день** (`maxHoursPerDay`, дефолт 24) | Жёсткий лимит — ввод выше блокируется. |
| **Период согласования** (`approvalPeriod`) | Неделя / месяц. |
| **Напоминания** (`reminderEnabled`, `reminderDayOfWeek`) | Авто-напоминание о незаполненном таймшите. |

---

## Закрытие периодов (lockdown)

Защита прошлых периодов от любых правок (независимо от согласования):
- **Дата закрытия** (`lockdownDate`) — записи с датой ≤ границы становятся read-only (🔒). Пусто = выключено.
- **Грейс-окно** (`lockdownGraceDays`) — сколько дней после конца периода ещё можно дозаполнять. Эффективная граница = `lockdownDate − грейс`.
- Обойти закрытие может уполномоченная роль (руководитель/админ) — правка логируется как override.

> Подробнее о read-only ячейках — [Согласование](03-approval.md#закрытие-прошлых-периодов-lockdown).

## Безопасность / ПДн

| Параметр | Что задаёт |
|---|---|
| **Показывать ФИО** (`revealEmployeeNames`) | Вкл → отчёты/доска/экспорт показывают ФИО сотрудников. Выкл → стабильный код (`Сотрудник·XXXX`) без персональных данных. Решение администратора по 152-ФЗ. |

> ⚠️ Включение раскрывает персональные данные во всех отчётах и экспортах. Применяйте с учётом требований 152-ФЗ.
