# Планирование и доска загрузки (Capacity Board)

Раздел для руководителей и РП: как планировать трудозатраты по проектам, отделам и сотрудникам, отслеживать загрузку и видеть «сколько уже распланировано».

---

## Доска загрузки (Capacity Board)

Доска показывает загрузку по периодам (недели/месяцы) в срезах:
- **Отделы** — загрузка подразделений.
- **Люди** — загрузка сотрудников (раскрытие отдела → сотрудники).
- Раскрытие отдела → проекты → доли по отделам.

**Метрики ячейки** (переключатель): Свободно, ч · Загрузка, % · План, ч · **Баланс, ч** (спрос − ёмкость, ▲ дефицит / ▼ профицит).

**Цветовая шкала (heatmap)** — занятость по порогам: свободно → оптимум → полная → перегруз. Перегруз (load > ёмкости) подсвечивается обводкой + ▲, при наведении — «перегруз: N ч > ёмкость M ч». Овербукинг **не блокируется** — только предупреждение.

**Ёмкость** = рабочие часы периода (по производственному календарю) × численность × коэффициент ёмкости − отсутствия. Выходные/праздники = 0.

## Планирование проекта

Кнопка **«✎ Планировать»** (только руководитель) на строке проекта открывает панель плана:

### Способ распределения
- **Равномерно** (по умолчанию) — плановые часы (`plannedEffort`) раскидываются равномерно по **рабочим** дням диапазона (выходные/праздники получают 0). Σ всегда = заданному объёму.
- **Вручную по месяцам** — таблица «месяц → часы», ручной раскид. Σ слотов сверяется с объёмом проекта.

### Период
- **С** (дата начала, редактируется) и **ПО** (дата завершения). Пустая «ПО» = открытый план до конца горизонта доски.

### Живое превью
- Мини-бар по периодам показывает раскид **до сохранения** (считается на лету по рабочим дням — то же, что увидите на доске).
- Строка-сверка **«Σ раскид = N ч · план M ч ✓/≠»**.
- Перегруз периода подсвечивается (план > ёмкости отдела) — мягкое предупреждение, сохранение не блокируется.

## Сколько распланировано (coverage)

Не путать три величины:
- **План (бюджет)** — `plannedEffort`, плановая трудоёмкость проекта.
- **Распланировано** — сколько часов фактически разнесено по периодам/людям (Σ слотов / равномерный раскид).
- **Факт** — сколько списано (time-entry).

Индикатор **«Распланировано N / План M · остаток»** показывает, всё ли разнесено:
- остаток = 0 → ✓ распланировано полностью;
- остаток > 0 → «N ч не распределено» (есть свободный бюджет для плана);
- остаток < 0 → переаллокация (запланировано больше бюджета).

> «Освоено (факт)» — отдельная величина (план − факт), не путать с «распланировано» (план − разнесённое).

## План по сотруднику

План задаётся **снизу-вверх**: на конкретного человека, отдел = сумма.

- При раскрытии отдела на доске — строки сотрудников с их плановыми часами по периодам.
- **Личная загрузка** сотрудника = Σ его персональных слотов по всем проектам + доля нераспределённого остатка отдела (по доле ставки FTE: полуставочник получает меньшую долю).
- Σ загрузок сотрудников отдела = загрузка отдела (инвариант консистентности).
- Если на сотрудника есть И персональный план, И бронь на тот же проект/период — в загрузку идёт **бóльшая** из двух (не суммируются дважды).

## Плановые загрузки отдела (без проекта)

Отдельная сущность **«Плановые загрузки»** — плановая занятость отдела, **НЕ привязанная к проекту** (резерв, внутренние работы, пресейл на уровне отдела). Это **не** детализация плана проекта — это самостоятельная ось спроса, которая суммируется к загрузке отдела наравне с проектами.

- Где: реестр «Плановые загрузки» (сайдбар) + правка из доски той же кнопкой «Планировать» на строке плана отдела.
- Поля: отдел · часы · период · категория.
- Отличие от плана проекта: план проекта = часы под конкретный проект; плановая загрузка = часы отдела без проекта.

> Раскид плана **по месяцам** (детализация конкретного проекта) правится через панель «Планировать» проекта — отдельного сырого реестра для слотов нет.

## Когда отдел освободится

На строке отдела показывается **«Свободен с …»** — первый период, где загрузка опускается ниже порога. Помогает ответить продажам «когда можно обещать старт нового проекта».

---

## Бронирование ресурсов (Booking)

Бронь — резерв сотрудника под проект или клиента (пресейл), отдельная ось от плана-оценки:
- **HARD** — подтверждённая, потребляет ёмкость (входит в Demand).
- **SOFT** — предварительная (пресейл), не потребляет ёмкость, показывается пунктиром (при включённой настройке предварительных броней).
- Бронь под **клиента без проекта** = пресейл-резерв.
- Овербукинг (бронь сверх ёмкости) — предупреждение, не блок.
