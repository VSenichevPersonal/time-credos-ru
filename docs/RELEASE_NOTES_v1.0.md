# Credos Time — Release v1.0.0

**Дата:** 2026-06-22
**Прод:** twenty-production-e5c5.up.railway.app (app «Credos Time»)
**Платформа:** Twenty CRM 2.14 SDK-app (`apps/time/`)
**Статус:** ядро PSA полностью в продакшене · 2599 unit-тестов · 0 failed

Внутренняя система учёта рабочего времени и ресурсного планирования для Кредо-С (~72 сотрудника, 5 производственных отделов: ОПИБ, ОИБ, ТЦ, ОВ, ОПР).

---

## Модули в продакшене

### Учёт времени (таймшит)
- Три режима ввода: **День** · **Неделя** · **Проект**
- Excel-стиль ввод часов (клик = правка, печать цифры = замена), форматы `8` / `1.5` / `1:30` / `1ч30м`
- Контролы строки в меню ⋯, копирование недели, «заполнить по норме»
- Валидация (warn/error), переработка, теги, индикатор комментария
- Легенда сигналов сетки (🔒 согласовано, переработка, норма дня)
- Индикатор **«чей таймшит»** (ФИО · отдел) — всегда виден

### Планирование ресурсов
- Доска капасити: срезы **Отделы / Люди**, метрики Свободно / Загрузка % / План / Баланс
- **Планирование по месяцам** (вручную) и **по сотруднику** (bottom-up, rollup по FTE)
- Консистентность отдел↔человек (отдел = контейнер, человек = детализация)
- **Бюджет проекта неизменен** + индикатор «распланировано X из Y, остаток» + предупреждение перерасхода
- Booking (бронь ресурса) soft/hard, Resource Gap, heatmap баланса ±10/20%
- Faded-индикация чужих слотов, tooltip перегруза

### Согласование (approval)
- Полная петля: submit · approve · reject (+причина) · recall · revoke
- Разделение обязанностей (SoD): согласующий ≠ владелец таймшита
- Защита от гонок (compare-and-set), пакетные операции с обработкой ошибок
- Аудит: кто согласовал / отклонил / отозвал

### Руководитель за команду (on-behalf)
- Руководитель отдела / руководитель проекта / админ **вносят и отправляют** трудозатраты за сотрудника (отсутствие, исправление)
- Серверная авторизация `canWriteFor`, селектор «Таймшит сотрудника», пометка «внёс руководитель»
- Сотрудник видит и правит внесённое за него (до согласования)

### Отчёты
- Сводка · Тренд · OLAP-drill (крошки + пилюли + cross-filter) · KPI по срезу
- **Проекты: бюджет / распланировано / факт** (3 раздельные величины + остаток + перерасход)
- Незаполненные таймшиты · Табель Т-13 · CSV-экспорт (RU-локаль, запятая)
- ПДн: ФИО при доверенном доступе, иначе стабильный КОД (152-ФЗ)

### Логи и защита периодов
- **Audit-log изменений** трудозатрат: кто / когда / что менял (часы, статус, удаление) — server-truth актор
- **Lockdown прошлых периодов:** дата закрытия + грейс, защита от правок задним числом, reopen только руководителем (логируется)
- Целостность: уникальность-дедуп, нормализация даты, гард пустых слотов, защита согласованных от каскадного удаления

### Безопасность (ИБ)
- **CISO-005:** серверный actor (resolveActor + userWorkspaceRef-мост, TOFU) — закрытие IDOR согласования
- onDelete RESTRICT (защита согласованных часов от потери)
- ПДн-минимизация (152-ФЗ), no-billable (без ставок/биллинга)

### Прочее
- Личный кабинет (мои часы / периоды / причина отклонения)
- Структура отделов (руководитель + иерархия), справочники, настройки (singleton)
- Производственный календарь (норма дня — единый источник: сетка = дашборд = сервер)

---

## Принятые продуктовые решения
1. **plannedEffort = бюджет-оценка** проекта, неизменен без явной корректировки
2. **Lockdown** прошлых периодов + reopen администратором
3. **On-behalf:** руководитель/PM/админ вносят и отправляют за сотрудника
4. **Финансы (себестоимость)** — отложено, пока только учёт часов
5. Без биллируемости (no-billable) — только cost-rate в будущем
