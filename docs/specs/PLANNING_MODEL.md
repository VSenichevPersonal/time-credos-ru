## Booking: Soft vs Hard

- Soft (tentative) — пресейл, неподтверждённая бронь
- Hard (confirmed) — подтверждённая аллокация
- Разные типы в BookingType enum (soft/hard)
- В UI: разные цвета/стили

## Аллокация

- Единица: **часы** (не проценты)
- человек × проект × период
- Два режима: базовый (плашки календаря) / детальный (таблица)
- Перегрузка (>100%): **НЕ блокируется**, показывается конфликт
- Цветовая индикация: красный при превышении

## Ёмкость

- Формула: FTE × Расписание × Календарь
- Отпуска/праздники уменьшают доступные часы
- Частичная занятость через schedule (не 40ч/нед)
- Индивидуальные графики работы

## Доступность вперёд

- На основе: брони + план + отпуска
- Горизонт: настраиваемый (недели/месяцы)
- Отображение: «Сводка по сотрудникам» (capacity heatmap)

## Мульти-отдел проекта

- Проект может использовать ресурсы нескольких отделов
- Часы плана распределяются по отделам

## План vs Факт

- Отчёт «Ресурсный план»: Plan / Estimate / Actual рядом
- Алерты: через цветовую индикацию перегрузки
- Дисперсия: отклонение план/факт в hours

## Скиллы и подбор

- Поиск сотрудника по компетенциям (навыки)
- Привязка: User → UserSkill → Skill + Role

## What-if сценарии

- Soft booking для неподтверждённых сделок
- Отдельные виды бронирований
- Переход soft→hard при выигрыше сделки

## Связка с CRM

- Opportunity → Project → Booking → TimeEntry
- Конвертация сделки в проект: НЕ автоматическая
- Прогноз загрузки от вероятности сделки: через soft booking
