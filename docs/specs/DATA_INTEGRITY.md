## Целостность

### Дубликаты
- ❌ НЕТ уникального ключа на проект×вид×день
- Разрешены несколько строк с одинаковыми параметрами
- Валидация через бизнес-правила при отправке

### rowVersion
- ✅ Оптимистичная блокировка (rowVersion: Int64)
- На TimeSheetLine, TimeAllocation, Project, Act

### Lock периода
- Согласованный таймшит: ЗАБЛОКИРОВАН от редактирования
- После отклонения: авто DRAFT, можно править
- Изменение согласованного: только через спец-процедуру

### Валидация часов
- Max/день: настраивается правилами валидации
- Отрицательные: запрещены (Decimal ≥ 0)
- Выходные/праздники: правило «Рабочий день до 24ч»
- День ±2 часа от расписания (предупреждение)

### Каскады удаления
- ❌ НЕТ каскадного удаления
- Блокировка удаления при наличии связанных данных
- Soft-delete: isActive = false

### Audit Trail
- ✅ Полноценный аудит (HistoryLogEntry, LoginLogEntry)
- Фиксируется: кто, когда, что изменил
- Журнал изменений + Журнал входов

## Связанность

### Ставка — где хранится
- НЕ на User напрямую
- Каскад: RateMatrix → User base rate → Project default
- Cost-rate и bill-rate: раздельные матрицы
- Резолв: best-match по аналитикам

### Календарь/Норма
- Индивидуальные графики (UserSchedule)
- Привязка: employee → schedule → weekly hours
- Праздники: ScheduleException
- Норма: часы/день из расписания

### Цепочка часы→акт→оплата
- Связи: ВЫЧИСЛЯЕМЫЕ (НЕ прямые FK!)
- TimeSheet → ActOfAcceptance: через ProjectTask
- Act → Invoice: ручное создание
- Invoice → Payment: через состояние (Paid)
- Триггеры: Lifecycle + Workflow (событийная модель)

### Opportunity → Project → Booking → Entry
- ❌ НЕТ авто-конвертации сделки в проект
- ❌ НЕТ прямых FK от пресейла до факта
- Связи: логические/ручные/воркфлоу
- Soft booking = пресейл-бронь
