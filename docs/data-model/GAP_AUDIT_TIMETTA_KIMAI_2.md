## Новые наблюдения аудита (чего нет даже в плане)

1. **Timetta «копирование таймшита целиком»** — у нас есть «копир. неделю» в сетке ✅, паритет.
2. **Kimai «активность по умолчанию» (default activity)** — можно ускорить ввод: запоминать последний вид работ на проекте. 🟢 микро-UX, в волну-3.
3. **Timetta «блокировка периода»** (lock закрытых месяцев — нельзя править после согласования периода) — у нас lock на уровне записи (статус), но не на уровне ПЕРИОДА. 🟡 для дисциплины учёта — в волну-4.
4. **Kimai «массовое редактирование»** выделенных записей (batch edit статуса/проекта) — у нас bulk-fill часов есть, batch-смены проекта/статуса нет. 🟡 волна-4.
5. **Оба: фильтр по статусу** записей в сетке (показать только черновики/на согласовании) — сейчас фильтры по проект/отдел/вид/категория, по статусу нет. 🟡 быстрый, в волну-3.

## Принцип приоритизации (без изменений)
Сначала (1) закрыть начатое (R3-viz, F-D интеграция), (2) ценность руковод. (настройки, аллокации), (3) удобство ввода (дубль/фильтры/теги/статус-фильтр), (4) дисциплина (lock/RBAC), (5) тяжёлое (календарь/P&L). Каждая UX-фича сверяется с референсами — не делаем хуже Timetta/Kimai.
