## Что УЖЕ есть (паритет с референсами) ✅
- Недельная сетка (Timetta TimeSheet→Line→Allocation), Decimal-часы.
- 3 режима ввода День/Неделя/Проект (Kimai мультирежим).
- Клавиатура-first, мультифильтры, **копировать неделю** (Timetta «копир. из предыдущего»), bulk-fill, недавние проекты (Kimai Recent), cheatsheet.
- План/недобор из **производственного календаря РФ** (Timetta schedule).
- Согласование Черновик→На согласовании→Согласовано/Отклонено + guards (Timetta lifecycle).
- Отчёты `/s/reports`: утилизация, загрузка/недогруз по отдел/проект/человек (Kimai Reporting).
- Карточки: проект (7 вкладок), запись. Капасити-доска «когда свободен».

## Коррекция плана (волны)

**Волна-2 (текущая, аналитика) — расширить:** дашборд «Отчёты» (UI-F) + «по людям» (UI-G) + **закрыть заглушки карточки проекта** F-A «Бюджет» (план vs факт) и F-B «Команда» (часы по сотрудникам из /s/reports). Дашборд и бюджет используют один источник `/s/reports`.

**Волна-3 (удобство ввода):** UI-A дубль строки + UI-B сохранённые фильтры + F-C теги + UI-D цвет-кодинг.

**Волна-4 (полнота):** UI-C настраиваемые колонки, F-D отсутствия, F-E напоминания, F-F экспорт.

**Бэклог «Финансы»:** F-G P&L/ставки (REQ-0002, после связки 1С). UI-E календарь — по потребности.

## Принцип приоритизации
Сначала то, что (1) **закрывает уже видимые заглушки** (Бюджет/Команда карточки проекта) и (2) даёт **ценность руководителю** (дашборд, бюджеты, утилизация). Удобство ввода (дубль/фильтры/теги) — волна-3. Тяжёлое (календарь, P&L) — позже.
