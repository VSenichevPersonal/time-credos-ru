# Трейсабилити модели: каждый объект/поле ← источник

**Дата:** 2026-06-20
**Назначение:** проследить происхождение каждой сущности и решения от источников — Директум 5 (реальные данные), Kimai (логика), Timetta (амбиция), Каталог услуг (виды работ), Битрикс (оргструктура). Чтобы на каждом шаге было видно, что и откуда взято.

Легенда: **Д5** = Директум 5 · **K** = Kimai · **T** = Timetta · **КУ** = Каталог услуг · **БТ** = Битрикс.

---

## Объекты

| Объект `tt*` | Источник | Обоснование |
|---|---|---|
| **ttDepartment** | БТ + КУ | 5 отделов из Битрикса (ОВ/ОИБ/ОПИБ/ТЦ/ОПР); у Д5/K/T нет — наше измерение |
| **ttEmployee** | Д5 + БТ | работники Д5 (36) сверены с реестром Битрикс (11/11 ОВ); в Twenty нет встроенного → свой объект |
| **ttProject** | Д5 + K + T | «Проект» Д5 = Customer→**Project** (K); развит до карточки (T-амбиция) |
| **ttStage** | T | этапы проекта — из Timetta-глубины (Д5 плоский, без этапов) |
| **ttActivity** | Д5 + КУ | «Тип работ» Д5 (69) + услуги ИБ/ТЦ из каталога; = Activity (K) |
| **ttTimeEntry** | Д5 + K | строка Д5 = Timesheet-запись K (duration-mode); атом учёта |
| **ttBillingLink** | наше (под 1С) | M:N к документам 1С — связь учёта с финансами (нет в источниках, задел) |

## Ключевые поля

| Поле | Источник | Обоснование |
|---|---|---|
| `TimeEntry.hours` = NUMBER decimal | Д5 + T | реально дроби 0.5/0.75 (Д5); Decimal-часы (T-выбор), не INT-секунды (K) |
| `TimeEntry.description` (свободный текст) | Д5 | «Состав работ» = 23069 уник → не справочник |
| `Project.category` (WorkCategory) | наше + Д5 | билляность/утилизация; в Д5 неявно (внутрянка = своя орг, 53%) |
| `Activity.group` | наше + Д5/КУ | один справочник с группировкой для аналитики (вместо 2 уровней Service/Activity) |
| `Project.plannedHours`, `Department.capacityFactor` | наше + (T) | планирование загрузки; T-идея ресурсов, упрощено до уровня отдела |
| `*.approvalRequired` | T (lite) | согласование (T-lifecycle), упрощено до отключаемого enum; в Д5 не было |
| `Project → Company` | K + Twenty | Customer→Project (K); Company = стандартный объект Twenty (не дублируем) |
| статусы enum (Draft/Submitted/Approved) | T (lite) | T-lifecycle → простые enum на старте (риск state-machine отложен) |

## Методологические решения

| Решение | Источник | Документ |
|---|---|---|
| Плоское хранение TimeEntry (журнал) | Д5 + K | реально работает 11 лет (Д5); = entry-centric K |
| Недельная сетка = VIEW над записями | T | подача Timetta (контейнер TimeSheet→Line→Allocation) как представление, без тяжёлой таблицы |
| Approval — слой поверх, отключаемый | T lite | процесс Timetta, упрощён |
| Билляемость → утилизация (KPI) | наше + K | billable-флаг K + категория проекта |
| Иерархия Company→Project→Activity→Entry | K (= Д5 = план) | совпали все три |
| Kimai — переписываем, не копируем | K (AGPL) | ADR-0001 (вирусная лицензия) |
| Decimal vs INT, PostgreSQL | T | выбор из лучшего |

## По волнам (что и откуда на каждом шаге)
- **Wave 1 SETUP** — SSOT категорий/групп/статусов ← модель (Д5+K+T+КУ синтез).
- **Wave 2 DATA** — 7 объектов ← см. таблицы выше. Company-связь ← Twenty-стандарт.
- **Wave SEED** — реальные ОВ (Д5, янв-июнь 2026) + мок ИБ/ТЦ (КУ-виды работ, по образцу Д5) + сотрудники (БТ roster).
- **Wave GRID** — недельная сетка (T-подача) + таймер (K-стиль).
- **Wave CAPACITY** — планирование (наше, T-идея ресурсов упрощённая).
- **Wave APPROVAL** — согласование (T lite).
