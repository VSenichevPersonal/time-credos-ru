# UX-спека: ввод трудозатрат (timesheet) — топовый UX

**Дата:** 2026-06-20
**Регистр:** product (app UI, дизайн SERVES функции). Применены законы impeccable.
**Цель:** заполнять трудозатраты быстро и приятно, как в лучших таймтрекерах (Harvest/Toggl/Clockify уровня), клавиатура-first.

---

## Сцена (определяет тему)
Консультант/инженер Кредо-С в офисе днём заполняет часы за день/неделю по нескольким проектам, между задачами, хочет сделать это за 20 секунд. → **Светлая тема**, спокойные tinted-нейтрали, один акцент (синий Twenty) ≤10%, плотная но дышащая сетка. Не «тёмный дашборд ради вида».

## Цвет/типографика (Twenty-токены)
- Tinted нейтрали (не #fff/#000), акцент = Twenty blue только на: активная ячейка, today, кнопка действия, заполненный итог.
- Часы — табличные цифры (tabular-nums), моноширинное выравнивание по правому краю.
- Иерархия: проект (вес 600) > вид работ (вес 400, приглушён) > часы (вес 500 tabular).
- Сегодня — тонкая акцентная подложка колонки. Выходные — чуть приглушены.

---

## 3 режима (переключатель — сегмент-контрол вверху)

### 1. НЕДЕЛЯ (все проекты) — основной
Строки = (проект + вид работ), колонки = Пн–Вс + Итого. Текущая сетка, докрутить UX.

### 2. ДЕНЬ (все проекты)
Один день, список записей за день: проект · вид работ · часы · описание. Крупнее ячейка часов, видно описание. Быстрый ввод нескольких записей за день подряд.

### 3. ПРОЕКТ (один проект × неделя)
Выбран один проект (из фильтра/селектора), строки = виды работ этого проекта, колонки = дни недели. Для глубокого заполнения одного проекта.

Переключатель: `[ День ][ Неделя ][ Проект ]` сегмент, состояние в URL/локально. Период (‹ дата ›) общий.

---

## Фильтры (удобные, МНОЖЕСТВЕННЫЙ выбор)
Панель-строка над сеткой, компактные чипы-дропдауны с multi-select + поиск внутри:
- **Проект** (multi, поиск по 42 проектам, по коду/клиенту)
- **Отдел** (multi: ОВ/ОИБ/ОПИБ/ТЦ/ОПР)
- **Вид работ** (multi, поиск по 38)
- **Категория** (multi: На клиента/Пресейл/Пилот/Внутренний/Инфра/Обучение)
- **Сотрудник** (multi — только для роли «Руководитель»)
- Период — навигация ‹ неделя/день ›
Выбранное — чипами с ✕, «Сбросить всё». Фильтры влияют на строки сетки и доступные опции добавления. (Сохранённые фильтры — фаза 2.)

---

## Клавиатурный ввод (КЛЮЧ — «начинаешь вводить с клавиатуры»)
- **Стрелки** ↑↓←→ — перемещение по ячейкам сетки.
- **Начал печатать цифру** → ячейка сразу в режиме ввода (без клика, без Enter для входа). Поддержка `,`/`.`, шаг 0.25.
- **Enter** — подтвердить + вниз; **Tab** — подтвердить + вправо; **Shift+Tab** — влево; **Esc** — отмена.
- **0/пусто/Del** в ячейке — удалить запись.
- Быстрое добавление строки: фокус на «+ строка» → автокомплит проекта (печатаешь код/клиента) → Tab → автокомплит вида работ → строка добавлена, курсор в первой ячейке.
- Дубль строки/недели (копировать прошлую неделю) — фаза 2.
- Видимая подсказка горячих клавиш (компактно, не мешает).

## Ячейка часов (UX)
- decimal, tabular-nums, правое выравнивание; пусто = «·» приглушённо.
- Заполненная — лёгкая акцентная заливка пропорционально (8ч = насыщеннее). Не градиент-текст, не side-stripe.
- Валидация 0..24, >12 — мягкое предупреждение цветом.

## Итоги/план
- Итог строки (справа), итог дня (низ), итог недели.
- «План на неделю: 40 ч» → недобор/норма/переработка цветом (не три отдельных метрики-плашки — одна строка-индикатор).
- (Опц.) мини-распределение по категориям (утилизация) — компактный бар, фаза 2.

## Пустые состояния
- Нет строк: «Добавьте проект — начните печатать код или клиента» + фокус на добавление.
- Неделя без часов: спокойная подсказка, не пугающая.

---

## Анти-паттерны (бан impeccable — НЕ делать)
- НЕ модалка для ввода часов (инлайн в ячейке).
- НЕ side-stripe бордеры, НЕ градиент-текст, НЕ hero-метрики-плашки, НЕ одинаковые карточки.
- НЕ em-dash в текстах.
- Карточки — только где реально нужно (сетка — таблица, не карточки).

## Технически (SDK-песочница)
- `defineFrontComponent`, Web Worker, дизайн-токены Twenty, фикс-размер виджета (скролл только тело таблицы).
- Данные через RestApiClient (Core REST) — как в Wave 5; при включённых /s/ — переключить на route.
- Файлы <200 строк: режимы/фильтры/ячейка/хук недели — отдельные подкомпоненты.
- Русский UI (L10N_GLOSSARY).

## Нейминг проектов — консистентная схема (пере-сид)

**Код проекта: `[ОТДЕЛ]-[ГОД]-[NNN]`**
- ОТДЕЛ — владелец: `ОВ` / `ОИБ` / `ОПИБ` / `ТЦ` / `ОПР`
- ГОД — год старта (2026)
- NNN — порядковый в рамках отдел+год (001, 002…), с нулями (сортируемо)
- Пример: **`ОИБ-2026-007`**

**Отображение в UI (single source):** `ОИБ-2026-007 · ТАМАК · Пилот защиты АРМ`
(код · клиент · краткое название). Поле `name` строим так же.

**Реальный код клиента/Директум** (2021-0544, КР-2026-2007, 30332) → отдельное поле **`externalCode`** (для будущей сверки/1С), НЕ в основном коде. Не мешаем внутренний код с клиентским.

**Пере-сид:** перегенерить коды всех 42 проектов по схеме `[ОТДЕЛ]-[ГОД]-[NNN]` (нумерация сквозная по отделу), сохранив привязку к реальным клиентам и externalCode. Записи трудозатрат пере-привязать.

## Дополнительные UX-оптимизации (исследование + предложение)

Курировано из практики топ-таймтрекеров (Harvest/Toggl/Clockify/Timetta/Kimai). По фазам:

### Фаза 1 (в этот заход — высокая отдача, дёшево)
1. **Копировать прошлую неделю** — одна кнопка реплицирует строки прошлой недели (часы по желанию). Killer-feature скорости.
2. **Недавние/избранные проекты** — частые проекты вверху автокомплита добавления.
3. **Bulk-fill** — ввёл часы в одну ячейку → Shift+Enter применяет на все будни строки.
4. **Умные дефолты** — последний вид работ по проекту подставляется при добавлении.
5. **Индикатор незаполненных дней** — дни ниже нормы подсвечены, видно где «дыры».
6. **Живой итог vs норма** — пока печатаешь, итог дня/недели обновляется с цветом.
7. **Cheatsheet горячих клавиш** — по `?`, ненавязчиво.

### Фаза 2 (позже)
8. **Drag-to-fill** по дням (как в таблицах).
9. **Шаблоны недели** (типовые паттерны 8×5).
10. **Цвет-кодинг по категории/отделу** для быстрого скана.
11. **Сохранённые фильтры** (мои проekты, мой отдел).
12. **Отправка на согласование прямо из сетки** (когда approval вкл).
13. **Мобильный day-режим** (ввод с телефона).
14. **Таймер → ячейка дня** (интеграция Wave 6).
15. **Поиск по всем записям**.

Рекомендация: Фаза 1 — встроить в текущую переработку timesheet (особенно №1 «копировать неделю» и №3 bulk-fill — максимум скорости). Фаза 2 — по мере.
