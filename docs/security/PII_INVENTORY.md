# Карта ПДн (PII Inventory) — time.credos.ru

152-ФЗ ROPD-lite: что из персональных данных обрабатывает модуль, где хранится, кто видит. Поддерживает CISO (owner), обновляется при изменении схемы (`apps/time/src/objects/`, `fields/`).

## Категории субъектов

- **Сотрудники Кредо-С** (15–20 чел.) — единственная категория субъектов ПДн. Клиентов-физлиц нет (Company = юрлица).

## Реестр полей ПДн

| Объект | Поле | Класс | ПДн? | Назначение | Минимизация |
|---|---|---|---|---|---|
| `credosTimeEmployee` | `firstName` / `lastName` / `middleName` | PII | да | идентификация сотрудника | обязательно для учёта |
| `credosTimeEmployee` | `email` | PII | да | связь с workspaceMember, уведомления | можно заменить на `workspaceMemberRef` если дублирует |
| `credosTimeEmployee` | `jobTitle` | PII (косв.) | да | роль/должность | да |
| `credosTimeEmployee` | `workspaceMemberRef` | идентификатор | косвенно | связь с учёткой платформы | да |
| `credosTimeEmployee` | `department` (rel) | организац. | нет | принадлежность отделу | да |
| `credosTimeEntry` | `hours`, `date`, `project`, `status` | чувств. (HR/коммерч.) | нет (но трудозатраты конкретного лица) | учёт времени | привязка к employee → косвенно профилирует |
| `credosTimeEntry` | `approvedBy` | идентификатор actor | косвенно | аудит согласования | да |

## Где хранится

- **Прод-хранилище:** Postgres воркспейса Twenty (Railway, env production «Twenty Credos Time»). Не наша БД — платформа.
- **В git (НЕДОПУСТИМО для реальных значений):** см. [finding CISO-001](findings/CISO-001-pii-in-git.md) — реальные ФИО+email в `seed-real.mjs`. Требуется обезличивание.
- **Источник реальных данных:** выгрузка Директум5 `research/directum5/*.xlsx` — должна быть gitignored.

## Кто видит (доступ)

| Роль | Доступ к PII сотрудников | Доступ к трудозатратам |
|---|---|---|
| Сотрудник (`defineApplicationRole`) | свой профиль | свои записи |
| Руководитель (`manager.role.ts`) | все профили отдела | правка/согласование |
| **Catalog-app / Sales (общий Employee, ADR-0003)** | ⚠️ потенциально все ФИО/email | — |

⚠️ **Риск межапповой видимости** — общий мастер-объект Employee (ADR-0003) делает PII доступной каталогу/продажам без явного RBAC. См. [CISO-004](RISK_REGISTER.md).

## Принципы 152-ФЗ (для внутреннего инструмента)

1. **Минимизация:** только поля, нужные для учёта. `middleName`/`email` — пересмотреть необходимость.
2. **Цель обработки:** учёт рабочего времени сотрудников Кредо-С. Не для передачи третьим лицам.
3. **Хранение:** внутри контура (Railway prod-env). Реальные ПДн — не в VCS.
4. **Доступ:** по ролям, least-privilege. Согласование — separation of duties.
