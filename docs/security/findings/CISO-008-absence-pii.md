# CISO-008 — credosTimeAbsence.note: потенциальные медицинские ПДн (P3)

**Статус:** OPEN · **Severity:** P3 · **Owner устранения:** arch (дизайн) + Dev 2 · **Дата:** 2026-06-20

## Суть

Объект `credosTimeAbsence` (`absence.object.ts`) содержит TEXT-поле `note` (nullable) с лейблом «Примечание». Типы отсутствий включают «больничный» — пользователи могут вносить медицинские сведения в поле `note` («больничный по диагнозу Х», «операция»). Это **медицинские ПДн** (специальная категория 152-ФЗ ст. 10), требующие явного согласия и повышенной защиты.

## Риск

- Поле TEXT без validation — любой текст.
- Нет field-level ограничений: весь объект видим через платформенный UI с ролью function-role / manager.role.
- Тип «больничный» прямо провоцирует указание диагноза в `note`.
- Класс: нечаянный сбор спецкатегорий ПДн без явного ЛНА.

## Смягчает

- Dev-среда, внутренняя сеть.
- Текущего ЛНА для absence нет → риск материализуется при реальном использовании.
- `note` nullable → пользователи не обязаны заполнять.

## Severity P3

Потенциальный риск, не актуальный в dev. Но инструктировать до релиза.

## Требование

1. **Инструкция пользователям** (help-текст или placeholder): «Не указывайте медицинские сведения. Для типа "больничный" — только факт отсутствия».
2. **ЛНА:** внести `credosTimeAbsence.note` в реестр обработки ПДн (152-ФЗ) только как «факт отсутствия без медицинских деталей».
3. **Field-level restriction (опционально):** скрыть `note` от ролей кроме HR/руководителя при появлении этих ролей (аналог CISO-003).
4. **Мониторинг:** при появлении данных в реестре — выгрузить `note` и проверить на наличие медицинских слов.

## DoD

- Help-text на поле note: «не указывайте диагнозы/медицинские сведения».
- `PII_INVENTORY.md` обновлён: absence.note включён как «фактические данные без медицинских деталей».
