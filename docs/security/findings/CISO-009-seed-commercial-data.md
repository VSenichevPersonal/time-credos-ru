# CISO-009: Коммерческая конфиденциальная информация в seed-данных

**Severity:** P3  
**Статус:** OPEN  
**Owner:** Dev 2 (fix seed-real.mjs), arch (commit)  
**Найдено:** 2026-06-20 Dev 2 (наблюдение → зарегистрировано CISO)  

## Суть

`apps/time/scripts/seed-real.mjs` содержит реальные наименования клиентов Кредо-С (ООО/ГУП/ФГБУ) и трудозатраты по ним в репозитории. Это коммерческая конфиденциальная информация (данные о клиентах = КТ), не employee-PII в смысле 152-ФЗ.

## Риск

- Git history = публичный при будущем open-source или при компрометации репозитория.
- Наименования заказчиков + объёмы работ = конкурентная информация.
- Нарушение стандартного NDA с клиентами (реальные названия → сид → тест-среда).

## Что именно

Поле `legal:` в объектах клиентов seed: реальные юридические названия организаций-заказчиков + проекты с трудозатратами. Не относится к сотрудникам (не регулируется 152-ФЗ напрямую).

## Требование к Dev 2

1. В `seed-real.mjs` заменить реальные наименования клиентов на синтетические: «ООО Заказчик-1», «ФГБУ НИИ-2» и т.п.
2. Реальные наименования → в `research/` (gitignored) если нужны для смыслового сида.
3. Аналогично CISO-001 паттерн: синтетика в git, реальные данные gitignored.

## Зависимость от CISO-001

Та же проблема что CISO-001 (ПДн), тот же фикс-паттерн. Может делаться одновременно при обезличивании seed-real.mjs.

## Обходной путь до фикса

Dev и staging — внутренние контуры. Пока риск принимаем как dev-среда. До деплоя в прод — обязательно устранить.
