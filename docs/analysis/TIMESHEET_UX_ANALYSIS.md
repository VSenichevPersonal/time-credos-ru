Легенда приоритета: **P0** обязательно · **P1** сильно желательно · **P2** опционально · **P3** позже/вне скоупа.

| # | Действие | Нужно? | Приоритет | Где разместить | Референс |
|---|---|---|---|---|---|

| A5 | Очистить неделю (все черновики) | **да** | **P2** | верх, в «ещё ▾» (опасное → спрятать, с confirm) | Timetta `removeLines` |

| A7 | Bulk-fill строки (часы во все будни) | есть | P1 | сделать заметнее: в контекст-меню строки + сохранить Shift+Enter | Timetta/наш U5 |

| A9 | Копировать прошлую неделю / …с часами | есть | P0 | верх — частое, периодное действие | Timetta `copyLines*` |
| A10 | Заполнить 8×5 | есть | P1 | верх | Timetta schedule-fill |
| A11 | Отозвать из согласования (revoke) | **да** | **P2** | подвал (когда статус «На согласовании») | Timetta `revoke` |
| A12 | Recent/избранные комбинации проект+работа | частично | P2 | add-row: чипы «недавние» над полями | Kimai Favorites |

**Принцип распределения (impeccable + правило «частые→верх, контекст→строка»):**
- **Верх (тулбар)** = действия над всем периодом, частые, не разрушающие: режим, навигация, копировать неделю, заполнить 8×5. Опасное (очистить неделю) — прячем в «ещё ▾».
- **Строка** = контекст конкретной строки: дублировать, очистить строку, удалить, bulk-fill, комментарий. Сейчас разбросано (⧉ у проекта, мини-кнопка в ячейке) — собрать в одно контекст-меню **⋯** на строке.
- **Ячейка (место ввода)** = быстрый ввод значения: чип «8» в пустую ячейку, поповер-комментарий.

## 3. Размещение и оптимальность

### 3.1 «В месте ввода» vs «вверху» — что куда

Заказчик хочет «больше кнопок в месте ввода ИЛИ вверху». Ответ: **и там, и там, но по разной природе действий.**

- **Вверху** хорошо ложатся действия НАД периодом: их немного, они частые, делаются раз за неделю. Сейчас их 3 (копировать ×2, 8×5) — это **верхняя граница, что влезает без перегруза**. Больше текстовых кнопок вверх не добавлять — разбивать на primary + «ещё ▾».
- **В месте ввода** (строка/ячейка) логичны контекстные действия конкретной строки. Сейчас они есть, но **спрятаны и несогласованы**: «⧉» без подписи у проекта + мини-кнопка-квадратик 10px в ячейке. Пользователь их не находит. Это и есть корень жалобы «хочется больше кнопок в месте ввода» — кнопки ЕСТЬ, но **невидимы**.

**Вывод:** проблема не в количестве, а в **обнаруживаемости и группировке**. Решение — собрать контекстные действия строки в одно меню **⋯**, а в ячейку добавить заметный быстрый ввод «8».

### 3.2 Группировка вверху (impeccable: иерархия primary/secondary)

Сейчас все 3 действия — одинаковые серые кнопки одного веса. Нет иерархии. Рекомендация:

```
[День|Неделя|Проект]            ‹ 15–21 июня › [Сегодня]   [Копировать ▾] [Заполнить 8×5]  [?]
                                                                  └─ Только строки
                                                                  └─ Строки и часы
                                                                  └────────────
                                                                  └─ ⚠ Очистить неделю
```

- «Копировать неделю» и «…с часами» — это **один смысловой кластер** (перенос прошлой недели). Объединить в split/dropdown **«Копировать ▾»**: первичное действие + варианты. Убирает 2 кнопки в 1, лечит непонятное «…с часами».
- «Заполнить 8×5» оставить отдельной кнопкой (частое, безопасное).
- «Очистить неделю» (новое, опасное) — внизу того же dropdown, с разделителем и confirm.
- «?» — справа, минимальный.

### 3.3 Контекст-меню строки (вместо разбросанных иконок)

```
● ПРОЕКТ-123  ⋯       Вид работ        8   8   8   ·   ·   Σ24
              └─ Дублировать строку
              └─ Заполнить будни (8 ч во все)
              └─ Очистить строку
              └────────────────
              └─ Удалить строку
```

Один always-visible (или hover) триггер **⋯** вместо «⧉». Подписи вместо символов. Bulk-fill из невидимого 10px-квадрата переезжает сюда (Shift+Enter остаётся для клавиатурщиков).

### 3.4 Быстрый ввод в ячейку

```
активная пустая ячейка:   [ ·  ⁸]   ← чип «8» одним кликом ставит норму дня
```

Мини-чип «8» (норма дня из `useDailyNorm`, не хардкод) в активной пустой ячейке. Это буквально «кнопка в месте ввода» — ровно то, что просит заказчик, и быстрее чем печатать.

## 4. Понятность (переименования и подсказки)

| Сейчас | Проблема | Рекомендация |
|---|---|---|
| «…с часами» | непонятно «…с чем»; зависит от соседней кнопки | в dropdown «Копировать» → пункт **«Строки и часы прошлой недели»** |
| «Копировать неделю» | не ясно, что без часов | пункт **«Только строки (без часов)»** |
| «⧉» | символ-загадка, без подписи | пункт меню **«Дублировать строку»** с текстом |
| мини-квадрат в ячейке | 10px, без подписи, неочевиден | перенести в контекст-меню «Заполнить будни» + tooltip |
| «Заполнить 8×5» | «×5» — жаргон | оставить (компактно, тултип объясняет) или **«Заполнить норму»** |
| Комментарий | есть только в Дне | добавить в Неделю (поповер на ячейке) |

Все кнопки уже имеют `title`-тултипы (хорошо). Главный дефицит — **подписи вместо символов** и **группировка кластеров**.

## 5. Ограничения песочницы (Remote DOM)

Front-components рендерятся в изолированном Remote DOM (Twenty Apps): нет доступа к host-DOM, нет нативного drag-and-drop, ограничен набор примитивов. Что это значит для рекомендаций:

| Рекомендация | Реализуемо? | Примечание |
|---|---|---|
| Dropdown «Копировать ▾» | да | состояние через useState, абсолютный слой как `cheatsheet` |
| Контекст-меню строки ⋯ | да | поповер на useState (паттерн `cheatsheet`/`autocomplete` уже есть) |
| Чип «8» в ячейке | да | обычная кнопка, как существующая мини-кнопка bulk-fill |
| Поповер-комментарий на ячейке Недели | да | как `autocomplete` dropUp / input в `day-row` |
| Очистить строку/неделю | да | bulk-remove через `remove`/новый upsertMany(0) — логика есть |
| Drag-перетаскивание строк/блоков | **нет** | нет host-DOM DnD; не пытаться (Kimai-календарь неприменим) |
| Таймер реального времени | технически да, но вне скоупа | требует фон-тика, отложить |

Все приоритетные рекомендации (A2–A8) **реализуемы** существующими паттернами (useState-поповеры, кнопки). Drag — единственное, что отпадает.

## 6. Топ-рекомендации

### Добавить ВВЕРХ (тулбар)
1. **P1** Свернуть «Копировать неделю» + «…с часами» в один **dropdown «Копировать ▾»** с понятными подписями (лечит непонятное «…с часами», экономит место).
2. **P2** В тот же dropdown добавить **«Очистить неделю»** (опасное → внизу, с confirm).

### Добавить В МЕСТО ВВОДА (строка/ячейка) — главный запрос заказчика
3. **P1** Заменить одинокую «⧉» на **контекст-меню строки ⋯**: Дублировать · Заполнить будни · Очистить строку · Удалить строку. Подписи вместо символов — кнопки станут видимыми.
4. **P1** **Чип «8»** (норма дня) в активной пустой ячейке — однокликовый быстрый ввод прямо в месте набора.
5. **P1** **Комментарий к ячейке в режиме Неделя** (поповер) — `commitDescription` уже есть в actions, не выведен в UI Недели.

### Понятность
6. **P1** Переименования: «Только строки» / «Строки и часы» вместо «Копировать неделю» / «…с часами»; текстовые подписи в контекст-меню вместо «⧉».

### Не делать
- Drag-and-drop (нет в Remote DOM), таймер, заявки на отсутствие, строки до/после — вне скоупа/неприменимо.

### Корневой вывод
Жалоба «мало кнопок в месте ввода» — на деле **проблема обнаруживаемости**: действия (дубль, bulk-fill) уже есть, но спрятаны в 10px-иконки без подписей. Не плодить кнопки, а **собрать контекстные действия строки в видимое меню ⋯**, добавить **чип «8» в ячейку**, и **сгруппировать копирование вверху в dropdown**. Это закрывает запрос минимальными средствами и без перегруза (правило «не переусложнять»).
