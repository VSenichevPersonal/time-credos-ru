## A1. Таймшит — тулбар и строки

**Зона кода:** `grid/toolbar.tsx`, `mode-switcher.tsx`, `period-nav.tsx`, `copy-menu.tsx`, `filters-bar.tsx`, `use-filters.ts`, `add-row.tsx`, `row-menu.tsx`, `use-timesheet-actions.ts`, `cheatsheet.tsx`.

**Текущее состояние (факты из кода):**

### Группа A1.1 — Режимы День/Неделя/Проект

**A1.1.** Нужны ли все 3 режима, или какой-то лишний/мёртвый?
— (а) оставить 3; (б) убрать «Проект» (агрегатный, редко нужен); (в) «День» сделать дефолтом для быстрого ввода.
— реф: Timetta — основной вид Неделя; «День» как фокус-ввод; агрегат «Проект» отдельным отчётом, не режимом таймшита.

**A1.2.** При переключении режима период сохраняется или сбрасывается на «сегодня»?
— (а) сохранять выбранную неделю/день; (б) сбрасывать на сегодня; (в) запоминать последний период per-режим.
— реф: Timetta — период сохраняется между видами.

**A1.3.** Режим «Проект» — это группировка строк по проекту внутри той же недели или отдельная агрегатная таблица?
— (а) группировка строк (свернуть по проекту); (б) сводка часов по проекту за период; (в) убрать, заменить фильтром «Проект».

**A1.4.** Где должен жить переключатель режимов — слева у заголовка (как сейчас) или объединить с периодом?
— (а) как сейчас; (б) сегмент-контрол по центру; (в) в выпадающем меню (экономия места при узком виджете).

## A2. Планирование — доска (capacity board)

**Зона кода:** `capacity/capacity-board.tsx`, `board-toolbar.tsx`, `board-rows.tsx`, `dept-row.tsx`, `employee-row.tsx`, `dept-plan-row.tsx`, `project-plan-row.tsx`, `planned-project-row.tsx`, `project-detail.tsx`, `summary-row.tsx`, `booking-marker.tsx`, `board-legend.tsx`, `period-header.tsx`, `mode-switcher.tsx`, `use-capacity.ts`, `use-plan-edit.ts`, `calc-load.ts`, `assignment-types.ts`, `cap-tokens.ts`.

**Текущее состояние (факты из кода):**
- Срезы (`CapAxis`): `dept` (Отделы) / `employee` (Люди) через Segmented. Планирование доступно **только в срезе Отделы** (`effectiveAxis = planning ? 'dept' : axis`).
- 4 метрики ячейки (Segmented): `free` (Свободно), `pct` (Загрузка %), `plan` (План), `gap` (Gap = load−capacity по Timetta). Цвета: зелёный (свободно) → прозрачно → терракот (перегруз); Gap синий профицит / янтарь-терракот дефицит (пороги ±5/15%).
- `LoadCell`: capacity, load (=план+HARD-бронь), free, ratio, hardBooking, softBooking, conflict.
- Раскрытие отдела → `ProjectDetail` (проекты + планы без проекта) → 3-й уровень: проект по долям отделов если ≥2 отдела.
- Booking: HARD (входит в load, сплошной `▪ Nч`) / SOFT (не потребляет ёмкость, пунктир, виден только при `settings.tentativeBookingEnabled`).
- Overbooking: `conflict = load > capacity` → терракот-обводка inset + иконка ▲. Не блокируется.
- Период: недели (16 шт) / месяцы (6 шт), горизонт фиксирован — **нет стрелок вперёд/назад**.
- Кнопка «✎ Планировать» (только `isManager`) → inline-ввод плановых часов + срок проекту (PATCH), autosave на blur/Enter.

**Текущее состояние (факты из кода):**

**Ключевые сквозные темы для решения заказчиком:**
1. **Норма дня SSOT** (8 vs календарь) — повторяется в A1, A4.
2. **Cross-filter пилюли** как единый паттерн (есть в Отчётах) — переиспользовать в Таймшите (A1.18) и доске.
3. **RBAC/identity** (CISO-005/007) — блокирует полный гейт в Отчётах (A3.28) и согласовании (A4.19).
4. **Remote DOM**: нет drag (A2.29), нет скачивания файла (A3.21), hover→кнопки (A1.27).
