## E1. Микро-взаимодействия

**E1.1** → **(а)+(б)** глиф «продлить на будни» (видимый, не ⇥-хоткей) + дубль в меню строки ⋯. RemDOM: drag-fill невозможен. Видимый fallback обязателен (imp).

**E1.2** → **(а)** порядок строк фиксирован (по проекту/алфавиту). Drag-reorder невозможен, ручной reorder не нужен. Ref=T.

**E1.3** → **(б)** Shift+клик задаёт диапазон от активной до кликнутой. Реализуемо на координатах `(row,col)` без host-DOM.

**E1.4** → **(а)** клик = inline-ввод, всё. Двойному клику отдельную роль НЕ давать — путает. Расширенная форма — через ⋯.

**E1.5** → **(в)** при печати цифры (seed) — заменить значение; при клике в ячейку — выделить всё. Excel-паттерн F2.

**E1.6** → **(б)** красная рамка input + подсказка форматов при невалидном вводе. Тихий откат (а) скрывает проблему.

**E1.7** → **(а)** placeholder в пустом input «8 / 8:30» + **(б)** полный список в cheatsheet. Формат богаче чем у T/K — проявить.

**E1.8** → **(а)** поле комментария под активной ячейкой в режиме edit. Иконка ✎ в ячейке недостижима — убрать.

**E1.9** → **(б)** норма дня + «повтор последнего» (↩). Настраиваемые пресеты — roadmap.

**E1.10** → **(а)** ⋯ всегда видим. RemDOM: hover ненадёжен. Надёжность > экономия места.

**E1.11** → **(а)** ячейка = только число/input (вариант C1). Закрывает корневой баг контролов. Решено в TIMESHEET_CELL_CONTROLS §F4.

**E1.12** → **(а)** точка-маркер комментария в углу. Совпадает с T/K. Различать от save/статус цветом.

**E1.13** → **(в)** подсказки-hint текстом под контролом (паттерн меню ⋯ уже есть). Свой JS-tooltip с позиционированием в RemDOM сложен. Критичные `title` → `aria-label`.

**E1.14** → **(а)+(в)** title locked-ячейки: «Согласовано» + «Отозвать согласование» как действие в меню строки. imp: blocked state объясняет why+how.

**E1.15** → **(а)** active остаётся после blur-commit (видно «где был»). Снимать только клик-вне сетки.

**E1.16** → **(б)** сохранять позицию `(row,col)` между неделями. Автофокус только при первой загрузке.

**E1.17** → **(а)** при входе в виджет клавиатурой — подсветить первую ячейку + **(в)** подсказка «стрелки для навигации». imp: focus-ring обязателен.

**E1.18** → **(б)** внутренний clipboard виджета (React-стейт), Cmd+V в активную ячейку. Системный clipboard в RemDOM недоступен.

**E1.19** → **(а)** копировать всю прошлую неделю. Выборочная вставка — избыточна для MVP.

**E1.20** → **(б)** Ctrl+D «заполнить вниз/вправо» (Excel-паттерн). Экономит при повторах.

**E1.21** → **(а)** оставить значение + «Не сохранено» + «Повторить». Не откатывать молча — потеря ввода хуже.

**E1.22** → **(а)** undo-тост «Строка удалена · Отменить» (5с) вместо confirm для обратимого. Confirm оставить на «Очистить неделю» (batch). imp: Undo > Confirm.

## E2. Motion / анимации

**E2.1** → **(в)** только fade opacity на появление поповеров (150мс). Без layout-анимаций. Плотный инструмент — сдержанно.

**E2.2** → **(а)** жёсткое правило: НЕ анимировать layout (width/height/top/left/margin). Только opacity/color/box-shadow/transform.

**E2.3** → **(б)** короткий тинт ячейки 100–150мс при успешном save. Обратная связь рядом с действием (imp).

**E2.4** → **(а)** один токен `--ease-out-quart` `cubic-bezier(0.25,1,0.5,1)` для всех входов. Убрать `ease` (imp банит).

**E2.5** → **(а)** 100мс feedback / 200мс меню, без анимаций >300мс. «Быстро и сдержанно».

**E2.6** → **(б)** crossfade 150мс (opacity) при смене режимов/вкладок. Slide = layout, против правила.

**E2.7** → **(б)** box-shadow transition 80–100мс на active-ячейку. Мягко, не тормозит навигацию.

**E2.8** → **(а)** skeleton-строки в форме таблицы вместо «Загрузка…». imp: skeleton > spinner.

**E2.9** → **(а)** статичный skeleton без shimmer. Надёжнее в RemDOM/Worker. Shimmer — позже если плавно.

**E2.10** → **(а)** держать анимации минимальными изначально → тумблер reduced-motion не нужен. prefers-reduced-motion в RemDOM недоступен, минимум анимаций снижает риск.

**E2.11** → **(б)** анимировать только тосты (вход fade/slide — важно заметить сообщение). Остальное — без анимации.

**E4.8** → **(а)** набор глобальных шорткатов виджета: `?` подсказка, смена недели, режимы, Ctrl+Enter submit. Ловить на контейнере.

## F3. Undo/redo + автосейв-конфликты

**F3.1** → **(а)** Undo последнего действия (ввод/очистка/удаление). Снимает страх при autosave. Полный стек — избыточен.

**F3.2** → **(а)** только последнее действие, в памяти сессии. После reload не живёт.

**F3.3** → **(в)** toast «Отменить» (5–6с) + Ctrl+Z. Оба канала.

**F3.4** → **(а)** удаление строки → toast-undo без модалки. Унифицировать с обратимыми (imp).

**F3.5** → **(а)** счётчик «будет удалено N записей» в модалке «Очистить неделю».

**F3.6** → **(а)** автокоммит при уходе (как blur). Без отдельного предупреждения.

**F3.7** → **(а)** оставить autosave по blur/Enter/Tab (явный коммит). Debounce при наборе — риск частых PATCH.

**F3.8** → **(б)** тулбар-индикатор + микро-«✓» на ячейке после save. Подтверждение рядом с действием.

**F3.9** → **(б)** НЕ откатывать молча — оставить ввод + «Повторить» в тосте. Потеря ввода хуже (см. E1.21).

**F3.10** → **(а)** только autosave (быстрее). Опция «требовать явное сохранение» — roadmap для осторожных.

**F3.11** → **(в)** мягкое «данные обновлены» при расхождении после сохранения. Полный optimistic-concurrency избыточен для масштаба. APPROVED уже lock (CISO-012).

**F3.12** → **(а)** «изменено другим, перезагрузите» + reload. Просто и надёжно.

**F3.13** → **(а)** presence не нужен, масштаб мал.

**F3.14** → **(а)** blur коммитит сразу, локальный draft-буфер не нужен.

**F3.15** → **(а)** восстанавливать период недели + фильтры (view-state в settings). Drill — известный пробел, отдельно.
