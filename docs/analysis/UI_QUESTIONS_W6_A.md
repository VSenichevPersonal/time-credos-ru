## Группа 1 — Расхождения код ↔ утверждённые решения (приоритет: разнобой видит пользователь)

### W6A.1 — «Gap» в коде vs «Баланс» в решении D4.1
Код: `board-toolbar.tsx` метрика-чип `"Gap"`, легенда `"Gap = спрос − ёмкость:"`, тултип `"Gap = спрос − ёмкость (▲ дефицит / ▼ профицит)..."`. Решение D4.1(а): основное название = **«Баланс»**, слово «Gap» только в hint. Код не перевели. Финальный текст чипа?
- (а) **«Баланс»** в чипе/легенде/заголовке колонки; «Gap» сохранить ТОЛЬКО внутри тултипа-формулы: `«Баланс = спрос − ёмкость (Gap). ▲ дефицит / ▼ профицит»`. ← по D4.1
- (б) Оставить «Gap» (короче, влезает в узкий чип; HR-аудитория поймёт).
- (в) «Баланс ч» (с единицей, как соседние «Свободно ч» / «План ч»).
Реф: Timetta использует «Дефицит/Профицит ресурсов»; Kimai термина нет. **Рекомендую (а)+(в): чип «Баланс», единица не нужна т.к. Gap может быть в % или ч — уточнить ниже W6A.2.**

### W6A.2 — Единицы у метрик-чипов доски: разнобой «ч» / «%» / без
Код `board-toolbar.tsx`: `"Свободно ч"`, `"Загрузка %"`, `"План ч"`, `"Gap"` (без единицы). У «Gap» единицы нет, у остальных есть. Финал?
- (а) Дать всем единицу единообразно: `«Свободно, ч»`, `«Загрузка, %»`, `«План, ч»`, `«Баланс, ч»` (запятая+единица — как в KPI-картах reports: `«Факт, ч»`).
- (б) Оставить без запятой (`«Свободно ч»`) — компактнее в узком чипе.
- (в) «Баланс» оставить без единицы (знак ▲/▼ важнее числа).
Реф: KPI-карты в reports уже используют шаблон `«{Label}, {unit}»` (`Факт, ч` / `Норма, ч`). **Рекомендую (а) — выровнять под reports-шаблон `«Label, ед»`.**

### W6A.3 — Десятичный разделитель часов: точка в коде vs запятая в решении G4.5
Решение G4.5(б): **в UI запятая** (RU-стандарт), точка только в CSV. Инвентаризация: код форматирует часы как `8.5`, `8.25` (точка) в report-tokens/format. Это расхождение видно везде (KPI, таблицы, тултипы). Финал?
- (а) **Запятая в выводе UI** (`8,5 ч`), точка только в экспортируемом CSV/числовом поле ввода. ← по G4.5
- (б) Точка везде (проще, не трогать formatHours).
Реф: 152-ФЗ/ГОСТ — десятичная запятая. Timetta RU-локаль = запятая. **Рекомендую (а). Уточнение: ввод принимает обе (G4.6), вывод — запятая.**

### W6A.4 — Знак недогруза/перегруза: формат по местам
Код: KPI `"Недогруз, ч"` с хинтом `"до нормы"`/`"перегруз"`; report-tokens — недобор со знаком `−` (`−10 ч`), переработка со знаком `+` (`+5 ч`). В KPI-карте знак не показан (отдельная метрика «Недогруз»), а в таблицах — со знаком. Какой формат финальный в КАЖДОМ месте?
- (а) **KPI-карта** «Недогруз»: модуль без знака (`10 ч`) + цветной хинт «до нормы»/«перегруз»; **ячейка таблицы** «Недогруз/Остаток»: со знаком (`−10` / `+5`). Разделить семантику: карта=сводка, ячейка=дельта.
- (б) Везде со знаком (единообразие).
- (в) Везде модуль + текстовый суффикс «не хватает»/«сверх».
Реф: Timetta «Недоступно/Переработка» без знака в карточках, со знаком в детализации. **Рекомендую (а).**

### W6A.5 — Размер ячейки таймшита код 58×32 vs решение 64×40 (W3A.25)
Не текст, но микродеталь-расхождение, влияет на touch-таргет (impeccable: ≥44px touch). Финал?

### W6A.6 — Таймшит «Записи»: нет записей за неделю
В инвентаре прямого empty таймшит-грида нет (грид всегда рисует дни). Но my-hours: `"За период нет записей. Внесите трудозатраты на странице \"Записи\"."` Нужен ли отдельный empty внутри самого таймшита (все дни пусты)?
- (а) Заголовок «Неделя пуста», подсказка «Кликните по ячейке или нажмите «Заполнить норму на неделю», чтобы начать». **Кнопка «Заполнить неделю».**
- (б) Не нужен empty — пустой грид сам по себе призыв к действию (плейсхолдер-норма в ячейках по W3A.17 уже подсказывает).
- (в) Тонкая подсказка-строка над гридом, не полноэкранный StateView.
Реф: Timetta показывает пустой грид + кнопку Fill. **Рекомендую (б)+(в): без полноэкранного empty, плейсхолдер+тонкий хинт.**

### W6A.7 — Согласование колонок empty/loading-текстов «…» (троеточие)
Код использует символ `…` (U+2026) во ВСЕХ loading: `"Загрузка часов…"`, `"Загрузка команды…"` и т.д. Утверждаем единый стиль?
- (а) **Везде `…` (U+2026, один символ)** — уже консистентно в коде, утвердить как SSOT-правило.
- (б) Убрать троеточие у скелетонов (D1.2: таблицы=скелетон без текста), оставить «Загрузка…» только для inline-блоков.
**Рекомендую (а)+(б): где скелетон — текста нет; где inline-текст — `…` единым символом.**

### W6A.8 — Reports «Сводка» (breakdown) empty: текст + действие
Код `breakdown-table.tsx`: заголовок `"Нет данных за период"`, подсказка `"По этому срезу за выбранный период списаний нет. Смените период или срез группировки."` Кнопки-действия нет (D1.12 предлагал «[Сменить период]»). Добавляем кнопку?
- (а) Оставить текст как есть + **кнопка-действие «Сменить период»** (скроллит/фокусит переключатель периода). ← D1.12
- (б) Текст как есть, без кнопки (период рядом в тулбаре, кнопка избыточна).
- (в) Уточнить подсказку: «…списаний нет. Попробуйте другой период или сгруппируйте иначе.»
**Рекомендую (б): период в одном экране сверху, кнопка дублирует. Текст утвердить как есть.**

### W6A.9 — Reports «Незаполненные» — три состояния, тексты финальные?
Код `missing-view.tsx`: (1) выключено: `"Напоминания выключены"` / `"Включите напоминания… — здесь появится сводка по недозаполнившим за неделю."` (2) все ОК: `"Все заполнили — недоборов нет"` / `"За {weekLabel} норма закрыта у всех сотрудников."` Утверждаем?

Реф: impeccable — empty с действием лучше empty-инструкции. **Рекомендую (а) для текста + (в) кнопку deep-link в «выключено».**

### W6A.15 — Текст confirm удаления недели (со счётчиком)

## Группа 5 — Форматы чисел/дат/процентов по конкретным местам

### W6A.24 — Где «8 ч», где «8», где «8,5», где «8:30» — карта форматов
Решения: E1.7 (ввод принимает 8/8.5/8:30), G4.13 (переключатель Decimal/HH:MM на вывод). Нужна карта «какой формат в каком месте вывода»:
- (а) **Ввод-ячейка таймшита**: плейсхолдер «8 / 8:30», принимает любой; **вывод-ячейка**: по настройке Decimal(`8,5`)/HH:MM(`8:30`). **KPI/таблицы/тултипы**: всегда Decimal с запятой + суффикс «ч» (`8,5 ч`). **Проценты**: целое `%` без десятых (`35%`), кроме когда <1% → «<1%».
- (б) Везде Decimal, HH:MM не выводить (только принимать на ввод).
Реф: Timetta — вывод по настройке пользователя; Kimai — HH:MM по умолчанию. **Рекомендую (а): единая карта, проценты целые.**

### W6A.25 — Формат заголовка недели: «13–19 янв 2026» (A1.8)
Решение A1.8: диапазон дат вместо номера недели. Код my-periods показывает «Неделя» + сокр.месяцы. Финал формата?
- (а) **«13–19 янв 2026»** (en-dash, сокр.месяц, год). Если неделя в двух месяцах: «28 дек – 3 янв 2026».
- (б) «13–19 января» (полный месяц, без года если текущий).
- (в) Добавить № недели мелким: «13–19 янв · нед.3».
Реф: A1.8 утвердил диапазон. **Рекомендую (а), переход через месяц по образцу «28 дек – 3 янв».**

### W6A.26 — Дата «последней активности» — относительная или абсолютная
Код project-team колонка «Последняя», project-summary «Последняя активность», формат ДД.ММ.ГГГГ. Финал?
- (а) Абсолютная **ДД.ММ.ГГГГ** (`10.05.2024`) — однозначно, сортируемо.
- (б) Относительная «3 дня назад» / «сегодня» / «вчера», абсолютная в тултипе.
- (в) Гибрид: ≤7 дней — относительная, дальше — абсолютная.
Реф: impeccable — относительная время дружелюбнее для «свежести». **Рекомендую (в): свежесть важна для «кто давно не списывал».**

### W6A.27 — Проценты: округление и пограничные значения
Утилизация/Загрузка/Выработка показываются как `%`. Финал правил округления?
- (а) Целое `%` (`35%`, `100%`, `120%`); `0%` показываем; `>0 и <1` → «<1%»; null/нет данных → «—» (не «0%»).
- (б) Одна десятая (`35,4%`) — точнее для KPI.
**Рекомендую (а): целое + явное «—» для отсутствия (НЕ «0%»).**

### W6A.28 — Пустое значение vs ноль: «—» / «0» / пусто
Где показывать «—» (нет данных), где «0»/«0 ч» (есть данные = ноль)? Сейчас в коде разнобой.
- (а) **«0 ч» — когда метрика посчитана и равна нулю** (списано 0); **«—» — когда данных нет/неприменимо** (плана нет, отдел без ёмкости). Никогда не показывать «0» вместо «нет данных».
- (б) Пустая ячейка вместо «—».
Реф: код уже различает «нет плана»/«плановые часы не заданы». **Рекомендую (а) как SSOT-правило null vs zero.**

## Группа 6 — Заголовки колонок/кнопок, порядок контролов, скелетоны, a11y

### W6A.29 — Сокращённые заголовки колонок календаря: расшифровка
Код month-table: «Кал. дней», «Раб. дней», «Вых.+празд.», «Коротких», «Раб. часов». Финал (узкие колонки + тултип-расшифровка)?

- (б) Развернуть: «Кал. дней»→«Календарных», «Раб.»→«Рабочих» (если влезает).
**Рекомендую (а): сокращения + обязательный тултип (W6A.22).**

### W6A.30 — Заголовки кнопок заполнения таймшита (D4.5/D4.6)
Решение: «Заполнить норму на всю неделю» (тулбар), «Заполнить неделю для всех», «Заполнить будни этой строки». Финал точных подписей кнопок?
- (а) Тулбар-кнопка: **«Заполнить неделю»** (норму×рабочие дни), тултип «Заполнит норму дня в каждый рабочий день недели». Строка: иконка-кнопка «Заполнить будни строки» (тултип). Manager-действие: «Заполнить неделю всем».
- (б) Длинные подписи целиком на кнопке (без тултипа).
**Рекомендую (а): короткая подпись + тултип-уточнение, не длинная кнопка.**

### W6A.31 — Скелетоны: форма по типу контента (D1.2)
Решение D1.2: скелетон для таблиц/сеток, текст «Загрузка…» для inline. Финал форм скелетонов по экранам?
- (а) **Таймшит-грид**: скелетон-сетка (строки×дни, серые прямоугольники ячеек). **Reports-таблицы**: 5–6 строк-плейсхолдеров + шапка. **KPI-карты**: 4 карты-плейсхолдера. **Доска планирования**: строки отделов + period-header. **Графики (тренд)**: прямоугольник-плейсхолдер с осью. Inline (my-hours по проектам) — текст «Загрузка проектов…».

**Рекомендую (а): форма скелетона повторяет реальный лейаут (impeccable — снижает CLS-«прыжок»).**

### W6A.32 — Порядок/группировка контролов тулбара доски и reports
Код доски: [Сегмент Отделы/Люди] [метрики] [Недели/Месяцы] [Планировать]. Reports: [виды] [период-нав] [гранулярность] [фильтр категория] [сегмент Отдел/Проект/Человек]. Финал порядка (слева→направо)?

### W6A.33 — a11y: тексты focus/hover и aria-label усечённых элементов

### W6A.34 — Заголовки колонок: «Часов» vs «Часы» vs «Факт, ч» — единый стиль
