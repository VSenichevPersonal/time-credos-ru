# Полнота раздела «Отчёты» — аудит vs Timetta + Kimai

Дата: 2026-06-21. Автор: аналитик AITEAM. Статус: READ-ONLY анализ.

Вопрос заказчика: «Все ли нужные отчёты есть? Сейчас только Сводка и Тренд».

Короткий ответ: видимых вкладок две (Сводка, Тренд), но за ними OLAP-drill по 6 осям, KPI и экспорт. Часть готовых бэк-агрегатов не выведена (табель Т-13, незаполненные сводно), двух стандартных PSA-отчётов нет совсем (проекты план/факт/остаток; отсутствия).

## Готово на бэке, НЕ выведено в UI
- `buildTimesheetGrid` — Табель Т-13 (матрица сотрудник×день, коды Я/ОТ/Б, CSV) — нет экрана.
- `computeEmployeeProjects` — проекты сотрудника — нет UI.
- `computeMissingTimesheets` — только баннер, нет сводной таблицы менеджеру.

## Приоритеты
- **P1 дёшево (бэк готов):** Табель Т-13 (нужен grid-компонент); Незаполненные таймшиты сводной таблицей.
- **P1 новый агрегат:** Отчёт по проектам план/факт/остаток (в часах, без денег — [[no-billable-concept]]). Аналог Timetta «Список проектов в работе». Высокая ценность РП.
- **P2 пресетом OLAP (не вкладкой):** Утилизация-рейтинг сотрудников (срез employee+сорт util); Статус согласования (фильтр по статусу).
- **P2 отдельный экран:** Отсутствия/отпуска — реестр поверх credos-time-absence.
- **P3:** Проекты сотрудника (бэк готов), Бронирование/загрузка.
- **Отложено — финансовая волна:** P&L, себестоимость, Акты. Биллинг/счета — НЕ делаем ([[no-billable-concept]]).

**Самая быстрая победа: Табель Т-13** — логика готова и протестирована, не хватает только UI-компонента.

Ключевые файлы: reports-dashboard.tsx, reports-timesheet-grid.ts (Т-13 готов, без UI), reports-calc.ts, missing-timesheets.ts, project-team.ts; объекты credos-time-absence/booking/dept-plan.
