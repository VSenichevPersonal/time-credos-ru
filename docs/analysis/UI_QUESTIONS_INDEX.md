## Сквозные находки (ФАКТЫ из кода, не гипотезы) — кандидаты в ТЗ/бэклог
**Реальные пробелы (не просто вопросы — отсутствующие фичи):**
- Отзыв согласования (un-approve) НЕ реализован · Уведомления НЕ реализованы · «Акт» = заглушка-1С (не акт согласования).
- Скачивание CSV/файла НЕ работает (Remote DOM нет Blob/download → текст в textarea) · печать/выгрузка-1С не реализованы (формат готов).
- Сохранённых видов «Мои проекты/Мой отдел» нет · глобального сброса фильтров нет · drill теряется при перезагрузке.
- Фильтров в UI 6-7, не 10 (нет WorkTypeGroup/Stage/Дата) · недели и произвольного диапазона как периода в отчётах нет.
- Booking: роль-заглушка нет (employee обязателен) · SOFT→HARD только ручной правкой · бронь на доске нередактируема.
- Планирование возможно только в срезе Отделы · горизонт фиксирован (нет стрелок).
- Табель Т-13: готов, конфликт ПДн (Т-13 юр. требует ФИО vs CISO-007 скрывает).
- Импорта справочников нет · календарь строится построчно · soft-delete/архивации в реестрах нет.

**UX/консистентность:**
- Норма дублируется в 3 местах (normHoursPerDay/fillTemplate/чип) — нужен SSOT; «8×5» хардкод 8 vs чип норма-дня.
- Скелетоны только в 2 reports-вью; остальное «Загрузка…» текстом · error-состояния разнородны (ErrorState с/без onRetry vs сырой текст) · empty разнородны (emoji vs текст vs нет).
- A11y: title (~76-92) >> aria-label (~24) · НЕТ role=grid ни в одной сетке · cheatsheet без Esc · контраст textFaint на грани.
- Копирайтинг: англицизм «Gap» · 3 действия «заполнить» неоднозначны · дубль «Сводка»+«Обзор» карточки · дубль календаря (PAGE_LAYOUT+VIEW).
