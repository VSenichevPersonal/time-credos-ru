# REQ-0013 — Проекты с несколькими отделами (2-3 и более)

**Статус:** PROPOSED (БЭКЛОГ — исследовать, заказчик 2026-06-21)
**Приоритет:** 🔴 (влияет на ядро: модель проекта, планирование, отчёты)

## Контекст
Сейчас у проекта ОДИН отдел (`project.departmentId`, MANY_TO_ONE). Реально проект может задействовать **2-3 отдела** (и больше) — напр. внедрение + ИБ-аудит + техцентр на одном клиенте. Нужно поддержать несколько отделов на проект.

## Сверка Timetta/Kimai / PSA
Timetta: проект → команда из разных подразделений, ресурсы/часы распределяются по участникам/отделам. Мульти-отдел — норма для PSA. Часы и план должны раскладываться по отделам, не висеть на одном.

## Решение (изучить, минимально)
- **Вариант A (рекомендую): join-объект `credosTimeProjectDepartment`** (project × department × доля: `plannedEffortShare` часы или %). Каждый отдел — своя доля плановых часов проекта.
  - Загрузка отдела = Σ долей проектов этого отдела (вместо project.plannedEffort целиком на один departmentId).
  - Отчёты byDept — раскладка факта/плана по отделам проекта.
- **Вариант B:** MANY_TO_MANY project↔departments без долей — проще, но не делит часы (тогда план дублируется/неясен). Хуже для capacity.
- Миграция: текущий `project.departmentId` → одна запись join (100% на текущий отдел). Обратная совместимость.

## Влияние (это ядро, не мелочь)
- `calc-load.ts` / `reports-calc.ts`: загрузка/норма раскладываются по отделам проекта (доли), не по одному departmentId.
- Карточка проекта: вкладка «Отделы» (участвующие + доли).
- Планирование: проект в детализации нескольких отделов (его доля в каждом).
- Связь: REQ-0011 (FTE по отделам), REQ-0012 (план без проекта), REQ-0004 (аллокации).
