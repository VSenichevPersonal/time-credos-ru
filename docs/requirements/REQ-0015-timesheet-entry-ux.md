# REQ-0015 — UX ввода таймшита: pre-submit валидация, шаблон 8×5, заливка

**Статус:** PROPOSED (волна-3/4 — заказчик 2026-06-21)
**Приоритет:** 🟡 (удобство ввода, БЕЗ блокера identity — делаем сразу)

## Сверка Timetta/Kimai
Timetta: проверка полноты перед отправкой периода; шаблоны недели. Kimai: дублирование/быстрое заполнение. Берём.

## Состав (всё ложится на стэк сейчас)
1. **Pre-submit валидация** — перед «Отправить на согласование» показать пробелы: незаполненные рабочие дни, дни ниже нормы (из произв.календаря). Клиентский расчёт на загруженной неделе. Не отправлять вслепую. ✅ легко, высокая ценность.
2. **Шаблон недели «8×5»** — один клик: заполнить стандартную неделю (8ч × рабочие дни по календарю) для строки/проекта. Дополняет «Копировать неделю» (старт с нуля). ✅ легко.
3. **Заливка по дням («fill»)** — кнопка «залить значение по рабочим дням строки» (Excel-эффект протяжки). ⚠️ Drag-to-fill в Remote DOM песочнице ненадёжен (нет host-DOM pointer) → реализуем КНОПКОЙ «залить», не drag. Песочница-safe, тот же результат.

## Границы / стэк
- Всё — клиентские действия на уже загруженных данных + RestApiClient upsert. Identity НЕ нужен.
- Drag буквальный — НЕ делаем (песочница). Замена — кнопка-заливка.
- Зона Dev 1 (grid).
