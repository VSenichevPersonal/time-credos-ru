# REQ-0007 — Таймшит: контекст «чей» + селектор по роли

**Статус:** PROPOSED (БЭКЛОГ — «можно потом», заказчик 2026-06-21)
**Приоритет:** 🟡 (UX таймшита; зависит от RBAC/identity)

---

## Сверка с Timetta/Kimai (обязательная)

| Инструмент | «За сотрудника» |
|---|---|
| **Timetta** | Руковод заполняет/согласует таймшиты подчинённых; ресурс-менеджер видит команду. |
| **Kimai** | Admin/teamlead вводит время за других; обычный юзер пишет в любой видимый проект (вне «команды» не блокируется). |

**Вывод:** модель заказчика совпадает с практикой обоих. Пункт 3 (не блокировать запись вне команды) = как Kimai, проще. Подтверждено.

---

## Суть (3 правила, заказчик)

1. **Сотрудник (обычный):** в шапке таймшита — только ПОКАЗ (read-only) «чей таймшит» + отдел (свой). Селектора нет.
2. **Руковод отдела / руковод проекта:** селектор выбора, чей таймшит смотреть/заполнять:
   - рук. отдела → любой сотрудник своего отдела;
   - рук. проекта → любой из команды проекта.
3. **Запись вне команды НЕ блокируется:** сотрудник может проставить часы в проект, в команде которого официально не состоит. Не запрещаем.

---

## Реализация (когда возьмём, минимально)

- Шапка «Таймшит» → «Таймшит · {ФИО} · {Отдел}» (показ контекста). Источник имени — WorkspaceMember (для юзеров), credosTimeEmployee (профиль).
- Селектор (пп.2) виден по роли: isManager отдела → список сотрудников отдела; рук. проекта → команда проекта. Выбор → таймшит грузится с фильтром по выбранному employee.
- Запись (пп.3) — без проверки членства в команде (как сейчас, ничего не добавляем).

## Зависимости / границы
- **Зависит от server-identity (CISO-005):** «кто я» и «мой отдел/проекты» — нужно знать актора на сервере. Пока identity нет — селектор «за другого» небезопасен (любой сможет). Поэтому ПОСЛЕ RBAC-волны.
- Пункт 1 (показ «чей») — можно раньше, как только знаем текущего юзера (workspaceMemberRef).
- Не переусложнять: без иерархий-делегирований, только отдел/команда-проекта.
- Связь: RBAC-волна (роль «Сотрудник»/«Руководитель»), REQ-0001 (actor-резолв), REQ-0006 (согласование за период — туда же «за сотрудника»).
