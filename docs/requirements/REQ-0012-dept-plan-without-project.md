# REQ-0012 — Планируемая загрузка отдела без проекта (резерв/прочее)

**Статус:** PROPOSED → в реализацию (заказчик 2026-06-21)
**Приоритет:** 🟡

## Контекст
Сейчас загрузка в Планировании = только `project.plannedEffort` (по departmentId). Нельзя запланировать загрузку отдела **без привязки к проекту** (резерв под будущие сделки, общие/прочие работы). Заказчик: это допустимо, нужно поддержать + засеять демо.

## Сверка Timetta/Kimai
Timetta — non-project bookings / reserved capacity (бронь ёмкости без конкретного проекта), в т.ч. tentative под пресейл. Валидный PSA-паттерн.

## Решение (минимально, не переусложнять)
Объект **`credosTimeDeptPlan`**: `department`(RELATION) × `startDate`/`endDate`(период) × `plannedEffort`(часы) × `label`(TEXT, напр. «Резерв», «Пресейл-бронь», «Прочее») × опц. `category`(SELECT).
- Загрузка отдела = Σ project.plannedEffort (как сейчас) **+** Σ credosTimeDeptPlan (раскид по периоду так же).
- В «Детализация по проектам» — строки «Без проекта · {label}» рядом с проектами.
- index-view + nav (управляемо) ИЛИ через «Планировать» (позже).

## Реализация
- **Dev 2:** объект + view + nav + интеграция в `calc-load.ts`/reports (раскид как projectHoursInPeriod) + сид демо (несколько записей по разным отделам). 
- **Dev 1:** строки «Без проекта» в детализации capacity.

## Связи
CAPACITY_PLANNING §2b («без плана» — это другое: проекты без часов; здесь — план без проекта). REQ-0009 (пресейл-бронь — частный случай). Не переусложнять.
