# REQ-0007 — Согласование по периодам (неделя/месяц) + обзор статусов

**Статус:** PROPOSED (БЭКЛОГ — «на будущее», заказчик 2026-06-21)
**Приоритет:** 🟡 средний (UX согласования; после OLAP/настроек)

---

## Сверка с Timetta/Kimai (обязательная)

| Инструмент | Гранулярность согласования |
|---|---|
| **Timetta** | Таймшит **периодом** — сотрудник отправляет неделю/период целиком, руковод согласует период. |
| **Kimai** | **Weekly approval** — согласование по неделям + блокировка (lock) согласованного периода. |

**Вывод:** оба референса согласуют **периодом (неделя/месяц)**, не построчно. Наша текущая модель (статус на каждой записи) для UX согласования хуже → переходим на период-based. Интуиция заказчика подтверждена.

---

## Суть (что делаем когда возьмём)

1. **Согласование периодом.** Единица согласования — (сотрудник × период[неделя или месяц]), не отдельная запись. Сотрудник «Отправить неделю» → все записи недели → SUBMITTED разом; руковод «Согласовать/Отклонить» период.
   - Технически: либо объект `credosTimeApprovalPeriod` (employee×period×status), либо агрегирование статусов записей периода (вывод). Решить на этапе дизайна (вероятно отдельный объект — чище для lock + обзора).
2. **Признак «согласование не требуется».** На уровне отдела/проекта (`approvalRequired`, уже есть) — если не требуется, период помечается «не требуется», не висит как незакрытый.
3. **Обзор-матрица статусов.** Экран: **компания / отдел / человек** (строки) × **месяц** (колонки) → статус: ✅ согласовано · ⏳ на согласовании · ✍️ черновик/не заполнено · ⊘ не требуется. Свёртка компания→отдел→человек (drill, переиспользует OLAP-движок REQ OLAP). Руковод видит, у кого что не закрыто за месяц.
4. **Блокировка (lock)** согласованного периода — нельзя править после согласования (Kimai lock, Timetta). Связь с REQ-0006 (lock периода — табель — там же отмечено).

---

## Связи
- Меняет lifecycle согласования (сейчас построчно в `credosTimeEntry.status` + view «Согласование»). 
- Обзор-матрица переиспользует OLAP-агрегацию (фаза 02) — ось «статус согласования» × период.
- `approvalRequired` (Department) уже есть → признак «не требуется».
- RBAC: кто согласует (isManager) — связь RBAC-волна + CISO-002/005.
- Lock периода — связь REQ-0006.
- НЕ блокирует текущие волны. После OLAP + настройки.
