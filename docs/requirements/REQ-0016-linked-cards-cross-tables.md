# REQ-0016 — Связанность карточек: кросс-таблицы во вкладках (Проект↔Отдел↔Сотрудник)

**Статус:** PROPOSED (заказчик 2026-06-22) — после REQ-0011 + OLAP
**Приоритет:** 🔴 (навигируемость данных, дополняет OLAP drill-down)

## Суть
Карточки Проекта / Отдела / Сотрудника связаны между собой кросс-таблицами во вкладках — клик по строке ведёт в связанную карточку. Единая навигируемая сеть данных.

## Матрица вкладок (что в какой карточке)
| Карточка | Вкладки-кросс-таблицы |
|---|---|
| **Проект** | Отделы (доли — REQ-0013 ✓) · Команда (сотрудники+часы) · Этапы ✓ · Записи · Бюджет ✓ |
| **Отдел** | Сотрудники (FTE+даты — REQ-0011) · Проекты (через доли) · Загрузка/ёмкость · Записи отдела |
| **Сотрудник** | Отделы (FTE+даты — REQ-0011) · Проекты (где работал) · Мои часы/периоды (REQ-0014 ✓) |

- Каждая строка кросс-таблицы **кликабельна** → переход в карточку связанной сущности (record page).
- Данные из связей: credosTimeProjectDepartment (проект↔отдел), credosTimeEmployeeDepartment (сотрудник↔отдел), credosTimeEntry (часы), credosTimeStage.
- Переиспользовать паттерн «Отделы» в карточке проекта (REQ-0013 доли-в-карточку) для остальных.

## Связь с OLAP
OLAP drill-down (фаза-02) + кросс-таблицы карточек = одна навигируемая аналитика: дашборд → клик → карточка → связанная вкладка → клик → другая карточка. Использовать общий механизм перехода (record page / page-layout tab front-component, useRecordId).

## Реализация (поэтапно, паттерн карточных вкладок)
- **Dev 1:** front-component кросс-таблицы по матрице, в page-layout каждой карточки. Кликабельные строки → navigate(record). Переиспуёт shared/ui (после консолидации R2-4).
- **Dev 2:** при нужде — агрегаты для вкладок (часы по сотруднику проекта и т.п.) через /s/reports или REST.
- Зависит: REQ-0011 (employee×dept), REQ-0013 (project×dept ✓), OLAP-движок.

## Границы
Не переусложнять: переиспользуем существующие связи и паттерн вкладок. Без новых объектов (только новые view-вкладки). RBAC ПДн по людям — за CISO-005 (детализация по сотрудникам).
