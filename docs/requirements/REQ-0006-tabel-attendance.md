# REQ-0006 — Табель учёта рабочего времени (признаки явки)

**Статус:** PROPOSED (БЭКЛОГ — заказчик 2026-06-21: «можно потом»)
**Приоритет:** 🟢 после ядра (OLAP/настройки/RBAC)
**Источник:** запрос заказчика — нужно формировать **табель** (аналог Т-13).

---

## Суть

Сделать так, чтобы из данных модуля можно было собрать **табель учёта рабочего времени**. Для этого ввести опциональные признаки явки/неявки:

| Признак | Т-13 код | Где живёт | Статус в модели |
|---|---|---|---|
| Отсутствие / отпуск | О / ОЗ | `credosTimeAbsence.absenceType` | ✅ есть (VACATION/UNPAID/OTHER) |
| Больничный | Б | `credosTimeAbsence.absenceType` | ✅ есть (SICK) |
| **Командировка** | К | `credosTimeAbsence.absenceType` | ➕ добавить значение `BUSINESS_TRIP` |
| **Работа в выходной/праздник** | РВ | признак на записи / вывод из календаря | ➕ новое |
| **Переработка (сверхурочно)** | С | признак на записи / вывод из нормы | ➕ новое |

---

## Объём (когда возьмём)

1. **Командировка** — добавить `BUSINESS_TRIP` в `ABSENCE_TYPE_OPTIONS` (select-options.ts) + label «Командировка». Минимально — расширение существующего объекта отсутствий. (Dev 2)
2. **Работа в выходной (РВ)** — два варианта:
   - (a) автоматически: запись `credosTimeEntry` на дату, которая в `credosTimeWorkdayCalendar` = выходной/праздник → помечается РВ при сборке табеля (вывод, без нового поля). **Рекомендуется** — без дублирования данных.
   - (b) явный флаг `isWeekendWork` на записи — если нужен ручной контроль.
3. **Переработка (С)** — два варианта:
   - (a) вывод: суммарные часы дня > нормы дня (из календаря) → избыток = переработка (расчёт при сборке табеля). **Рекомендуется**.
   - (b) явный флаг/поле `overtimeHours` на записи — если нужен ручной ввод сверхурочных.
4. **Сборка табеля** — отчёт/экран «Табель»: сотрудники(строки) × дни месяца(колонки), в ячейке код (Я/О/Б/К/РВ/С) + часы. Источник: TimeEntry (часы/дни) + Absence (О/Б/К) + Calendar (рабочий/выходной → норма, РВ, С). Экспорт (связь F-F).

---

## Принцип
Признаки — **опциональные** (не усложняют обычный ввод часов). Большинство выводится из уже имеющихся данных (записи + календарь + отсутствия) → минимум новых полей. Командировка — единственное явное расширение справочника отсутствий.

## Связи
- Расширяет F-D (`credosTimeAbsence`), использует `credosTimeWorkdayCalendar` (норма/выходные/праздники) и `credosTimeEntry` (факт-часы).
- Экспорт табеля → F-F (CSV/Excel, под кадровый учёт / 1С:ЗУП).
- НЕ блокирует текущие волны. Берём после OLAP + настройки + RBAC.
