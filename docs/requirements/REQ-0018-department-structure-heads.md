# REQ-0018 — Структура отделов: руководитель + иерархия

**Статус:** PROPOSED (заказчик 2026-06-22) — после SCOUT-B (Dev2 занят)
**Приоритет:** 🔴 (питает approval-маршрутизацию + RBAC-скоупинг + isManager)

## Суть (заказчик)
Завести структуру отделов — кто чей руководитель. У отдела есть **руководитель** (сотрудник). Возможна **иерархия отделов** (родительский отдел).

## Модель
| Поле/связь | На объекте | Тип |
|---|---|---|
| `head` руководитель отдела | credosTimeDepartment → credosTimeEmployee | RELATION MANY_TO_ONE (nullable) |
| `parentDepartment` иерархия | credosTimeDepartment → self | RELATION MANY_TO_ONE (nullable, опц.) |
| (вывод) `isManager` | employee = head любого отдела → руководитель | вычисляемо ИЛИ синхр. с employee.isManager |

- **Руководитель отдела** = `department.head`. Сотрудник, который head ≥1 отдела → это «руководитель» (источник для isManager — связать с A2/approval).
- **Иерархия** (опц.): parentDepartment → дерево отделов (для скоупинга «руковод видит свой + подчинённые отделы»).

## Зачем (связи)
- **Approval-маршрутизация:** заявку/таймшит согласует head отдела сотрудника (REQ-0007, A2 isManager).
- **RBAC-скоупинг:** руковод видит свой отдел (+ дочерние) — закрывает CISO-007 по-человечную детализацию (после CISO-005 server-identity: head → видит ФИО своей команды).
- **isManager SSOT:** сейчас bool на employee (ручной); department.head = объективный источник «кто руковод».

## Реализация
- **Dev 2:** поля head + parentDepartment на credosTimeDepartment + обратные связи; backfill-скрипт (проставить heads по данным — если известны, иначе пусто/вручную); опц. синхр employee.isManager = (head хоть одного отдела). + view-поля.
- **Dev 1:** в карточке отдела — поле «Руководитель» (выбор сотрудника) + «Вышестоящий отдел»; в карточке сотрудника — «Руководит отделами» (обратная).
- Связь: A2 (isManager), REQ-0007 (согласование), CISO-005/007 (скоуп), ADR-0006 (модель сотрудника).

## Границы
Не переусложнять: head + опц. parent. Без матричной/множественной начальственности (один head на отдел). Многоуровневый скоуп — после base.
