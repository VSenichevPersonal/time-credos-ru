# REQ-0011 — FTE сотрудника по отделам + планируемая численность

**Статус:** PROPOSED (БЭКЛОГ — исследовать, заказчик 2026-06-21)
**Приоритет:** 🟡 (точность ёмкости/планирования)

---

## Контекст / правка текущего

Сейчас в Настройках «Числ.» (headcount отдела) — **редактируемое поле** (вводится). **Неверно:** численность должна **вычисляться**, не заноситься. → исправление (не бэклог): headcount = число активных сотрудников отдела (а в перспективе — Σ их FTE).

## Сверка Timetta/Kimai
- **Timetta/PSA:** ёмкость считается по **FTE** (доля ставки). Сотрудник может быть частично занят / делиться между подразделениями. Планирование ресурсов учитывает дробную доступность. → модель заказчика = стандарт.

## Бэклог (исследовать + предложить)

### 1. FTE сотрудника по отделам (карточка сотрудника)
- В карточке каждого сотрудника — **% FTE на отдел**. Сотрудник может: работать 100% в одном отделе; делиться 50/50 на 2 отдела; 3 отдела по долям и т.п.
- Модель (изучить): либо поля на сотруднике, либо join-объект **`credosTimeEmployeeDepartmentFte`** (employee × department × ftePercent) — чище для мульти-отдела. По умолчанию 100% на основной отдел.
- **Headcount отдела (вычисляемый)** = Σ ftePercent его сотрудников (дробный). Заменяет ручную «Числ.».
- Ёмкость отдела = headcount(FTE) × норма(календарь) × коэф.ёмкости.

### 2. Планируемая численность на будущие периоды
- Отдельный подраздел **«Планируемая численность»**: задавать численность отдела на будущие периоды с учётом **роста/сокращения**.
- **Дробная** (человек выходит с середины месяца → 0.5 за месяц; полставки → 0.5).
- Модель (изучить): объект **`credosTimePlannedHeadcount`** (department × период × planned FTE) или из дат найма/увольнения + FTE сотрудников.
- Capacity вперёд = planned headcount (если задан) иначе текущий → точный прогноз «когда отдел сможет взять проект» с учётом будущего штата.

## Связи / границы
