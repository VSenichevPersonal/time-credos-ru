# REQ-0011 — Сотрудник × отдел(ы) в % FTE с датами (в реализацию)

**Статус:** IN PROGRESS (заказчик 2026-06-22: спроектировать и сделать)
**Приоритет:** 🔴 (ядро ёмкости; зеркало REQ-0013 «Доли отделов» проекта)

## Суть (заказчик)
Сотрудник мапится на ОДИН или НЕСКОЛЬКО отделов в процентах, с **датой старта и датой финиша**. Пример: Иванов 100% ОИБ; Петров 50% ОИБ + 50% ОПИБ; новичок с 15.07; уволенный по 31.08.

## Сверка Timetta
Ёмкость по FTE, частичная занятость между подразделениями, даты найма/перевода/увольнения → доступность по дате. Стандарт PSA.

## Модель — join-объект `credosTimeEmployeeDepartment` (Вариант A, как project-department)
| Поле | Тип | Назначение |
|---|---|---|
| employee | RELATION→credosTimeEmployee (MANY_TO_ONE) | сотрудник |
| department | RELATION→credosTimeDepartment (MANY_TO_ONE) | отдел |
| ftePercent | NUMBER (0..100) | доля ставки в этом отделе |
| startDate | DATE_TIME | с какой даты действует (приход/перевод) |
| endDate | DATE_TIME nullable | по какую (null = бессрочно/текущий) |

- **Headcount отдела (вычисляемый, заменяет простой count):** Σ (ftePercent/100) сотрудников, у кого запись активна в периоде (`startDate ≤ конец периода И (endDate пуст ИЛИ endDate ≥ начало периода)`). Дробный.
- **Ёмкость отдела** = headcount(FTE, по дате) × норма(календарь) × coefФ. → на будущие периоды учитывает приходы/уходы (плановая численность вперёд = бесплатно из дат).
- **Fallback (обратная совместимость):** нет записей employee-department → старый count по `employee.departmentId` 100%.
