# REQ-0002 — Финансы: PNL по проекту

**Статус:** PROPOSED (бэклог, НЕ сейчас — по указанию arch)
**Источник:** arch волна-2 бэклог «Финансы»; референс Timetta (`research/timetta/`)
**Затрагивает:** новые поля/объекты ставок + `logic-functions/` (PNL), связка 1С (`credosTimeBillingLink`)

## Контекст

После связки с 1С появляется доходная часть (акты/оплаты по `credosTimeBillingLink`). PNL по проекту = доход − затраты. Затраты = Σ (часы сотрудника × внутренняя ставка). Сейчас ставок в модели НЕТ (billable-флаг удалён, осталась `WorkCategory`).

## Требование (черновик, уточнить при старте)

1. Внутренняя ставка сотрудника/роли (себестоимость часа) — новый объект или поле (решить: ставка по сотруднику vs по грейду/роли; матрица ставок как в Timetta `RateMatrices`).
2. Доход проекта — из `credosTimeBillingLink` (акты/оплаты) или плановый бюджет проекта.
3. `/s/pnl` (или расширение `/s/reports`): по проекту — доход, затраты (Σ часы × ставка), маржа, маржа %.

## Зависимости

- **Блок:** связка 1С (доходная часть) + модель ставок. До этого — только затратная оценка (часы × ставка), без дохода.
- Реюз агрегатов из [REQ-0003](REQ-0003-reports-aggregates-contract.md) (Σ часы по проекту/сотруднику).

## Открытые вопросы

- Ставка: по сотруднику, по грейду, или матрица (проект×роль)? — домен-решение с заказчиком.
- ПДн/конфиденциальность ставок — RBAC (только руководство видит PNL) → координация CISO.
