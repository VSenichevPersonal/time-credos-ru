# ADR-E1: Матрица ставок (Rate Matrix)

**Статус:** Проект
**Дата:** 2026-06-21

---

## Контекст

Для расчёта себестоимости и биллинга нужна система ставок. Референсная система использует best-match матрицу с 7 аналитиками.

## Решение

### Архитектура: Best-Match Matrix

Ставка определяется поиском строки с полным совпадением по набору аналитик. Каскадного наследования нет.

### Минимальный набор аналитик (MVP)
1. **Role** (роль исполнителя) — обязательно
2. **LegalEntity** (юрлицо) — обязательно

### Полный набор (целевой)
3. Level (должностной уровень)
4. Grade (грейд)
5. Competence (компетенция)
6. ResourcePool (ресурсный пул)
7. Location (локация)

### Два типа матриц
- **Billing Rate** — стоимость часа для клиента (→ счета, выручка)
- **Cost Rate** — внутренняя себестоимость часа (→ P&L, затраты)

### Алгоритм выбора ставки
1. Определить значения аналитик для контекста (кто, где, когда)
2. Найти строку с полным совпадением в активной матрице
3. Если нет совпадения → базовая ставка пользователя
4. Если нет пользовательской → стандартная ставка проекта

### Жизненный цикл матрицы
- Draft → Active → Archived
- На дату действует последняя активная версия

## Последствия

- Rate Matrix — отдельная сущность с versioning по датам
- Порядок аналитик в структуре матрицы важен (приоритет поиска)
- Матрицы привязаны к LegalEntity
- MVP: достаточно Role × LegalEntity = ставка

## Дополнения (из API-данных)

### RateMatrixLine — поля (проверено $metadata)
| Поле | Тип | Обязательно | Описание |
|------|-----|-------------|----------|
| rateMatrixId | Guid | да | → RateMatrix |
| roleId | Guid | нет | Роль |
| levelId | Guid | нет | Уровень |
| gradeId | Guid | нет | Грейд |
| competenceId | Guid | нет | Компетенция |
| locationId | Guid | нет | Локация |
| legalEntityId | Guid | нет | Юрлицо |
| resourcePoolId | Guid | нет | Ресурсный пул |
| rate | Decimal | да | Значение ставки |
| effectiveDate | Date | нет | Начало действия |
| expiryDate | Date | нет | Окончание |

### RateMatrix — поля
| Поле | Тип | Описание |
|------|-----|----------|
| typeId | Guid | → RateMatrixType (Billing/Cost) |
| effectiveDate | Date | Дата активации матрицы |
| expiryDate | Date | Срок действия |
| rateMatrixStructure | Collection(RateMatrixAnalytics) | Набор аналитик |
| lines | → RateMatrixLine[] | Строки матрицы |

### Реальные данные стенда
- Cost Matrix: 8 линий (Role × LegalEntity)
- Billing Matrix: 3 линии (Role × LegalEntity)
- Все остальные аналитики = null
- Это подтверждает MVP: достаточно Role + LegalEntity

### Важные механики
- «Заполнить матрицу» — авто-генерация всех комбинаций (удаляет старые строки!)
- RateBC — авто-конвертация в базовую валюту по курсу на EffectiveDate
- EffectiveDate + ExpiryDate — версионирование ставок по датам
