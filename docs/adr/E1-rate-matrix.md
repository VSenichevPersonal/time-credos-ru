# ADR-E1: Матрица ставок (Rate Matrix)

**Статус:** Проект
**Дата:** 2026-06-21

---

## Контекст

Для расчёта себестоимости и биллинга нужна система ставок. Референсная система использует best-match матрицу с 7 аналитиками.

## Решение

### Архитектура: Best-Match Matrix

Ставка определяется поиском строки с полным совпадением по набору аналитик. Каскадного наследования нет.

### Минимальный набор аналитик (MVP)
1. **Role** (роль исполнителя) — обязательно
2. **LegalEntity** (юрлицо) — обязательно

### Полный набор (целевой)
3. Level (должностной уровень)
4. Grade (грейд)
5. Competence (компетенция)
6. ResourcePool (ресурсный пул)
7. Location (локация)

### Два типа матриц
- **Billing Rate** — стоимость часа для клиента (→ счета, выручка)
- **Cost Rate** — внутренняя себестоимость часа (→ P&L, затраты)

### Алгоритм выбора ставки
1. Определить значения аналитик для контекста (кто, где, когда)
2. Найти строку с полным совпадением в активной матрице
3. Если нет совпадения → базовая ставка пользователя
4. Если нет пользовательской → стандартная ставка проекта

### Жизненный цикл матрицы
- Draft → Active → Archived
- На дату действует последняя активная версия

## Последствия

- Rate Matrix — отдельная сущность с versioning по датам
- Порядок аналитик в структуре матрицы важен (приоритет поиска)
- Матрицы привязаны к LegalEntity
- MVP: достаточно Role × LegalEntity = ставка
