# ADR-0007 — Норма рабочих часов: единый источник

**Статус:** ACCEPTED (2026-06-22)
**Контекст:** аудит консолидации (CONSOLIDATION_PLAN Y1/T2) выявил расхождение источников нормы.

## Проблема
Норма часов считается в ДВУХ местах по-разному:
- **Фронт (таймшит):** `grid/format.ts` — `DAILY_NORM_HOURS = 8` (фикс), неделя = 40ч.
- **Сервер (/s/reports, capacity):** `reports-calc.ts` — норма = Σ часов РАБОЧИХ дней из производственного календаря РФ (`credosTimeWorkdayCalendar`), праздники/предпраздничные учтены.

→ В праздничные/короткие недели **сетка показывает 40ч, дашборд — меньше**. Сетка ≠ дашборд = потеря доверия пользователя.

## Решение
**Единый источник нормы — производственный календарь РФ** (`credosTimeWorkdayCalendar`), как на сервере. Фронт-таймшит ДОЛЖЕН брать норму дня/недели из календаря, а не из фикс-константы `DAILY_NORM_HOURS=8`.

- Норма дня = часы рабочего дня из календаря (8 / 7 короткий / 0 выходной-праздник).
- Норма недели = Σ норм дней недели.
- Фикс-константа `DAILY_NORM_HOURS` остаётся только как **fallback**, если календарь недоступен (degraded), с пометкой.
- Pre-submit валидация (REQ-0015) и «недобор» в сетке считаются от календарной нормы → совпадают с дашбордом.

## Реализация (рефактор, не новая фича)
- Фронт грузит календарь (уже грузится в capacity; для таймшита — добавить или переиспользовать) → норма дня из него.
- `grid/format.ts` норму берёт из календаря-данных, не из константы.
- Зона: Dev2 (контракт нормы/календарь) → Dev1 (потребление в сетке). Последовательно.

## Альтернативы (отклонены)
- Оставить фикс-40ч на фронте — проще, но сетка≠дашборд (текущий баг). Отклонено.
- Считать норму только на сервере и отдавать фронту по REST — вариант, но календарь уже доступен клиенту; лишний round-trip.

## Связи
