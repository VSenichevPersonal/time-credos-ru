# ADR-0005. Прод-топология — отдельный Twenty 2.14 для time + синк Company по API

**Статус:** ACCEPTED (arch CONFIRMED 2026-06-20 22:05, Стратегия C)
**Дата:** 2026-06-20
**Связано:** [ADR-0001](0001-platform-and-data.md) (платформа Twenty/один workspace), [ADR-0002](0002-sdk-app-isolated-repo.md) (app=install-юнит), [ADR-0003](0003-catalog-separate-app-shared-master-data.md) (общие мастер-данные), [ADR-0006](0006-employee-model.md), CISO-004, 152FZ-001, `docs/devops/UPSTREAM_SYNC_ASSESSMENT.md`, `docs/security/PII_152FZ_REVIEW.md`

## Контекст

Два инстанса Twenty (поднял заказчик):
1. **CRM-форк CredosCRM1** = **Twenty v1.19.0** (SDK 0.7.0, точка форка 13 марта; +851 коммит, ~3 мажорные линии отрыва от upstream).
2. **Наш dev** = **чистый Twenty 2.14** (Railway, «Twenty Credos Time»).

Наш app требует **Twenty ≥2.14** → **на форк v1.19 не встанет**. Следствие: пока CRM = v1.19, app в его workspace не инсталлируется → **`Company` не шарится напрямую** (ADR-0003 «общие мастер-данные» не выполним между разными версиями).

Доп. ограничение (152-ФЗ): dev на **Railway вне юрисдикции РФ** (нет РФ-региона) → для прода это **блокер ст. 18.5** (локализация БД ПДн граждан РФ) — см. 152FZ-001 (P0 прод).

## Решение

**Стратегия C** (по `UPSTREAM_SYNC_ASSESSMENT.md` §4, рекомендована DevOps): time-app разворачивается на **отдельном чистом инстансе Twenty 2.14** (клон dev-конфигурации), **в РФ-контуре** (не Railway). `Company` (и при необходимости штат) **синхронизируются по REST API** между CRM-форком и time-инстансом, не шарятся через общую БД. Апгрейд форка CredosCRM1 до 2.x (Стратегия A rebase или B пере-форк) — **отдельный независимый трек**, не блокирует time-app.

- time-app **не ждёт** полного upstream-sync форка (апгрейд CRM v1.19→2.x — отдельный трек).
- Соответствует ADR-0002 (app = самостоятельный install-юнит).
- `ENCRYPTION_KEY` + `APP_SECRET` зафиксировать до старта; `yarn twenty app:install`.

## Альтернативы (отклонены)

1. **A. Апгрейд CRM-форка до 2.x сейчас** — отклонено (на данном этапе): ~3 мажорные линии, 851 коммит форка (573 трогают `credos/`) + расходящийся upstream → дорого и рискованно. Делаем отдельным треком, не блокируя time.
2. **Прямой shared-DB install в форк (ADR-0003 буквально)** — невозможно между v1.19 и 2.14 (несовместимость схемы/SDK).
3. **Оставить прод на Railway** — отклонено: нарушает 152-ФЗ ст. 18.5 (локализация ПДн в РФ). Прод-таргет обязан быть РФ-контур.

## Последствия

**+** time-app разблокирован для прода независимо от темпа апгрейда форка. Чистый 2.14 = совместимость с актуальным SDK.
**−** `Company`/штат дублируются между инстансами → нужен **синк по API** (направление, частота, ключ матчинга — отдельная задача DevOps/Dev2). Пересекается с ADR-0003/ADR-0006/CISO-004 (дубль ПДн, межинстансная видимость).
**Прод-гейты (до запуска):** (1) РФ-контур хостинга (152FZ-001 P0); (2) локализация РФ + ЛНА (152-ФЗ); (3) `ENCRYPTION_KEY`/`APP_SECRET`; (4) модель синка Company.
