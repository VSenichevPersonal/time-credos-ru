**Статус:** принятый план

**Назначение:** наполнить систему данными для разработки/демо: реальная выборка ОВ + сгенерированные мок-данные по остальным отделам.

## 1. Принцип

| Отдел | Источник данных | Тип |

Данные внутренние (Кредо-С) — реальные имена ОВ оставляем как есть; мок-сотрудники др. отделов — выдуманные правдоподобные.

## 3. Мок-данные ОИБ / ОПИБ / ТЦ (генерация — ПОЗЖЕ)

**Порядок:** сначала разбираем и грузим реальные данные ОВ → фиксируем модель на них → **потом синтезируем мок других отделов ПО ОБРАЗЦУ модели данных ОВ** (та же структура: Project/Stage/Activity/Employee/TimeEntry/category), подставляя справочники услуг ОИБ/ОПИБ/ТЦ из каталога.

ОВ — эталон структуры. Мок других отделов = та же модель, другое наполнение (виды работ, проекты, сотрудники, клиенты под профиль отдела).

Цель — реалистичный объём и структура за тот же период, основанные на каталоге услуг.

## 4. Клиенты (Company) для мока

Мок-проекты привязываем к:

- мок-компаниям с правдоподобными названиями (для ИБ — условные «Банк X», «Завод Y», госорганизация — под профиль услуг ОИБ/ОПИБ/ТЦ).

## 5. Технически
- Генератор — отдельный скрипт (Python/TS) в этом репо, на выходе — записи под модель `defineObject`.
- В SDK-app сид через `definePostInstallLogicFunction` (post-install) либо отдельная загрузка.
- Период фиксирован → данные детерминированы (seed для random — постоянный, без `Math.random()` в самом app).
